# 下一任智能体说明（交接文档）

本文档供接手本项目的智能体（或开发者）快速了解当前状态与约定操作，尤其是**封闭环境测试**的流程。

---

## 一、项目概览

- **项目名称**：直播电商中台管理系统（lvbcsym）
- **技术栈**：前端 React + Vite + TypeScript；后端 Node + Express + SQLite
- **关键路径**：
  - 前端：`frontend/`，入口 `frontend/src/main.tsx`
  - 后端：`backend/`，入口 `backend/src/index.ts`
  - 工作流引擎：`backend/src/workflow_engine/`
- **启动**：根目录 **`快速启动.bat`**（最简单，推荐），或 `启动.ps1`（功能完整），或分别 `cd backend && npm run dev`、`cd frontend && npm run dev`
- **访问**：前端 http://localhost:5173，后端 http://localhost:3000，健康检查 http://localhost:3000/health

## 二、封闭环境测试约定（重要）

为**保护原项目**，封闭环境测试在**独立副本**中进行，**不直接在本仓库根目录做破坏性测试**。

### 用户侧约定

- **用户会手动新建一个文件夹**（例如 `lvbcsym-closed-test` 或任意名称），用于封闭测试。
- 该文件夹可能放在本仓库外（如 `d:\Work space\lvbcsym-closed-test`）或仓库内（如 `d:\Work space\lvbcsym\closed-test`），以用户实际创建为准。

### 智能体侧操作（当用户提出“做封闭环境测试”或“在副本里测试”时）

1. **确认目标目录**  
   向用户确认：封闭测试用的文件夹完整路径（即用户已手动创建好的那个目录）。

2. **复制项目到该目录**  
   - 将当前项目**除以下项外**的全部内容复制到用户指定的目录：  
     - 排除：`.git`、`node_modules`（根目录及 `backend`、`frontend` 下的）  
     - 若用户指定的目录内已有内容，按用户要求决定是否先清空再复制。
   - Windows 示例（PowerShell）：  
     ```powershell
     robocopy "d:\Work space\lvbcsym" "用户指定的路径" /E /XD .git node_modules "用户指定的路径" /NFL /NDL /NJH /NJS
     ```  
     注意：需把「用户指定的路径」替换为实际路径；若副本建在项目内（如 `closed-test`），需在 `/XD` 中加入该文件夹名，避免递归复制自身。

3. **在副本中提供简短说明**  
   在复制后的目录根下新建或更新一份**封闭环境测试说明**（如 `封闭环境测试说明.md`），内容建议包含：  
   - 说明本目录为原项目的测试副本，原项目路径与用途；  
   - 在副本内安装依赖：根目录、`backend`、`frontend` 分别 `npm install`；  
   - 启动方式（与主项目一致，如 `快速启动.bat` 或分别启动前后端）；  
   - 测试入口与测试账号（见下）；  
   - 注明：副本内可随意修改/重置，不影响原项目。

4. **测试执行参考**  
   全量功能测试步骤以项目内 **《完整功能测试指南.md》** 为准；智能体在副本中执行测试或指导用户执行时，直接引用该文档即可。
