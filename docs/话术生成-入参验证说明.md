# 话术生成 - 入参验证说明

本文档以**执行工具 → 话术生成**界面截图中的产品参数为基准，验证前后端入参是否一致、是否被正确使用。

---

## 一、截图中的产品参数（入参）

| 表单项 | 截图中的值 | 对应 API 字段 | 说明 |
|--------|------------|---------------|------|
| 产品名称* | 猫笼 | `productName` | 必填 |
| 产品SKU（可选） | 空 | `productSku` | 不传或 undefined |
| 价格（可选） | 空 | `price` | 不传 |
| 产品特点（可选） | 不锈钢 | `features` | |
| 目标人群（可选） | 多猫家庭 | `targetAudience` | |
| 话术类型 | 完整销售流程话术(推荐) | `scriptType` | 前端选项 value 为 `full-sales` |
| 营销方案/促销活动（可选） | 空 | `promotion_info` / `promoCopy` | 不传 |
| 店铺 | 测试-宠物 | `storeId` | 为当前选中店铺 id |
| 界面语言 | CN 中文 | `language` | 由 locale 推导为 `zh-CN` |

---

## 二、前端发送验证

**位置**：`frontend/src/components/AIFeatures.tsx`，生成按钮回调内调用 `generateScriptStream({ ... })`。

- `productName`: `scriptForm.productName.trim()` → **"猫笼"** ✓  
- `productSku`: `scriptForm.productSku.trim() || undefined` → **undefined** ✓  
- `price`: `scriptForm.price.trim() || undefined` → **undefined** ✓  
- `features`: `scriptForm.features.trim() || undefined` → **"不锈钢"** ✓  
- `targetAudience`: `scriptForm.targetAudience.trim() || undefined` → **"多猫家庭"** ✓  
- `scriptType`: `scriptForm.scriptType` → **"full-sales"**（选项 value 为 `full-sales`）✓  
- `promoCopy`: 仅当话术类型为 closing/promotion/full-sales 时传，为空则 **undefined** ✓  
- `storeId`: `selectedStore.id` → **测试-宠物 的店铺 id** ✓  
- `language`: `scriptLanguageFromLocale(locale)` → **zh-CN** ✓  

结论：截图中的产品参数与前端发往 `POST /api/ai/script/stream` 的 body 一致，**已实现**。

---

## 三、后端接收与使用验证

**位置**：`backend/src/routes/ai/script.ts` → `parseScriptRequestBody(req.body)`。

- `productName` → `effectiveProduct` / `userInput.productName` → **"猫笼"** ✓  
- `productSku` → `userInput.productSku` → **undefined** ✓  
- `price` → `userInput.price` → **undefined** ✓  
- `features` → `userInput.features` → **"不锈钢"** ✓  
- `targetAudience` → 与店铺 `targetAudience` 二选一 → **"多猫家庭"** ✓  
- `scriptType` → 支持 `full_process` 映射为 `full-sales` → **full-sales** ✓  
- `promotion_info` / `promoCopy` → `promotionInfo`，空则 **undefined** ✓  
- `storeId` → 用于查询店铺，得到 `storeContext`（名称、region、品类等）✓  
- `countryCode` / `country` → 未在表单填写时由店铺 `region` 或 `language` 推导 ✓  

**市调与提示词**：

- `runScriptResearch(userInput, storeContext)` 使用上述 `userInput` 与店铺上下文。  
- 因 `promotionInfo` 为空，`useCozePrompts = false`，使用 `buildLLMSystemPrompt` + `buildLLMUserMessage`（非 Coze 风格提示词）。  
- 市调摘要与提示词中会包含：产品名 猫笼、特点 不锈钢、目标人群 多猫家庭、话术类型 完整销售流程、店铺 测试-宠物 的品类与数据（若有）。  

**展示**：

- LLM 返回全文后经 `extractScriptBodyForDisplay(content)` 提取「完整话术」正文再给终端展示。  

结论：截图中的产品参数在后端**已完整接收并参与市调与生成**，**已实现**。

---

## 四、输出与截图对照

截图中的生成结果为「直播话术框架(5-15分钟循环)」及六段：【圈人群】【塑品】【打消顾虑】【利益点】【售后保障】【逼单】；内容中体现「不锈钢猫笼」「多猫家庭铲屎官」，与入参 产品名称、产品特点、目标人群、话术类型 一致，说明**参数已生效**。

若配置了话术 LLM，最终展示为 Coze 原生返回的完整内容（含优化说明、产品信息表、话术正文等），仅做长度截断。

## 五、结论

以图片中的产品参数（猫笼、不锈钢、多猫家庭、完整销售流程话术、测试-宠物、无价格/SKU/营销方案）入参：

- 前端：表单与 `generateScriptStream` 传参**一致且已实现**。  
- 后端：解析、市调、提示词、LLM 调用、展示过滤**均使用上述参数，已实现**。  

如需同时走 Coze 风格提示词（仅中文 + 国家/货币等），可在「营销方案/促销活动」中填写任意非空内容，或后续在产品上增加「强制使用 Coze 提示词」选项。
