# 月同比待办生成规则说明

**功能**：基于月同比数据（按自然日对比），自动生成数据分析类待办  
**类型**：规则生成（不调用 LLM）  
**触发条件**：店铺有至少 2 个月的运营数据

---

## 📅 月同比定义

**月同比（Month over Month）**：按自然日区间对比上月同期数据。

### 对比规则

| 当前区间 | 对比区间 | 说明 |
|---------|---------|------|
| 2025-02-01 ~ 2025-02-10 | 2025-01-01 ~ 2025-01-10 | 同样的自然日（月初10天） |
| 2025-02-15 ~ 2025-02-20 | 2025-01-15 ~ 2025-01-20 | 同样的自然日（月中） |
| 2025-03-01 ~ 2025-03-31 | 2025-02-01 ~ 2025-02-28 | 同样的日期范围（自动处理天数差异） |
| 2025-01-01 ~ 2025-01-15 | 2024-12-01 ~ 2024-12-15 | 自动处理跨年 |

**关键特性**：
- ✅ 按**自然日**对比（不按周）
- ✅ 自动处理跨年（1月 → 上年 12月）
- ✅ 自动处理月份天数差异（3月31日 → 2月最后一天）
- ✅ 确保对比相同的日期区间（如都是月初、月中、月末）

## 💻 代码实现

### 核心函数

```typescript
// backend/src/routes/ai-refactored.ts

/**
 * 获取上月同期数据（按自然日区间对比）
 */
async function getMonthOverMonthStats(
  storeId: string,
  currentDateFrom: string,  // 如 '2025-02-01'
  currentDateTo: string     // 如 '2025-02-10'
): Promise<StatsAggregate | null>

/**
 * 基于同比环比生成待办（规则生成，不调用 LLM）
 */
function generateComparisonTasks(
  currentStats: StatsAggregate,
  yoyStats: StatsAggregate | null,  // 年同比数据
  momStats: StatsAggregate | null,  // 月同比数据
  storeInfo: any
): Array<{ title: string; description: string; priority: string; source: string }>
```

### 集成位置

在 `generateSuggestedTodosForStore` 的主流程中，LLM 生成后追加：

```typescript
// 增加月同比/年同比待办（规则生成，不调用 LLM）
const yoyStats = await getYearOverYearStats(storeId, dateFrom, dateTo)
const momStats = await getMonthOverMonthStats(storeId, dateFrom, dateTo)
const comparisonTasks = generateComparisonTasks(currentStats, yoyStats, momStats, storeInfo)
if (comparisonTasks.length > 0) {
  tasks.push(...comparisonTasks)
}
```

## 🔄 与 LLM 待办的配合

### 生成顺序

1. **LLM 智能待办**（优先）  
   - 基于按日明细深度分析
   - 结合店铺上下文
   - 个性化、可执行性强

2. **月同比待办**（补充）  
   - 规则生成，数据驱动
   - 识别明显数据异常（下降/增长超过阈值）
   - 提供对比视角

3. **阶段待办**（兜底）  
   - LLM 失败时的兜底
   - 基于店铺阶段的通用建议

### 去重策略

- 月同比待办与 LLM 待办会进行去重（标题前 6 字相同视为重复）
- 保留 LLM 待办（优先级更高）
- 月同比待办作为补充

## 📌 使用场景

### 场景 1：新店铺（首月运营）

**数据情况**：只有当月数据，无上月数据  
**月同比待办**：0 条（无对比基准）  
**其他待办**：LLM 智能待办 + 阶段待办

### 场景 2：第 2 个月运营

**数据情况**：当月 + 上月数据  
**月同比待办**：根据变化幅度生成 0-6 条  
**其他待办**：LLM 智能待办 + 月同比待办 + 阶段待办

### 场景 3：成熟店铺（多月数据）

**数据情况**：连续多月数据  
**月同比待办**：月同比 + 年同比（如果有去年数据）  
**其他待办**：LLM 智能待办 + 同比待办 + 阶段待办
