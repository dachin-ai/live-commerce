# 提示词模板优化记录

**日期**：2026-02-10  
**目的**：简化提示词、放宽格式限制、扩展数据维度、移除冗余上下文

---

## 🎯 优化内容

### 1. 强调直播场景聚焦 ✨ 最新

**新增说明**（在 systemPrompt 中）：
```
【直播场景要求】生成的待办必须与**直播强相关**，聚焦直播运营场景：直播内容优化、直播话术、直播节奏、直播转化、直播时段、直播商品推荐、直播互动等。禁止生成与直播无关的营销建议（如线下活动、社交媒体广告、SEO优化等）。
```

**原因**：确保 Coze 生成的待办聚焦在直播运营上，而不是偏离到其他营销渠道。

**示例对比**：
- ❌ 不符合：「开设线下体验店」、「投放 Facebook 广告」、「优化网站 SEO」、「建立电商平台旗舰店」
- ✅ 符合：「优化直播话术提升转化率」、「调整直播时段匹配流量高峰」、「在直播中增加商品推荐环节」、「分析高 GMV 直播场次的成功要素」

---

### 2. 放宽输出格式限制

**旧版**：
```
【输出格式】有且仅有一个 JSON 对象，直接以 {"tasks":[ 开头，无任何前缀或后缀。
两种格式均可：① {"tasks":[{"title":"标题","description":"描述","priority":"urgent或normal"}]}；
② {"tasks":[{"task":"标题","expected_outcome":"预期效果","action_steps":["步骤1","步骤2"],"priority":"high或normal"}]}
（high 等同 urgent）。条数控制在 6～10 条。每条须结合上述数据写出具体数字与可执行动作。只输出 JSON。
```

**新版**：
```
【输出格式】返回一个包含 tasks 数组的 JSON 对象。每条任务需包含：标题、描述（可执行动作）、优先级。条数控制在 6～10 条。只输出 JSON，无任何前缀或后缀。
```

**原因**：过于详细的格式规定限制了 Coze 的灵活性，简化后让 Coze 按其擅长的方式输出。

**后端增强**：
- 支持多种 JSON 格式：`{"tasks":[...]}`, `{tasks:[...]}`, `{ "tasks": [...]}`, `[{...}]`
- 支持多种字段名：title/task/name/content, priority/level/importance/high/critical

### 2. 移除店铺阶段判断

**旧版**：
```
【核心销售指标（最近30天）】
- 总订单数：636 单
- ...
- 店铺阶段：成熟期  ← 删除
```

**新版**：
```
【核心销售指标（最近30天）】
- 总订单数：636 单
- ...
（无阶段）
```

**原因**：当前阶段判断逻辑（基于 GMV、时长、场次）不够合理，Coze 可以基于完整数据自行判断店铺所处阶段。

### 3. 扩展按日数据维度

**旧版**（9 列）：
```
日期 GMV 时长(h) 观看 订单 互动 场次 转化率(%) 时均GMV
```

**新版**（19 列）：
```
日期 GMV 时长(h) 观看 在线 订单 完成 互动 点赞 评论 分享 关注 商品曝光 商品点击 场次 转化率(%) 点击率(%) 互动率(%) 时均GMV
```

**新增维度**：
- 互动细分：点赞、评论、分享、关注
- 商品数据：曝光次数、点击次数
- 转化细分：点击率、互动率
- 订单细分：在线人数、完成订单数

**好处**：
- 可分析互动结构（点赞多但评论少 → 内容吸引力有限）
- 可识别订单履约问题（订单 vs 完成 → 完成率异常）
- 可评估商品推荐效率（曝光 vs 点击 → CTR 异常）

### 4. 精简业务上下文

**旧版**（冗长）：
```
【业务上下文】
- 阶段与重点：成熟期，关注：品牌建设、私域运营、供应链优化
- 趋势分析：stable（近期GMV与转化较稳定）
- 数据异常：...
- 时间与季节：冬季 2月 | 气温：热带旱季（...户外与旅游、防晒、补水...）
- 即将节日：情人节4天后
- 已有待办：...
```

**新版**（精简）：
```
【业务上下文】
- 已有待办：...（仅在有已有待办时显示）
```

**原因**：阶段、趋势、季节、节日等逻辑已内置给 Coze，无需在提示词中赘述。

### 5. 删除生成要求

**旧版**：
```
【生成要求】
请基于以上完整数据（特别是⭐按日明细），按「最近发展区」生成 6～10 条待办：
1. 结合具体数字（如"转化率 3.6%"、"12月12日 GMV 最高"）
2. 给出可执行动作（如"在高峰时段增加 XX 商品推广"）
3. 考虑业务上下文（阶段、节日、气温、趋势）
4. 优先当下最应投入的任务（非泛泛建议）
直接输出 JSON，以 { 开头。
```

**新版**：删除整个部分

**原因**：这些要求已在 systemPrompt 和"最近发展区"定义中说明，重复赘述会增加 token 消耗且限制 Coze 发挥。

### 6. 放宽内容长度限制

| 项目 | 旧限制 | 新限制 | 说明 |
|------|--------|--------|------|
| **标题** | 50 字符 | 100 字符 | 支持更详细的标题（如"复制最佳日高转化策略（12-23转化率11.15%）"） |
| **描述** | 200 字符 | 2000 字符 | 保留完整的分析内容（现状、问题、行动、ROI、时间节点等） |

**测试验证**：
- Coze 输出描述 465 字符 → 后端解析 465 字符 → 数据库存储 465 字符 → 前端显示 465 字符 ✅

## 🚀 使用指南

### 前端重新生成

1. 删除现有待办（如果是旧版本生成的，描述被截断为 200 字符）
2. 点击「智能生成」
3. 新生成的任务将包含**完整描述**（最多 2000 字符）

### 数据重新导入（可选）

如需要 19 列扩展数据：
1. 在前端「数据导入」上传包含扩展列的 Excel
2. 系统会自动识别点赞、评论、分享、商品曝光/点击等字段

## 📊 改进效果

### 旧版问题
- ❌ 描述被截断为 200 字符，丢失 ROI、时间节点等关键信息
- ❌ 格式限制过严，Coze 必须精确匹配字段名
- ❌ 提示词冗长（含大量赘述），token 消耗高

### 新版优势
- ✅ 完整保留 Coze 输出（最多 2000 字符描述）
- ✅ 灵活解析多种 JSON 格式
- ✅ 提示词精简约 40%，聚焦核心数据
- ✅ 19 列扩展数据支持更精准分析
