## 三、新功能建议（规范，非单次任务）

| 规范 | 说明 |
|------|------|
| 新 API 独立文件 | 不继续堆在 ai-refactored.ts；新能力单独路由/文件再挂载 |
| 新接口权限 | 统一 authenticate；仅管理员则 requireAdmin，文档写明 |
| 契约先行 | 新接口请求/响应类型或 OpenAPI 先定再实现 |
| 店铺页面 | 需当前店铺的页面用 StoreProvider，与现有一致 |
| 角色实现 | 与 isAnchor/isOperator/isManager、预留接口说明对齐 |
| 长请求 | 前端 loading + 超时或轮询/SSE |
| 模块测试 | 新功能纳入 module-test-workflow、各模块步骤总结 |

*以上为「新功能落地时须遵守」的条目，每轮分工不单独勾选，但评估时可检查是否违反。*

## 五、与评估循环的关系
