# 简化流程：评估与分工循环

本流程将协作简化为**固定循环**：AI 负责评估与分工（前端），你负责具体角色工作交互；每轮结束后进入下一轮，循环往复。

---

## 一、循环示意

```
┌─────────────────────────────────────────────────────────────────┐
│  【AI】全面评估系统 → 收集各角色进度 → 分发下一步工作内容（前端）   │
└───────────────────────────────┬─────────────────────────────────┘
                                │
                                ▼
┌─────────────────────────────────────────────────────────────────┐
│  【你】具体角色工作交互（在前端/代码中执行、测试、反馈）              │
└───────────────────────────────┬─────────────────────────────────┘
                                │
                                ▼
                    你说「进入下一轮」或「本轮完成」
                                │
                                └──────────────► 回到【AI】开始下一轮
```

---

## 二、每轮 AI 做什么（规划师视角）

### 1. 全面评估系统

- **模块进度**：对照《各模块步骤总结》看 1～13 模块状态（已完成 / 待开始 / 进行中）。
- **前端现状**：关键页面与组件（Dashboard、分析、执行工具、店铺/任务/布局）是否有明显缺口或已知 bug。
- **后端与数据**：API 是否正常、数据库/工作流产出是否可用。
- **系统层面问题（规划清单）**：对照 **《系统问题-规划清单》**（`docs/系统问题-规划清单.md`）检查：
  - **冲突与不一致**：文档与代码（ai 路由、限流、备份文件）、权限（version-logs、config、auth /me）、类型（Task、前后端契约）、单文件过大（ai-refactored.ts）。
  - **优化方向**：后端限流/缓存/日志、前端超时与错误提示、角色与预留接口对齐、文档分类与归档。
  - **新功能建议**：新 API 独立文件、authenticate/requireAdmin、契约先行、StoreProvider/角色对齐、纳入模块测试流程（见清单中的「新功能建议」规范）。
  - **短期优先项**：权限收紧、文档与代码统一、Task 类型对齐、模块 8～10 角色矩阵规划。
- **风险与阻塞**：上述清单中仍为「未开始」或「进行中」且会阻塞本轮的事项。

### 2. 收集各角色进度

- **来源**：你口头/文字反馈（如「规划师已定主题」「程序工程师部署包已出」「小白这轮疑问是…」），或 AI 从仓库/产出目录推断（如 `outputs/第N轮迭代/`、最近提交、文档更新）。
- **产出表格**：各角色本轮的「进度」与「卡点」一栏表，便于下一轮分工不重复、不遗漏。

### 3. 分发下一步工作内容（前端）

- **范围**：以**前端**为主（页面、组件、交互、联调），必要时附带后端或配置小任务；**可包含「关闭规划清单项」的任务**（如：对齐 Task 类型、定角色矩阵）。
- **形式**：按**角色**或按**模块/页面**给出「下一步任务清单」，每条可执行、可验收。
- **优先级**：先填《系统问题-规划清单》中的短期优先项与未关闭冲突项，再填已开模块的缺口和你的卡点，最后开新模块（如角色权限 8～10）。

---

## 三、每轮你做什么（具体角色工作交互）

- **规划师**：定主题/路线图（可在文档或工作流页操作）。
- **行业专家 / 资深用户**：按当前主题与方案，在前端实际使用产品（看板、分析、工具页），记录体验与 A/B 想法。
- **程序工程师**：按分工做前端开发或联调（改组件、调接口、修样式），或跑部署/脚本。
- **小白**：提疑问、暴露盲点（可写进 `疑问库.md` 或直接发给我）。

你只需在前端/代码里**做具体事**，进度与卡点在下轮开始时告诉我（或由我根据仓库/产出推断）。

---

## 四、如何进入下一轮

- 你 say：「进入下一轮」「本轮完成，下一轮」「开始下一轮」等。
- AI 将：  
  1）再次做**全面评估系统**；  
  2）**收集各角色进度**（你的反馈 + 仓库/产出）；  
  3）**分发下一步工作内容（前端）**；  
  4）你继续做**具体角色工作交互**。  
- 可选：每轮 AI 把「本轮评估 + 分工」写入 `docs/循环记录/第N轮-评估与分工.md`，便于追溯。

---

## 五、小结

| 步骤 | 负责方 | 动作 |
|------|--------|------|
| 全面评估系统 | AI | 模块进度、前端/后端现状、风险 |
| 收集各角色进度 | AI | 你的反馈 + 仓库/产出推断 |
| 分发下一步工作内容（前端） | AI | 可执行任务清单（按角色或按页面） |
| 具体角色工作交互 | 你 | 在前端/代码中执行、测试、反馈 |
| 进入下一轮 | 你 | 说「进入下一轮」→ AI 重新评估与分工 |

循环即可按此简化流程持续进行。
