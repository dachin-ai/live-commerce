## 🔄 重新导入步骤

### 1. 准备 Excel 文件

确保您的 Excel 中包含以上列名（中文或英文均可）。例如：

| 日期 | GMV | 直播时长 | 观看 | 订单 | 点赞 | 评论 | 分享 | 商品曝光次数 | 商品点击次数 | ... |
|------|-----|----------|------|------|------|------|------|--------------|--------------|-----|

**注意**：
- 系统会**自动识别**多种列名格式
- 如果 Excel 没有某列（如"商品曝光"），该字段会存为 0
- "总互动"可以省略，系统会自动用 点赞+评论+分享 计算

### 2. 在前端重新导入

1. 进入「店铺管理」→ 选择店铺
2. 点击「数据导入」
3. 选择包含扩展字段的 Excel 文件
4. 点击「开始导入」

系统会：
- **删除该店铺在相同日期范围内的旧数据**
- 重新解析并保存扩展字段
- 显示导入摘要（含新字段的总数）

### 3. 验证导入结果

导入后，可以查询验证：

```bash
# 在 backend 目录运行（需 PostgreSQL 客户端）
psql -U postgres -d live_commerce -c "SELECT * FROM stats WHERE \"storeId\" = '你的店铺ID' ORDER BY date DESC LIMIT 1;"
```

应该能看到 `likes`, `comments`, `shares`, `productViews` 等字段有值。

## 📝 LLM 输入格式变化

### 旧格式（9列）
```
日期	GMV	直播时长(h)	观看	订单	互动	场次	转化率(%)	时均GMV
```

### 新格式（19列）✅
```
日期	GMV	时长(h)	观看	在线	订单	完成	互动	点赞	评论	分享	关注	商品曝光	商品点击	场次	转化率(%)	点击率(%)	互动率(%)	时均GMV
```

**好处**：
- Coze 可分析互动结构（点赞多但评论少 → 内容吸引力有限）
- 可识别商品推荐效率（曝光vs点击 → CTR异常）
- 可发现订单完成率问题（订单vs完成 → 履约/库存问题）

## 🔧 排查：导入后点赞/评论/分享等仍为 0

若重新导入后仪表盘上点赞、评论、分享、关注、商品曝光/点击、点击率、互动率仍显示 0，多半是 **Excel 表头与系统识别的列名不一致**。

### 1. 查看系统实际读到的表头

在后端项目根目录启动时加上环境变量，再执行一次导入，控制台会打印当前解析到的表头列名：

```bash
# Windows PowerShell
$env:DEBUG_EXCEL_HEADERS="1"; node dist/index.js

# 或 Linux/macOS
DEBUG_EXCEL_HEADERS=1 node dist/index.js
```

导入后控制台会输出类似：`[excelParser] 检测到的表头列名: 日期 | 直播标题 | 点赞 | 评论 | ...`

- 若这里**没有**「点赞」「评论」等列名，说明表头在第 1 行而数据在第 2 行，或表头在第 3 行（系统会自动尝试第 3 行作为表头）。
- 若列名与上表「Excel 列名」不一致（例如是「点赞量」「Comment count」），请把 Excel 表头改成上述支持的任一列名，或联系开发在 `backend/src/utils/excelParser.ts` 中增加该列名映射。

### 2. 表头在第几行

- 若 Excel **第 1 行**是「数据周期：xxxx」之类、**第 2 行**是空或标题、**第 3 行**才是「日期、GMV、点赞…」，系统会自动用第 3 行作为表头，一般无需改。
- 若您的表头在第 1 行，且第 1 行列名已在「支持的列名」列表中，通常能正确解析。

### 3. 确认后端与数据库已更新

- 确保已执行过 stats 表扩展（`likes`、`comments`、`shares` 等列存在），并**重启过后端**，再重新导入。
