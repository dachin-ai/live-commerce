# 模块 1：登录与认证 — 调试与测试

## 对标样本
- **淘宝/抖音开放平台**：OAuth2、Token 持久化、未登录重定向、登录态失效跳转登录
- **行业共性**：用户注册、找回密码、登录后个人中心（资料与修改密码）

## 一、调试清单（先做）

### 1. 确保后端已启动
```bash
cd backend
npm run dev
```
- 看到 `🚀 服务器运行在 http://localhost:3000` 即表示成功
- 浏览器访问 http://localhost:3000/health 应返回 `{"status":"ok",...}`

### 2. 确保前端已启动
```bash
cd frontend
npm run dev
```
- 访问 http://localhost:5173 应跳转到 http://localhost:5173/login

### 3. 已做的代码修复与功能（本模块）
- **auth.ts**：登录响应增加防御性检查；提供 `useRegister`、`requestPasswordReset`、`resetPasswordWithToken`、`updateProfile`、`changePassword`
- **Login.tsx**：登录成功后跳转 `/`；错误信息优先展示后端返回；增加「忘记密码？」「立即注册」入口；注册成功后跳转登录并 Toast 提示
- **Register.tsx**：注册页（姓名、邮箱、密码、确认密码），校验一致性与长度，成功后跳转登录
- **ForgotPassword.tsx**：输入邮箱请求重置；开发环境下后端返回 `resetToken`，前端提供「前往重置密码」跳转
- **ResetPassword.tsx**：凭 URL 中 `?token=xxx` 设置新密码；成功后跳转登录
- **Profile.tsx**：个人中心（与 Sidebar 同布局），编辑姓名、修改密码（需当前密码）；侧栏用户菜单「个人中心」跳转 `/profile`
- **后端**：`POST /auth/forgot-password`、`POST /auth/reset-password`、`PUT /auth/me`、`POST /auth/change-password`；表 `password_reset_tokens`

### 4. 若登录仍失败，请检查
- 控制台 Network：`POST /api/auth/login` 是否发出、状态码与响应体
- 后端终端：是否有报错（如数据库、bcrypt、jwt）
- 数据库是否有初始用户：无则删除 `backend/data.db` 后重启后端，会自动创建 admin/operator/anchor
