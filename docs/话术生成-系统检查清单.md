# 话术生成 · 系统检查清单

用于确认话术生成与 Coze 对接的当前状态是否符合约定，便于发布前或问题排查时快速核对。

---

## 1. 请求体与 Coze 兼容

| 检查项 | 约定 | 当前实现 | 位置 |
|--------|------|----------|------|
| *.coze.site 使用旧版 body | URL 含 `coze.site` 时使用 `content.query` + `project_id`，避免空响应 | ✅ `useLegacy = COZE_LEGACY_BODY \|\| requestUrl.includes('coze.site')` | scriptLLM.ts `buildCozeStreamRunBody` |
| 非 coze.site 使用 Agent 格式 | 仅 `content` + `session_id` | ✅ 同上，else 分支 | 同上 |
| 环境变量兜底 | `COZE_LEGACY_BODY=1` 时强制旧格式 | ✅ 支持 | 同上 |

## 3. 后端对输出：仅翻译，无其他处理

## 4. 流式与重试

| 检查项 | 约定 | 当前实现 | 位置 |
|--------|------|----------|------|
| 首次无输出重试 | 立即再调一次 streamOnce | ✅ | script.ts |
| Coze 二次仍空则延迟重试 | 2.5s 后第三次 streamOnce | ✅ 仅 `isCozeAgent` 时 | script.ts |
| SSE 解析 | 支持 `data: {...}`、裸 `{...}` 行、`data:` 与 JSON 分两行 | ✅ `split(/\r?\n/)`，trim 后识别 data:/{ | scriptLLM.ts |
| 0 yield 诊断 | 记录 contentType、rawPreview、bufferTail；若为 JSON 则记 code/msg | ✅ empty_stream_no_payloads / empty_stream_json_response | scriptLLM.ts |
