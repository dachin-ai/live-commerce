# 配置与特性开关说明

本文集中说明系统中与**运营/业务**相关的配置项与特性开关，便于部署、运维与业务理解。技术细节与步骤见各专项文档。

---

## 一、LLM 配置

| 项 | 说明 |
|----|------|
| **作用** | 统一配置供**智能待办、话术生成、异常分析**等所有 LLM 能力使用；支持 Coze、豆包、OpenAI 等，后续也可通过 API 部署其它模型。配置后选定用户可使用 LLM 功能，并在「LLM 调用方式」页按场景选择调用方式。 |
| **谁配置** | 管理员在「管理员」-「LLM 配置」中填写 API 地址与密钥并勾选可使用用户；或部署人员通过环境变量配置。 |
| **非管理员权限** | 同一处「可使用 LLM 的用户」勾选列表同时控制**话术生成**与**智能生成待办**：全选 = 全体可用；不勾选任何人 = 仅管理员可用；勾选部分用户 = 仅这些用户（及管理员）可用。非管理员若未在列表中，前端会提示无权限并引导联系管理员开通。 |
| **详细步骤** | [话术生成 LLM 配置说明](./话术生成LLM配置说明.md)、[待办与执行工具 - LLM 接入与配置说明](./待办与执行工具-LLM接入与配置说明.md) |

## 二、环境变量与后端开关

### 2.1 话术/待办 LLM 相关（后端）

| 变量/开关 | 默认值 | 说明 |
|-----------|--------|------|
| **SKIP_TODO_LLM** | 未设置 | 设为 `1` 时，**待办智能生成**跳过 LLM，仅用规则引擎，响应约 2 秒内。用于快速调试或 MCP 多轮测试。正式验证 LLM 路径时需去掉后重启后端。 |
| **LLM_ONCE_TIMEOUT_MS** | 25000（25 秒） | 一次性 LLM 调用（含待办智能生成）超时时间；超时后自动回退到规则引擎。 |
| **SCRIPT_LLM_URL** / **SCRIPT_LLM_API_KEY** | — | LLM 的 API 地址与密钥（智能待办/话术/异常分析共用）；未设置时也可由管理员在「LLM 配置」中配置并写入数据库。 |

**SKIP_TODO_LLM 示例**（后端启动前执行）：

- Windows PowerShell：`$env:SKIP_TODO_LLM="1"; cd backend; npm run dev`
- Linux/macOS：`SKIP_TODO_LLM=1 npm run dev`（在 backend 目录下）

详见 [各模块步骤总结 - 模块 5：LLM 调用慢与待办调试](./各模块步骤总结.md#模块-5llm-调用慢与待办调试按-module-test-workflow-时适用)、[Coze大模型优化说明](./Coze大模型优化说明.md)。

### 2.2 PostgreSQL 数据库连接

| 变量 | 默认值 | 说明 |
|------|--------|------|
| **PG_HOST** | `localhost` | PostgreSQL 主机地址 |
| **PG_PORT** | `5432` | PostgreSQL 端口 |
| **PG_DATABASE** | `live_commerce` | 数据库名 |
| **PG_USER** | `lvbcsym` | 数据库用户名 |
| **PG_PASSWORD** | `lvbcsym2026` | 数据库密码 |

> 自 2026-04-30 起，系统从 SQLite 迁移至 PostgreSQL 18。以上变量配置在 `backend/.env` 中，切换开发/生产环境只需修改这些值。

### 2.3 邮箱认证（SMTP）

| 变量 | 必填 | 说明 |
|------|------|------|
| **SMTP_HOST** | 是* | SMTP 服务器地址（如 `smtp.qq.com`、`smtp.163.com`、`smtp.gmail.com`）。 |
| **SMTP_PORT** | 否 | 端口，默认 `587`；SSL 一般为 `465`，此时建议设 **SMTP_SECURE**=`true`。 |
| **SMTP_SECURE** | 否 | 设为 `true` 或 `1` 时使用 TLS（通常端口 465）。 |
| **SMTP_USER** | 是* | SMTP 登录用户名（一般为发件邮箱）。 |
| **SMTP_PASS** | 是* | SMTP 密码或授权码（QQ/163 等需使用授权码）。 |
| **MAIL_FROM** | 否 | 发件人地址，不填时使用 SMTP_USER。 |
| **APP_NAME** | 否 | 发件人名称前缀，默认「直播电商中台」。 |

\* 必填指：要**真正发送邮件**时，SMTP_HOST、SMTP_USER、SMTP_PASS、MAIL_FROM 均需配置。未配置时，忘记密码仍可走验证码流程，但**不会发邮件**；开发环境下接口会在响应中返回验证码便于测试。

**用途**：忘记密码时向用户邮箱发送 6 位验证码；验证码 10 分钟内有效。

**如何配置（三步）：**

1. **在 backend 目录下编辑 `.env` 文件**（没有则复制 `.env.example` 为 `.env`）。
2. **填写以下变量**（以 QQ 邮箱、163 邮箱为例）：

   **QQ 邮箱：**
   - 先在 QQ 邮箱网页版 → 设置 → 账户 → 开启「POP3/SMTP」或「IMAP/SMTP」，并获取**授权码**（不是 QQ 登录密码）。
   - `.env` 示例：
   ```env
   SMTP_HOST=smtp.qq.com
   SMTP_PORT=465
   SMTP_SECURE=true
   SMTP_USER=你的QQ邮箱@qq.com
   SMTP_PASS=刚才获取的授权码
   MAIL_FROM=你的QQ邮箱@qq.com
   APP_NAME=直播电商中台
   ```

   **163 邮箱：**
   - 在 163 邮箱 → 设置 → POP3/SMTP/IMAP 中开启 SMTP，并设置**授权码**。
   - `.env` 示例：
   ```env
   SMTP_HOST=smtp.163.com
   SMTP_PORT=465
   SMTP_SECURE=true
   SMTP_USER=你的163邮箱@163.com
   SMTP_PASS=授权码
   MAIL_FROM=你的163邮箱@163.com
   APP_NAME=直播电商中台
   ```

3. **保存后重启后端**（如 `npm run dev`），再在前端使用「忘记密码」输入邮箱，即可收到验证码邮件。

若未配置上述变量，系统不会发邮件；开发环境下接口会在响应中返回验证码，便于本地测试。

- **限流**：当前 15 分钟内最多 500 次请求/IP，见 [OPTIMIZATION.md](./OPTIMIZATION.md)。
- **项目结构**：AI 路由入口为 `ai-refactored.ts`，见项目结构说明。

### 2.4 CORS 与鉴权 Cookie（2026-05-01 新增）

| 变量 | 默认值 | 说明 |
|------|--------|------|
| **CORS_ORIGIN** | `http://localhost:5173` | 允许携带 Cookie 的前端域名。生产环境设为实际域名如 `https://yourdomain.com`。`credentials: true` 模式下不允许 `*` 通配。 |

**鉴权机制**：JWT 以 `httpOnly` + `secure`（生产） + `sameSite=strict` Cookie 存储，前端无法通过 `document.cookie` 或 `localStorage` 读取 Token，防止 XSS 窃取凭证。后端 `middleware/auth.ts` 优先从 Cookie 读 Token，回退 Bearer Header 以兼容外部 API 调用。

## 三、前端与业务可见行为

| 配置/开关 | 用户可见影响 |
|-----------|--------------|
| LLM 未配置 | 话术生成页提示「无法生成话术」并引导管理员配置；待办智能生成仍可用（规则引擎）。 |
| LLM 已配置 | 话术生成可用；智能生成任务可能显示「含 X 条 AI 智能建议」，终端用户可在「LLM 调用方式」中选择调用方式。 |
| SKIP_TODO_LLM=1（后端） | 智能生成仅规则路径，约 2 秒内返回，Toast 无「含 X 条 AI 智能建议」。 |

**登录与账号**  
- **邮箱认证**：已支持。配置 SMTP 后忘记密码会发验证码；未配置时开发环境接口返回验证码。  
- **自动登出周期**：会话有效期为 **7 天**（JWT 与服务端 session 均为 7 天），超时后需重新登录。  
- **首次登录欢迎语**：账号**首次登录**时展示一次欢迎语。  
- **新 IP 教程**：从**新 IP 首次登录**时展示一次快速上手指南（教程）；同一 IP 再次登录不再展示。
