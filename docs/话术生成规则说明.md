## 话术逻辑与类型结构（接口产出）

接口按 `scriptType` 与用户输入（产品名称、价格、产品特点、目标人群、营销方案/促销活动）产出**带完整逻辑结构**的话术内容，不再返回通用「开场白+主要内容 1.2.3+结束语」模板。

| 话术类型 | 产出结构 |
|----------|----------|
| **完整销售流程话术（推荐）** | 六步：圈人群+塑品 → 卖点提炼 → 打消顾虑 → 利益点 → 售后保障 → 逼单；每步含建议时长与承上启下；利益点/逼单带入用户填写的营销方案与价格。 |
| **促销活动话术** | 营销方案 + 价格 + 限时感 + 行动召唤。 |
| **逼单技巧话术** | 营销方案 + 库存/限时 + 数三个数 + 小黄车。 |
| **场景化塑品话术** | 目标人群 + 场景痛点 + 产品与卖点 + 价格 + 行动召唤。 |
| **人群互动话术** | 产品与卖点 + 价格 + 扣1/公屏 + 小黄车。 |

若前端传入 `storeId`，正文头部会带「【基于店铺】店铺名（平台）」与数据参考说明。

**LLM 配置**：话术生成仅支持 LLM，未配置时前端会提示「无法生成话术」并给出配置步骤。由部署/管理员在后端设置 `SCRIPT_LLM_URL`、`SCRIPT_LLM_API_KEY`（或 `OPENAI_API_BASE`、`OPENAI_API_KEY`）并重启服务后即可使用。详见 `docs/话术生成LLM配置说明.md`。

## 品类循环框架与润色（接 LLM 且带店铺品类时生效）

当**已配置 LLM** 且请求带有**店铺品类**（如通过 `storeId` 拉取到 categories）时，市调摘要会注入「品类循环框架与润色要求」，提示词逻辑为：

1. **第一步（框架）**：请设计一套适用于【店铺所在品类】的直播话术框架，包含 1、2、3、4、5、6 等策略模块，要求适配 **5～15 分钟循环讲解**；我的产品及参数为：【用户输入参数（产品名、价格、卖点、目标人群等）】。
2. **第二步（润色）**：请把以上框架**润色为一个主播在【店铺所在国家】直播间能直接念的话术稿**，语气自然、可照念或稍作调整，符合平台合规与品类最佳实践。

「店铺所在国家」由店铺 `region` 或话术 `language` 推断（如泰国、越南、中国、美国等），规则见 `scriptResearch.ts` 中 `getStoreCountry`。  
未接 LLM 或未带店铺品类时，不注入上述要求，仍按原有话术类型与模板产出。

## 可视化动作：根据实际话术抓取卖点，配置驱动

### 原则

- **不是固定列举**：防水、不锈钢、静音、耐磨等仅为**举例**，不作为唯一或权威的卖点集合。
- **根据实际话术抓取卖点**：本环节建议的「演示型动作」来源于**该段话术正文**中出现的表述，再结合用户填写的产品特点（features）。即：用「这一段实际说了什么」来抓取对应卖点，再设计对应可视化动作。
- **配置驱动动作设计**：卖点关键词与「演示动作」的对应关系由配置 `featureDemoHints` 决定；实际使用时应按业务话术在 `visualSegmentRules.json` 的 `featureDemoHints` 中配置「话术里会出现的关键词 → 本环节可做的演示动作」。

### 匹配逻辑

1. 对每个话术段落（如「第二步：卖点提炼」），取该段**正文**为 `segmentText`。
2. 匹配来源 = `segmentText` + 用户填写的 `features`（以段落正文为主）。
3. 用该文本与 `featureDemoHints` 中每条 `keywords` 做匹配（大小写不敏感）；命中则用该条的 `suggestion` 模板（支持 `{productName}`、`{features}`）生成一条演示建议。
4. 若该环节规则开启了 `allowFeatureDemos`，则先输出上述演示建议，再补「也可手卡/特写」等通用建议。

### 配置说明

| 项目 | 说明 |
|------|------|
| `featureDemoHints` | 卖点关键词→演示动作。配置后**完全替换**代码内置的举例列表；实际使用请按业务在此配置对应关键词与动作。 |
| 内置举例 | 代码中防水/不锈钢/静音/耐磨等仅为举例，当配置未提供或为空时使用；不作为业务默认依据。 |
| 规则位置 | `config/script-rules/visualSegmentRules.json`；缺省使用 `scriptVisualRules.ts` 内置规则与举例。 |

### 小结

- 卖点来源：**实际话术段落正文**（主）+ 用户填写的产品特点（辅）。
- 动作设计：**完全由配置** `featureDemoHints` 驱动；内置仅为举例，应根据实际话术在配置中维护「关键词→演示动作」。
