# 场景化塑品话术 - 提示词与规则

对应类型 id：`scenario`。用于建立产品使用场景、介绍产品并展示价值。

---

## 类型与目标

| 项目 | 内容 |
|------|------|
| 目标 | 建立产品使用场景 |
| 适用场景 | 产品介绍、展示价值 |
| 建议时长 | 30-60 秒 |

---

## 必须包含的要素

1. **场景代入**：先给出具体使用场景或人群状态（如「很多铲屎官最怕笼子难洗、猫一啃就坏」）
2. **痛点描述**：简短点出该场景下的痛点或需求
3. **产品即解决方案**：自然带出本品如何解决（避免「安全、好洗、耐用」式罗列）
4. **效果展示**：可配合画面感或用户反馈（如「猫在里面磨爪、睡觉，不用担心啃笼子」）
5. **行动召唤**：一次清晰的下单/互动引导

---

## 该类型专属规则

- **先场景再卖点**：禁止干巴巴罗列参数或顿号一口气念完多个卖点；必须先有场景/痛点，再带出产品如何解决。
- **代入感**：让观众能「想象自己在用」或「联想到自己的经历」，可用小故事、用户反馈、画面感描述。
- **主次分明**：核心卖点 1～2 个重点讲，次要卖点一带而过，不要平铺。
- **可标注演示**：关键卖点处可带出「大家看这里」「可以看公屏/手卡」等，便于配合画面与手卡。

## 系统提示词（实际发送给 LLM）

```
你是直播话术策划。只输出「可念的话术正文」，禁止输出：市场分析、策略建议、优先级建议、行动清单、报告、中英对照列表等非话术内容。
当前生成「场景化塑品话术」。目标：建立产品使用场景。场景：产品介绍、展示价值。须包含要素：场景代入、痛点描述、产品即解决方案、效果展示、行动召唤。
通用规则:1)依摘要输入/店铺/合规/品类 2)禁词禁用慎词有据 3)语种与摘要一致 4)数字泛化 5)自然口语 6)紧扣产品与品类卖点 7)场景/痛点/故事带出卖点
```

（若有品类循环框架，会追加框架说明。）

## 用户消息模板

**任务行：**

```
【任务】仅生成「场景化塑品话术」，产品：{产品名称}。目标：建立产品使用场景。请按要素（场景代入、痛点描述、产品即解决方案、效果展示、行动召唤）组织内容，只输出主播可念的话术正文，不要输出分析、建议、报告、清单等非话术内容。
```

**随后：** 市调摘要。**结尾：** `请依上仅输出话术正文。` 或 `请先出框架再出可念稿。`

---

## 通用规则与禁止输出

- **通用**：同 README 与 scriptResearch 中的通用规则（合规、语种、数字泛化、自然口语、紧扣产品与品类、场景化带出卖点）。
- **禁止输出**：市场分析、策略建议、报告、行动清单、中英对照等非话术内容。
