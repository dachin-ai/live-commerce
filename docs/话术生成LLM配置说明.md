# 话术生成 LLM 配置说明

话术生成功能依赖 LLM（大模型）或 **Coze 智能体**。配置方式二选一：**管理员在系统后台配置**（推荐，配置后全体用户可用）或 **部署人员在环境变量中配置**。

**待办与执行工具中哪些需要 LLM、如何完成配置**：见 [待办与执行工具 - LLM 接入与配置说明](./待办与执行工具-LLM接入与配置说明.md)。

**默认智能体**：系统默认使用 Coze 发布站点 `https://zbmr4xq6rm.coze.site/stream_run`，管理员只需在后台填入该站点对应的 **API Token** 即可使用。

---

## 谁可以配置

- **管理员**：登录后进入「管理员」页面，在「话术 LLM 配置」中填写 API 地址（默认已填 Coze 智能体地址）与 API 密钥（Token）并保存，**配置立即生效，全体用户可使用话术生成**。
- **部署/运维**：若无管理员账号或希望用环境变量统一管理，可在后端所在机器配置环境变量并重启后端（见下文）。
- **普通用户**：无法配置，需联系管理员或部署人员。

---

## 方式一：管理员在后台配置（推荐）

1. 使用**管理员账号**登录系统。
2. 进入 **管理员** 页面（侧栏或顶部入口）。
3. 在「话术 LLM 配置」卡片中填写：
   - **API 地址**：默认已填 **Coze 智能体** 地址 `https://zbmr4xq6rm.coze.site/stream_run`，也可改为豆包、OpenAI 等；
   - **API 密钥**：Coze 发布站点使用 API Token（Bearer）；豆包/OpenAI 使用控制台 API Key；
   - **模型 / 接入点 ID**（可选）：**Coze 不需要**；豆包必填 `ep-` 接入点 ID；OpenAI 可留空。
4. 点击「保存配置（全体用户可用）」。
5. 保存后立即生效，无需重启后端；所有用户（含普通用户）均可使用「创建脚本」的话术生成功能。
6. **长期生效**：配置写入数据库（`system_config` 表），会持久保存；后端重启后会自动从数据库加载，无需重新配置。

---

## 方式二：环境变量配置（部署人员）

### 豆包（火山方舟，推荐国内使用）

- **API 地址**：`https://ark.cn-beijing.volces.com/api/v3`
- **API 密钥**：在 [火山引擎控制台](https://console.volcengine.com/ark) 获取。
- **模型**：必须填写**推理接入点 ID**（在方舟控制台「在线推理」中创建接入点后，复制 `ep-` 开头的 ID），不能填模型名称。

### 其他厂商

- **OpenAI**：API 地址一般为 `https://api.openai.com/v1`，密钥在 OpenAI 控制台获取；模型可留空或填 `gpt-4o-mini`。
- **国内兼容 OpenAI 的厂商**：使用厂商提供的 API 地址和 API Key（如阿里云、腾讯云、智谱、DeepSeek 等）。

### 2. 在后端项目里设置环境变量

在**后端项目根目录**（即 `backend` 所在目录或 `backend` 目录，取决于你启动后端时的当前目录）创建或编辑 `.env` 文件。

**方式 A：使用话术专用变量（推荐）**

```env
SCRIPT_LLM_URL=https://api.openai.com/v1
SCRIPT_LLM_API_KEY=sk-你的API密钥
```

可选：指定模型，不填则默认 `gpt-4o-mini`：

```env
SCRIPT_LLM_MODEL=gpt-4o-mini
```

**方式 B：使用通用 OpenAI 变量**

若项目里已有 OpenAI 相关配置，可直接复用：

```env
OPENAI_API_BASE=https://api.openai.com/v1
OPENAI_API_KEY=sk-你的API密钥
```

后端会优先读取 `SCRIPT_LLM_*`，没有时再使用 `OPENAI_*`。

### 3. 重启后端服务

修改 `.env` 后必须**重启后端**，环境变量才会生效。

- 本地开发：重新执行 `npm run dev` 或你使用的启动命令。
- 服务器/容器：重启对应进程或容器。

---

## 配置是否生效

- **管理员后台配置**：保存后立即生效，无需重启。
- **环境变量配置**：修改后需重启后端。

配置正确后，在前端「创建脚本」选择店铺、填写产品信息并点击「生成」，应能正常得到话术内容，而不再出现「无法生成话术」的黄色提示框。后端优先使用环境变量；未设置环境变量时使用管理员在后台保存的配置。

---

## 常见问题

| 问题 | 处理 |
|------|------|
| 仍提示「无法生成话术」 | 确认 `.env` 在**后端进程的工作目录**下、变量名无拼写错误、已重启后端。 |
| 国内访问 OpenAI 超时 | 使用国内兼容 OpenAI 的 API 地址与密钥，并设置 `SCRIPT_LLM_URL` 为该地址。 |
| 不想把密钥写在 .env | 可在系统环境变量中设置同名变量（如 `SCRIPT_LLM_API_KEY`），后端会读取。 |

---

## 代码中的读取顺序

后端 `backend/src/services/scriptLLM.ts` 与 `scriptLLMConfig.ts` 的读取顺序：

1. **环境变量**：若已设置 `SCRIPT_LLM_URL` + `SCRIPT_LLM_API_KEY`（或 `OPENAI_API_BASE` + `OPENAI_API_KEY`），优先使用。
2. **数据库配置**：若未设置环境变量，则使用管理员在后台保存的配置（表 `system_config`），启动时加载到内存，供全体用户使用。

模型名仍来自环境变量 `SCRIPT_LLM_MODEL` 或 `OPENAI_MODEL`，默认 `gpt-4o-mini`。
