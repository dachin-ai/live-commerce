# 一模块一调试、调试好再测试循环

## 原则

1. **一个模块一个模块来**：只做当前模块的调试与测试，不做其他模块。
2. **先调试后测试**：先把该模块功能调通（修 bug、确认接口与数据），再跑该模块的测试循环。
3. **通过再下一模块**：当前模块测试循环全部通过后，再进入下一个模块。

---

## 流程

```
选定模块 → 对标成熟样本 → 调试（修代码/环境）→ 本地验证通过
    → 跑该模块测试循环（按《完整功能测试指南》+《模块对标与全量测试计划》）
    → 记录结果 → 下一模块
```

**多轮循环**：若某轮测试有失败，则「调试 → 自验 → 再测」重复多轮，直到该模块全部通过。详见 **《测试调试多轮循环优化》**（`docs/测试调试多轮循环优化.md`），每轮可复用 **《每轮检查清单-模板》**（`docs/每轮检查清单-模板.md`）。

## 每模块文档约定

每个模块建议有单独说明（如 `模块N-xxx-调试与测试.md`），包含：

1. **对标样本**：对标的成熟产品/标准
2. **调试清单**：环境、依赖、已做代码修改
3. **测试循环**：具体步骤与预期（可引用《完整功能测试指南》）
4. **通过标准**：何时视为该模块通过、可进入下一模块
