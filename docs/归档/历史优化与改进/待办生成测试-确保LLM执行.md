# 待办生成测试 — 确保 LLM 内容实现

## 前置条件（务必满足）

1. **后端启动时不要设 SKIP_TODO_LLM**  
   - 若之前为快速调试设过 `SKIP_TODO_LLM=1`，请去掉后重启后端，否则不会调 LLM。

2. **话术 LLM 已配置**  
   - 管理员登录 → 管理员功能 → 话术 LLM 配置：填写 API 地址与密钥并保存。  
   - 未配置时只会走规则引擎，不会有「含 X 条 AI 智能建议」。

3. **前后端已启动**  
   - 后端：`cd backend && npm run dev`（端口 3000）  
   - 前端：`cd frontend && npm run dev`（端口 5173）

---

## Browser 测试步骤（在 Cursor Browser Tab 中执行）

1. **打开前端**  
   访问：`http://localhost:5173`

2. **登录**  
   邮箱：`admin@example.com`，密码：`123456`

3. **选店铺**  
   顶部店铺下拉选择有数据的店铺（如 greenpet）。

4. **找到「待处理任务」**  
   在主区找到橙色「待处理任务」卡片，确保已展开。

5. **点击「智能生成」**  
   - 按钮会变为「生成中…(约30秒)」  
   - 等待完成（LLM 约 15–30 秒；若 LLM 超时或未配置则走规则，约 2–5 秒）

6. **验证 LLM 内容已实现**  
   - **Toast**：若出现「成功生成 N 个任务（**含 X 条 AI 智能建议**）」→ 说明本次有 X 条来自 LLM，LLM 内容已实现。  
   - **任务标题**：LLM 生成的任务多为场景化表述（如「今晚7点加播2小时」「每10分钟福袋拉互动」）；若多为「提升转化率至 X%」「增加直播时长至 X 小时/周」则为规则模板。  
   - **条数**：任务条数不限制，可能为 3～10+ 条（规则+LLM 合并去重后写入）。

7. **可选：看控制台**  
   - 若后端日志有 `[LLM 为主] 生成 X 条智能待办 + Y 条规则补充`，说明当次请求走了 LLM 并写入了任务。

---

## 若 Toast 只有「成功生成 N 个任务」、无「含 X 条 AI 智能建议」

说明当次未使用 LLM，请检查：

- 话术 LLM 是否已配置并保存  
- 后端是否在未设置 `SKIP_TODO_LLM=1` 的情况下启动  
- 后端日志是否有 `[LLM 智能待办] 调用失败，走规则兜底`（LLM 超时或报错时会回退到规则）
