## 📋 按 Coze 建议优化的关键点

### 1. ⭐ 强调按日明细数据的重要性

**Coze 建议**：
> 按日明细数据是 Agent 生成高质量待办的**核心依据**！影响待办质量 80%。

**优化前**：
```
【最近30天按日明细】（原始用户上传数据，供智能体直接分析）
```

**优化后**：
```
【⭐ 按日明细数据】（最重要！影响待办质量 80%，请基于此分析趋势、发现规律、识别最佳/最差日）
```

**效果**：Coze bot 会更重视这部分数据并深入分析。

---

### 2. 结构化销售指标

**Coze 建议**：提供结构化的 metrics（total_orders、total_viewers、total_revenue、currency、conversion_rate、stage）

**优化前**：
```
【最近30天数据】GMV 290600.45 | 时长 127h | 观看 17452 | 订单 636 | 转化率 3.6%
```

**优化后**：
```
【核心销售指标（最近30天）】
- 总订单数：636 单
- 总观看数：17452 人
- 总收入（GMV）：290600.45 泰铢
- 转化率：3.64%
- 直播总时长：127.8 小时
- 时均 GMV：2274 泰铢
- 店铺阶段：成熟期
```

**改进点**：
- ✅ 每行一个指标，清晰可读
- ✅ 明确货币单位（泰铢、人民币等）
- ✅ 明确标注「店铺阶段」（Coze 建议的 stage 字段）

---

### 3. 增强历史对比（含变化百分比）

**Coze 建议**：明确说明 GMV/订单数变化百分比（如"较上周下降 15%"）

**优化前**：
```
【历史对比】前期平均 GMV 0 | 平均转化 0%
```

**优化后**：
```
【历史对比】
前期平均 GMV 139419 泰铢（本期下降 15.2%） | 平均转化 4.1%（本期下降 0.5个百分点）
```

**改进点**：
- ✅ 明确货币单位
- ✅ 计算并标注变化百分比
- ✅ GMV 用百分比，转化率用百分点（更准确）

---

### 4. 明确标注平台和国家

**Coze 建议**：platform 和 country 是生成本地化待办的关键

**优化前**：
```
【店铺】测试-宠物（TikTok）| 区域：泰国 | 类目：宠物用品
```

**优化后**：
```
【店铺基本信息】
- 店铺名称：测试-宠物
- 平台：TikTok
- 国家/区域：泰国
- 类目：宠物用品
```

**改进点**：
- ✅ 结构化展示
- ✅ 明确「平台」和「国家/区域」字段（Coze 可识别）

---

### 5. 优化生成要求（4 点明确指引）

**Coze 建议**：明确告诉 bot 如何使用数据生成待办

**优化前**：
```
请根据以上必要参数与店铺直播明细，按「最近发展区」生成待办...
```

**优化后**：
```
【生成要求】
请基于以上完整数据（特别是⭐按日明细），按「最近发展区」生成 3-6 条待办：
1. 结合具体数字（如"转化率 3.6%"、"12月12日 GMV 最高"）
2. 给出可执行动作（如"在高峰时段增加 XX 商品推广"）
3. 考虑业务上下文（阶段、节日、气温、趋势）
4. 优先当下最应投入的任务（非泛泛建议）
```

**改进点**：
- ✅ 明确条数要求（3-6 条）
- ✅ 4 点生成规则（结合数字、可执行、有上下文、优先级）
- ✅ 强调⭐按日明细的使用
