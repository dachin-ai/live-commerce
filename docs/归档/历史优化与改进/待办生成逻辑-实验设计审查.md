## ⚠️ 存在的问题和风险

### 问题1：历史数据不足时的边界情况 ⚠️

#### 问题描述
```typescript
// 如果历史数据不足（<2期），可能导致不一致

  // 使用历史数据计算动态阈值
```

#### 风险分析
- **不可控性**：新店铺和老店铺的判断逻辑不同
- **偶然性**：刚好在第2期数据时，判断逻辑发生切换

#### 建议改进
```typescript

if (historicalStats.length === 2) {
  // 使用历史数据权重较低的混合模型

  threshold = weight * historicalAvg + (1 - weight) * industryBenchmark
} else if (historicalStats.length > 2) {
  // 使用历史数据权重较高的混合模型
  const weight = Math.min(0.7, historicalStats.length / 10)
  threshold = weight * historicalAvg + (1 - weight) * industryBenchmark
}
```

### 问题2：异常检测的阈值硬编码 ⚠️

#### 问题描述
```typescript
// 异常检测阈值是硬编码的

  // GMV下降50%触发异常
}

  // 转化率下降30%触发异常

```

#### 风险分析
- **不可控性**：阈值（0.5, 0.7）是主观设定的，缺乏数据支撑
- **偶然性**：不同店铺的波动特征不同，统一阈值可能不合理

#### 建议改进
```typescript
// 基于历史波动率动态调整阈值
function getAnomalyThreshold(historicalStats: any[]): number {
  // 计算历史数据的标准差
  const values = historicalStats.map(s => s.totalGMV)
  const mean = values.reduce((sum, v) => sum + v, 0) / values.length
  const variance = values.reduce((sum, v) => sum + Math.pow(v - mean, 2), 0) / values.length
  const stdDev = Math.sqrt(variance)
  
  // 基于3σ原则（99.7%置信区间）
  const threshold = mean - 3 * stdDev
  
  return threshold
}

// 使用动态阈值
const anomalyThreshold = getAnomalyThreshold(recentStats)
if (currentStats.totalGMV < anomalyThreshold) {
  // 触发异常
}
```

### 问题4：店铺阶段判断的边界模糊 ⚠️

#### 问题描述
```typescript
// 店铺阶段判断
if (gmv < 10000 || sessions < 10) {
  return { stage: 'cold_start', ... }
} else if (gmv < 100000) {
  return { stage: 'growth', ... }
} else {
  return { stage: 'mature', ... }
}
```

#### 风险分析
- **不可控性**：阈值（10000, 100000）是硬编码的，不同品类可能不适用
- **偶然性**：GMV刚好在10000或100000附近时，任务生成会突变

#### 建议改进
```typescript
// 基于品类的动态阈值
function getStageThresholds(categories: string[], platform: string): {
  coldStartGMV: number,
  growthGMV: number
} {
  // 不同品类的阈值不同
  const categoryThresholds: { [key: string]: { cold: number; growth: number } } = {
    '宠物用品': { cold: 5000, growth: 50000 },   // 低客单价
    '服饰鞋包': { cold: 10000, growth: 100000 },
    '珠宝文玩': { cold: 50000, growth: 500000 }, // 高客单价
  }
  
  // 查找匹配的品类
  for (const category of categories) {
    if (categoryThresholds[category]) {
      return {
        coldStartGMV: categoryThresholds[category].cold,
        growthGMV: categoryThresholds[category].growth
      }
    }
  }
  
  // 默认阈值
  return { coldStartGMV: 10000, growthGMV: 100000 }
}

// 使用动态阈值
const thresholds = getStageThresholds(categories, platform)
if (gmv < thresholds.coldStartGMV || sessions < 10) {
  return { stage: 'cold_start', ... }
} else if (gmv < thresholds.growthGMV) {
  return { stage: 'growth', ... }
}
```

- **作用**: 减少偶然性，同一输入可重复得到相同异常判断

### 3. 生成元数据与日志 ✅
- **接口**: `POST /generate-tasks` 响应增加 `metadata`：
  - `generated`, `skipped`, `totalCandidates`, `anomalyCount`
  - `thresholdSource`: `'historical'` | `'industry_benchmark'`
  - `generationSummary`: 简短可读摘要
- **日志**: `[任务生成] storeId=xxx, candidates=n, created=m, skipped=k anomaly=a session=s anchor=an`、`[去重] 任务已存在，跳过: xxx`
- **作用**: 可追溯、可复现，便于排查与实验对比

### 4. 平台数据源与单场/主播维度（续完成）✅
- **平台**: `getConversionRateBenchmark(..., platform)` 根据店铺平台显示数据源（TikTok → TikTok平台，抖音 → 抖音平台等），调用处传入 `storeInfo?.platform`。
- **单场维度**: `generateSingleSessionTasks(storeStats)` 生成「复盘最近一场直播」等，由 `TASK_GEN_CONFIG.enableSingleSessionTasks` 开关控制。
- **主播维度**: `generateAnchorTasks(storeStats)` 生成「优化主播话术脚本」「主播长时间直播状态管理」「主播互动技巧培训」等，由 `TASK_GEN_CONFIG.enableAnchorTasks` 控制。
- **元数据**: 响应 `metadata` 增加 `sessionCount`、`anchorCount`。
