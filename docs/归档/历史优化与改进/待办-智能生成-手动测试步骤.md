# 待处理任务 · 智能生成 — 手动测试步骤

按以下步骤在浏览器中验证「智能生成」与任务列表、Toast。

## 前置

- 后端：`cd backend; npm run dev`（端口 3000）
- 前端：`cd frontend; npm run dev`（端口 5173）

## 步骤

1. **打开**  
   浏览器访问：http://localhost:5173

2. **登录**  
   - 邮箱：`admin@example.com`  
   - 密码：`123456`  
   - 点击登录。

3. **进入待处理任务**  
   - 登录后进入首页/Dashboard。  
   - 找到「待处理任务」区域（或侧边栏/执行工具中的 AI 助手/待办）。  
   - 若需先选店铺，选择一家有数据的店铺（如 greenpet）。

4. **点击「智能生成」**  
   - 在待处理任务卡片内点击「智能生成」按钮。  
   - 等待结果：规则路径约 2–5 秒；若走 LLM 可能 15–30 秒。

5. **查看结果**  
   - **任务列表**：是否出现新任务卡片；是否有「全部/紧急/普通」等 Tab；任务标题与描述是否正常。  
   - **Toast**：是否出现成功或提示类 Toast（如「成功生成新任务」或错误提示）。

6. **可选**  
   - 打开开发者工具 → Console：确认无报错。  
   - Network：确认 `/api/ai/generate-tasks` 请求存在且状态为 200。

## 通过标准

- 点击「智能生成」后能等到结果（不卡死、不白屏）。  
- 任务列表有更新或明确无任务提示。  
- Toast 有相应提示。  
- 控制台无红色报错、接口无 5xx。
