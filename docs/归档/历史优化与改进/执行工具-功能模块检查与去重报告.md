依据「各功能模块是否有重复、生成结果是否正常」的检查要求，结合 module-test-workflow 的严苛/小白人设，完成以下结论与改动。

## 一、重复模块检查结论与处理

### 1. 已消除的重复

| 原模块 | 处理方式 | 说明 |
|--------|----------|------|
| **创建脚本** + **话术生成** | 保留「话术生成」、移除列表中的「创建脚本」 | 二者共用同一套表单（产品名、话术类型、语言）与同一后端流式接口 `/api/ai/script/stream`，仅入口文案不同，属重复。保留「话术生成」为唯一入口。访问 `/tools/script` 仍会展示同一表单（兼容旧链接）。 |
| **市场分析** + **市场调研** | 保留「市场分析」、移除「市场调研」 | 后端均为占位（`/api/ai/analyze-market`、`/api/ai/market-research` 未实现），概念重叠。合并为单一入口「市场分析」，减少混淆。 |

### 2. 保留且不合并的模块（概念不同）

| 模块 | 说明 |
|------|------|
| **生成报告** vs **数据统计** | 报告为叙事/文档类，数据统计为指标汇总；后端均为占位，但定位不同，保留两项。 |
| **店铺对比**（含时效对比） | 店铺时效对比已内置到店铺对比界面：同一入口一次生成同时请求综合对比与时效对比，结果区展示「综合对比」+「时效对比」两段。 |

### 3. 图标区分（减少视觉重复）

- **数据统计**：图标由与「生成报告」相同的 `BarChart3` 改为 **`Activity`**，与「生成报告」的柱状图区分。
- **生成报告**：继续使用 `BarChart3`。

## 三、生成结果与后端状态

| 模块 | 后端接口 | 状态 | 生成结果是否正常 |
|------|----------|------|------------------|
| 话术生成 | `POST /api/ai/script/stream` | 已实现（LLM 流式） | 正常：有配置时返回话术正文；未配置时返回明确错误与引导文案。 |
| 待办生成（Dashboard 智能生成任务） | `POST /api/ai/generate-tasks` | 已实现（规则 + 可选 LLM 异常分析） | 正常：生成任务并写入待办；异常时走 LLM 会追加 1～3 条建议。 |

| 市场分析 | `POST /api/ai/analyze-market` | 未实现（前端占位） | 占位：返回「市场分析功能待接入」，无报错。 |
| 商品推荐 | `POST /api/ai/recommendations` | 未实现（前端占位） | 占位：返回空列表等，无报错。 |
| 数据统计 | `POST /api/ai/stats` | 未实现（前端占位） | 占位：返回空统计对象等，无报错。 |
| 店铺对比（含时效） | `POST /api/ai/compare-stores` + `POST /api/ai/compare-efficiency` | 均未实现（前端占位） | 一次生成同时请求两项，结果区展示「综合对比」「时效对比」两段；占位时无报错。 |
