# LLM 修复记录 - 2026-02-10

---

## 🐛 发现的问题

### 问题 1：Coze bot 超时（40秒无返回）

**原因**：
- 输入 prompt 过长（2100+ 字符）
- Coze bot 自动调用外部工具（search_festival_events、search_competitor_insights）
- 工具调用耗时长，累计超过 40 秒
- Bot 返回 markdown 分析而非 JSON

**证据**：
```
coze-debug.log: yieldedChunks: 0, yieldedLen: 0, aborted: true
coze-stream-debug.log: 
  - tool_request: search_festival_events
  - tool_response: 【泰国 2025节庆活动】...
  - answer: "# 泰国美妆..." (markdown，非 JSON)
```

---

### 问题 2：数据库中过时的配置值

**问题**：
```sql
SELECT value FROM system_config WHERE key = 'llm_mode_todo'
-- 返回: 'coze_original' (已停用的旧值)
```

**影响**：虽然代码有兜底（默认 coze_agent），但不规范。

---

### 问题 3：LLM 返回的 tasks 未标记 source

**原因**：
- `generateIntelligentTodosWithLLM` 返回 `IntelligentTodoItem[]`（无 source 字段）
- `generateSuggestedTodosForStore` 取到 tasks 后未标记为 `source: 'llm_intelligent'`
- 导致统计时认为 LLM 0 条

**影响**：
- 前端显示「LLM 0 条」（实际 LLM 有返回）
- 日志与实际不符
- 诊断误导

---

## ✅ 实施的修复

### 修复 1：增强 systemPrompt（禁止工具调用）

```diff
- const systemPrompt = `你是专业的直播电商待办助手...`
+ const systemPrompt = `【严格要求】禁止调用任何工具（search_festival_events、search_competitor_insights 等），禁止输出分析、markdown、标题（#、##）。立即直接输出 JSON 对象，以 { 开头。

+ 你是专业的直播电商待办助手...`
```

**文件**：`backend/src/routes/ai-refactored.ts` L1487  
**效果**：Coze bot 不再调用工具，直接输出 JSON

---

### 修复 2：缩短按日明细（降低 token）

```diff
  SELECT ... FROM stats 
  WHERE storeId = ? AND date >= ? AND date <= ? 
- ORDER BY date ASC
+ ORDER BY date DESC LIMIT 15
```

改为子查询保证顺序：
```sql
SELECT ... FROM (
  SELECT * FROM stats WHERE storeId = ? AND date >= ? AND date <= ? 
  ORDER BY date DESC LIMIT 15
) ORDER BY date ASC
```

**文件**：`backend/src/routes/ai-refactored.ts` L116  
**效果**：只传最近 15 天数据，减少 prompt 长度约 30-40%

---

### 修复 3：修复数据库配置值

```sql
UPDATE system_config

WHERE key = 'llm_mode_todo'
```

**状态**：✅ 已执行
