# 今日对话回顾与规则恢复说明

## 一、今日对话从报错开始的改动

1. **报错：Cannot find module './routes/ai'**  
   - 原因：入口 `index.ts` 引用 `./routes/ai`，但项目入口为 `ai-refactored.ts`（归档前备份 `ai.ts.backup` 已删除）。  
   - 处理：将入口改为 `import aiRoutes from './routes/ai-refactored'`。

2. **报错：Failed to resolve import "../services/ai"**  
   - 原因：`TaskList.tsx`、`AIFeatures.tsx` 从 `../services/ai` 引入 `generateTasks` 等，但 `frontend/src/services/ai.ts` 不存在。  
   - 处理：新建 `frontend/src/services/ai.ts`，实现 `generateTasks`、`generateScript` 及其余 AI 接口的封装。

3. **需求：促销活动的输入框消失了**  
   - 处理：在「创建脚本」中，当话术类型为**逼单 / 促销 / 完整销售流程**时都显示「营销方案 / 促销活动（可选）」输入框，并在提交时传入 `promoCopy`。

4. **需求：话术逻辑全部消失了**  
   - 原因：`POST /api/ai/script` 当时只返回通用模板（开场白 + 主要内容 1.2.3 + 结束语），未按话术类型和用户输入生成。  
   - 处理：在 `ai-refactored.ts` 中新增 `buildScriptContentByType()`，按 `scriptType`（完整销售流程 / 促销 / 逼单 / 场景化 / 人群互动）和用户填写内容生成带六步等结构的话术正文。

5. **更早的需求：按实际话术里会出现的卖点词设计动作**  
   - 处理：在 `scriptVisualRules.ts` 中改为以**该段话术正文（segmentText）**为主、用户填写的产品特点为辅做卖点匹配，再按配置的 `featureDemoHints` 生成演示型建议；内置关键词仅作举例，实际以配置为准。

## 二、被错误删除或清空的规则（已恢复）

在修复上述问题或修改「按实际话术设计动作」时，以下规则**源文件曾被打空**。**已从 `backend/dist/rules/*.js` 反推并恢复**为 TypeScript 源码：

| 文件 | 作用 | 当前状态 |
|------|------|----------|
| `backend/src/rules/platformCompliance.ts` | 各平台违规词、慎用词、合规检查 | **已恢复** |
| `backend/src/rules/scriptResearch.ts` | 话术市调：用户输入 + 店铺数据 + 平台合规 + 品类实践 → 结构化市调结果 | **已恢复** |
| `backend/src/rules/scriptSynthesis.ts` | 话术综合产出：市调 → 模板 → 平台合规 → 最终话术 | **已恢复** |
| `backend/src/rules/fullSalesScript.ts` | 完整销售流程六步话术（圈人群、卖点、打消顾虑、利益点、售后、逼单） | **已恢复** |
| `backend/src/rules/scriptPrompts.ts` | 按话术类型 + 语言调用 fullSalesScript 或各类型模板（中/英/泰） | **已恢复** |

说明：

- **入口**：`POST /api/ai/script` 已**接回**「市调 → 综合产出」管线：构建 `storeContext`（店铺、品类、近期统计）与 `userInput` → `runScriptResearch` → `synthesizeScript` → `injectVisualSuggestionsIntoContent` → `segmentForVisual`、`getVisualLegend`，返回 `content`、`visualParts`、`visualLegend`。
- **scriptVisualRules**：「按实际话术段落抓取卖点 → 配置驱动演示动作」已保留；在得到 `synthesis.content` 后注入本环节建议并产出分段与图例。
