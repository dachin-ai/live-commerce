# 用户体验与 UI 优化建议（终端用户 & 对标产品视角）

**角色**：资深终端用户 + UI 优化师  
**依据**：`docs/各模块步骤总结.md` 对标样本、现有实现与行业共性  
**目标**：在功能已闭环基础上，提升「易用、可信、高效、一致」的终端体验，并对齐淘宝/京东/抖音罗盘/钉钉/飞书/有赞等成品方案。

---

## 一、信息架构与导航

### 1.1 对标：淘宝/京东商家后台、抖音罗盘

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **当前页面包屑** | 除侧栏高亮外，主内容区顶部增加「店铺管理 > 数据看板」式面包屑，便于从外链/分享进入时快速定位。 | 中 |
| **全局「帮助/新手引导」** | 对标罗盘/千牛有「新手指南」「指标说明」。建议：Dashboard 首次进入或空数据时，弱提示「如何导入数据？」并链到《TikTok数据导入使用指南》或简短引导。 | 中 |
| **侧栏折叠态图标+ Tooltip** | 收起侧栏时仅显示图标，建议所有菜单项在 hover 时显示 Tooltip（如「店铺管理」「数据分析」），避免用户记图标。 | 高 |
| **消息入口** | 当前「消息」为禁用态。若短期不实现，建议改为「消息（即将上线）」或隐藏，减少无效点击。 | 低 |

### 1.2 角色与菜单

- **主播/运营/管理**：已按角色区分能力；建议在侧栏或顶栏用轻量标签展示当前角色（如「以主播身份使用」），与钉钉/飞书「当前身份」一致。
- **权限不足时**：进入无权限页面时，除 403 重定向外，可提供一句说明+可去页面（如「您暂无店铺对比权限，可前往数据分析查看单店数据」）。

## 二、数据看板（Dashboard）体验

### 2.1 对标：淘宝/京东数据看板、抖店罗盘

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **数据周期标签更醒目** | 当前有「数据周期：本周」等，可考虑放在统计区正上方或筛选按钮旁，用徽章/标签样式，一眼可见「当前看的是哪段时间」。 | 高 |
| **时效/GPM/人均观看时长释义** | 首次出现或 hover 时给出一句话解释（如「时效：每小时成交额，衡量单位时间产出」），与罗盘「指标说明」对齐。 | 高 |
| **空状态引导** | 「该时间段暂无数据」时，已提示「可尝试选择其他时间周期或导入数据」；可再加一行短链：「导入数据」按钮或链接，减少一步操作。 | 中 |
| **加载态统一** | 统计区加载用骨架屏或与卡片同尺寸的占位，避免布局跳动；与「加载中...」文字可并存。 | 中 |
| **环比/同比标识** | 若有环比数据，明确标「较上周 ±X%」或「较上月」，避免用户误解为「较昨日」。 | 中 |
| **大数可读性** | 万/十万级 GMV 用「1.2 万」「10.5 万」缩写（可选），与罗盘/生意参谋一致，减少位数扫读负担。 | 低 |

### 2.2 筛选面板

- **展开/收起记忆**：考虑记住「筛选面板是否展开」（如 layoutPreferences），下次进入 Dashboard 时恢复，减少重复点击。
- **「自定义」日期**：当前未选日期时已做默认最近 7 天；在 UI 上可标注「未选择时默认最近 7 天」，降低困惑。
- **数据项勾选**：至少保留一项的 Toast 已实现；可考虑在「数据项目」区域标题旁加一句「至少保留一项」，预防性说明。

## 三、店铺与创建流程

### 3.1 对标：抖音罗盘/千牛、多步骤表单

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **无店铺时的 CTA** | 「请选择店铺」+「创建新店铺」已清晰；可增加一句「创建后即可导入直播数据、查看统计」。 | 低 |
| **创建店铺进度/步骤指示** | 多步骤表单可增加顶部步骤条（1 基本信息 2 类目与价格 3 品牌策略），与千牛/罗盘创建流程一致。 | 中 |
| **国家-货币** | 已实现国家切换→推荐货币与价格单位联动；在价格区间旁固定显示单位（如 ฿ THB），避免用户忘记当前货币。 | 已较好 |
| **AI 品牌策略** | 点击「AI自动生成品牌策略」后，若耗时较长，建议按钮变为「生成中…」并禁用，与智能生成任务一致。 | 中 |
| **商店列表选中态** | 已高亮；可补充：列表项 hover 时整行可点击的视觉反馈（如背景色），减少「只能点名字」的探索成本。 | 高 |

## 四、执行工具页

### 4.1 对标：抖音电商工具、有赞/微盟、钉钉/飞书

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **工具卡片描述** | 每个工具除名称外，一行说明「输入什么→得到什么」（如「上传直播明细 Excel，自动汇总到看板」），提升扫一眼即懂。 | 高 |
| **需选店铺的统一样式** | 未选店铺时统一为「请先选择店铺」卡片+图标，与当前部分页面一致；执行工具页已有一块，可复用为公共空状态组件。 | 中 |
| **话术生成** | 已标明「基于店铺 XXX」、数据来源；可考虑在结果区增加「复制整段」「分段复制」按钮，方便直接贴到提词器。 | 中 |
| **生成中状态** | 所有「生成」类按钮在请求中统一为「生成中…」+ 可选倒计时/进度提示（如智能生成约 30 秒），避免重复点击与焦虑。 | 高 |
| **使用指南入口** | 已有「使用指南（提示词模板）」；可放在工具列表上方或固定为「？」图标，首次进入工具页时弱提示。 | 低 |
| **暂无数据/待接入** | 「暂无综合对比数据」「暂无时效对比数据」等建议改为「功能开发中，敬请期待」或简短说明，避免用户误以为是自己数据问题。 | 中 |

## 五、数据分析页

### 5.1 对标：抖音罗盘、生意参谋

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **诊断建议可操作化** | 当前有「转化率偏低」「直播时长不足」等；可考虑每条后跟「查看建议」或链到执行工具（如话术生成、复盘报告），形成「诊断→行动」闭环。 | 中 |
| **店铺对比** | 运营/管理可见；若店铺数不足 2，可提前在入口处提示「需至少 2 个店铺」，与执行工具页「至少需要2个商店」一致。 | 低 |
| **时间周期与筛选** | 与 Dashboard 对齐：支持「选择月份」「选择年份」等，并显式显示当前数据周期。 | 视实现 |
| **图表图例与单位** | 趋势图已显示「成交额」等；纵轴单位与当前「显示为」货币一致，并可在图例旁标注单位（如 ¥ 万）。 | 中 |

## 六、登录与账号

### 6.1 对标：淘宝/抖音开放平台、行业共性

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **登录页品牌与信任** | 当前「直播电商中台」+ 图标已清晰；可加一句 Slogan 或「用于店铺数据管理与直播运营」，强化场景。 | 低 |
| **错误提示** | 已区分网络超时、邮箱密码错误等；可对「账号不存在」「密码错误」做更友好文案（不暴露安全信息）。 | 中 |
| **个人中心** | 修改资料/密码/邮箱的反馈已用 Toast；成功后可考虑短暂显示「已保存」状态，再关闭或跳转。 | 低 |
| **找回密码** | 开发环境「前往重置密码」已说明；正式环境需对接邮件/短信，并在页面上注明「将向您的邮箱发送重置链接」。 | 产品 |

## 七、布局与视觉一致性

### 7.1 对标：Notion/飞书、数据看板产品

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **布局设置入口** | 右下角齿轮已存在；可增加「恢复默认」前的二次确认（如「确定恢复为默认布局吗？」），防止误触。 | 低 |
| **卡片与间距** | 统计卡片、任务卡片、工具卡片风格统一（圆角、阴影、内边距），已基本一致；保持各页 card 类统一。 | 已较好 |
| **主色与状态色** | 成功/错误/警告 Toast 已有区分；全局主色（如蓝色）与链接、按钮、高亮保持一致。 | 已较好 |
| **深色模式** | 若后续支持，需统一侧栏、卡片、表格对比度与可读性；非必须，可作为后续迭代。 | 低 |

## 八、错误与边界

### 8.1 对标：成熟 Web 应用

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **接口失败** | 已用 Toast；可对「网络错误」「服务暂时不可用」提供「重试」按钮（如刷新列表、重新请求）。 | 中 |
| **401 跳转登录** | 已实现；跳转前可 Toast「登录已过期，请重新登录」，减少困惑。 | 中 |
| **导入失败** | 已有错误提示；若为格式问题，可列举「请检查：日期列、表头名称、文件编码」，与《TikTok数据导入使用指南》呼应。 | 中 |
| **数值异常** | 模块 13 将做数值范围保护；前端已对环比等做 ±1000 限制；大额 GMV 等建议统一用千分位与单位，避免 NaN/Infinity 展示。 | 中 |

## 九、性能与体感

### 9.1 对标：商家后台体感

| 建议 | 说明 | 优先级 |
|------|------|--------|
| **首屏与路由** | 已懒加载、代码分割；保持首屏 < 2s 体感，长列表用虚拟滚动。 | 保持 |
| **智能生成等待** | 已有时长提示与超时回退；可考虑在等待期间展示「正在分析店铺数据…」「正在生成建议…」等分步提示，降低焦虑。 | 中 |
| **切换店铺** | 切换后统计/任务/素材已按 storeId 隔离；加载时建议保留上一店铺的骨架或占位，避免白屏再刷出。 | 低 |

## 十、建议实施顺序（按优先级与依赖）

1. **高优先级（可立即做）**  
   - 侧栏收起时菜单项 Tooltip  
   - 数据周期标签更醒目 + 时效/GPM/人均观看时长释义  
   - 商店列表整行可点 + hover 反馈  
   - 执行工具「生成中…」统一 + 工具卡片一行说明  
   - 数据周期与指标说明的「？」或帮助入口  

2. **中优先级（短期迭代）**  
   - 面包屑、筛选展开记忆、环比标识、创建店铺步骤条  
   - 诊断建议可操作化、图表单位标注  
   - 401 前 Toast、导入失败检查清单、重试按钮  

3. **低优先级（体验加分）**  
   - 消息入口文案或隐藏、无店铺时 CTA 文案、大数缩写  
   - 布局重置二次确认、深色模式（可选）
