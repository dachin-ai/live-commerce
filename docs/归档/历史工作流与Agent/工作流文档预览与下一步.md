# 工作流文档预览修复与下一步计划

## 一、已修复：文档点击「未登录」问题

**原因**：角色产出中的文件链接为 `<a href="/api/workflow/outputs/...">`，点击后由浏览器直接请求该 URL，不会携带前端的 JWT，请求先经过工作流路由的 `authenticate`，返回 `{"error":"未登录"}`。

**实现**：

1. **后端**  
   - 新增带鉴权接口：`GET /api/workflow/outputs/file?roundLabel=xxx&path=yyy`（需登录且仅管理员）。  
   - 在 `workflow` 路由内实现：校验路径、防路径穿越，读取 `outputs/第N轮迭代/` 下文件，文本类返回 `{ content }`，`.json` 返回解析后的 JSON。  
   - 移除对 `outputs/` 的静态挂载，产出文件统一经该接口访问。

2. **前端**  
   - 新增 `getOutputFileContent(roundLabel, filePath)`，通过 api 客户端（带 Token）请求上述接口。  
   - 角色产出中的文件名改为按钮，点击后调用该接口，在弹窗中展示内容（`<pre>` 展示原文，JSON 已格式化）。

效果：管理员在工作流页面点击任意产出文件名，会通过已登录会话拉取内容并在弹窗中预览，不再出现「未登录」错误。

## 二、下一步计划建议

| 优先级 | 项 | 说明 |
|--------|----|------|
| P1 | 文档预览体验 | 对 `.md` 做简单 Markdown 渲染（如引入 react-markdown），JSON 保持当前格式化展示即可。 |
| P2 | 下载按钮 | 在预览弹窗中增加「下载」按钮，将当前展示内容以原文件名触发下载（Blob + a.download）。 |
| P3 | 试运行与诊断 | 在「立即执行一轮」完成后，若某角色失败，在轮次详情中展示错误信息或日志链接，便于排查。 |
| P4 | 定时/触发配置 | 在管理端增加「下一轮计划时间」或「触发条件」配置（如 cron 表达式），与 scheduler 联动，实现 4–7 天/轮。 |

以上可作为后续迭代的待办参考；当前文档界面已能正常预览且带鉴权。
