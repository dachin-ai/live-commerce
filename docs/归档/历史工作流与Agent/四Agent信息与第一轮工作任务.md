# 四 Agent 信息与第一轮工作任务

本文档由**规划师**整理：将工作流引擎中的角色归纳为**四名 Agent**，并为其安排**第一轮**具体工作任务。  
（代码中为五角色；第一轮以「规划师、行业专家、资深用户、程序工程师」四名 Agent 为主；**小白**由用户本人担任，基于当轮产出提疑问、暴露盲点。）

- **小白**：由你（用户）担任，每轮结束后可查看 `outputs/第N轮迭代/疑问库.md` 并补充自己的疑问。
- **触发第一轮**：已提供脚本 `backend/scripts/trigger-workflow-round.ts`，执行 `cd backend && npx tsx scripts/trigger-workflow-round.ts`（需先启动后端）；或管理员在前端「工作流」页点击「立即执行一轮」。

---

## 一、四名 Agent 信息表

| 序号 | 角色 ID | 角色名 | 职责简述 | 输入（上游） | 产出物 | 代码位置 |
|------|---------|--------|----------|--------------|--------|----------|
| 1 | **planner** | **规划师** | 制定优化方向与路线图；若有上一轮则基于上一轮产出做延续 | 上一轮：主题.md、路线图.md（首轮无） | 主题.md、路线图.md | `backend/src/workflow_engine/roles/planner.ts` |
| 2 | **industry_expert** | **行业专家** | 基于规划师主题与路线图，输出电商直播最佳实践 | 当轮：主题.md、路线图.md | 行业最佳实践.json | `backend/src/workflow_engine/roles/industry_expert.ts` |
| 3 | **senior_user** | **资深用户** | 基于行业最佳实践，输出体验测试与 A/B 测试建议 | 当轮：行业最佳实践.json | 用户体验测试报告.md、AB测试建议.md | `backend/src/workflow_engine/roles/senior_user.ts` |

**说明**：第五角色「小白」(novice) 产出疑问库.md，第一轮可由规划师在收尾时做简要「首轮疑问汇总」并入路线图或单独小节，或留待第二轮再启用小白 Agent。

## 二、第一轮执行顺序与依赖

```
规划师(1) → 行业专家(2) → 资深用户(3) → 程序工程师(4)
   ↓              ↓                ↓                 ↓
主题+路线图   行业最佳实践    体验报告+A/B建议    技术方案+部署包
```

- **顺序**：必须按 1→2→3→4 执行，下游依赖上游当轮产出。
- **触发方式**：管理员在工作流页点击「立即执行一轮」；或调用 `POST /api/workflow/trigger`（不传 roundId 时自动生成新轮次）。
- **产出目录**：`outputs/第1轮迭代/`（或当前轮次标签对应目录）。

## 三、第一轮各 Agent 具体工作任务

### Agent 1：规划师（planner）

| 项 | 内容 |
|----|------|
| **第一轮输入** | 无上一轮时无文件输入；若有上一轮则读取上一轮 主题.md、路线图.md |
| **第一轮任务** | 1）撰写《季度优化主题》，明确本轮聚焦（如直播转化率与时长优化）；2）撰写《路线图》，列出 4 步：数据基础强化、用户体验与 A/B 测试、技术实施与部署包生成、疑问与盲点收集 |
| **产出物** | 主题.md、路线图.md（落盘至 `outputs/第1轮迭代/`） |
| **验收标准** | 主题.md 含「本轮聚焦」与轮次/时间；路线图.md 含至少 4 项可执行步骤；若为第 2 轮及以后，需体现对上一轮的延续 |
| **依赖** | 无（首棒） |

---

### Agent 2：行业专家（industry_expert）

| 项 | 内容 |
|----|------|
| **第一轮输入** | 当轮 主题.md、路线图.md（由 output_collector 读取） |
| **第一轮任务** | 1）解析规划师主题与路线图；2）生成结构化《行业最佳实践》JSON，包含 area、recommendation、priority、source 等；3）内容需与「转化率」「A/B」等主题呼应 |
| **产出物** | 行业最佳实践.json |
| **验收标准** | JSON 可解析；含 practices 数组；至少包含话术、互动、商品展示等方向；若主题含「转化率」则推荐中体现转化率相关建议 |
| **依赖** | 规划师产出就绪 |

---

### Agent 3：资深用户（senior_user）

| 项 | 内容 |
|----|------|
| **第一轮输入** | 当轮 行业最佳实践.json |
| **第一轮任务** | 1）基于行业最佳实践撰写《用户体验测试报告》；2）撰写《A/B 测试建议》，与行业方案中的话术/互动等结合 |
| **产出物** | 用户体验测试报告.md、AB测试建议.md |
| **验收标准** | 报告含「依据（行业最佳实践摘要）」与「测试结论」；A/B 建议含至少 2 项可执行建议（如标题样式 A/B、开播时间 A/B），并与行业方案有引用关系 |
| **依赖** | 行业专家产出就绪 |

---

### Agent 4：程序工程师（engineer）

| 项 | 内容 |
|----|------|
| **第一轮输入** | 当轮 行业最佳实践.json、用户体验测试报告.md、AB测试建议.md |
| **第一轮任务** | 1）基于上游三件产出撰写《技术实施方案》；2）调用 AI 代码生成（若已配置）或使用基础模板，生成部署包（如 README.md、部署脚本等）；3）将技术方案与部署包落盘 |
| **产出物** | 技术实施方案.md、部署包/（内含 README.md 及可选脚本） |
| **验收标准** | 技术实施方案.md 中「依据」列出上游产出；实施项包含配置更新与脚本/部署包；部署包目录下至少有一份说明或脚本 |
| **依赖** | 行业专家、资深用户产出就绪 |

## 四、第一轮收尾（可选：疑问汇总）

- **方案 A**：不单独运行「小白」Agent，由**规划师**在下一轮（第 2 轮）开始时，在主题/路线图中增加「首轮疑问汇总」小节（如：数据采集频率、A/B 流量分配、部署回滚等），实现闭环。
- **方案 B**：第一轮仍按现有引擎执行**五角色**，小白在程序工程师之后运行，产出 疑问库.md，供下一轮规划师参考。
- **建议**：若你明确只启用「四名 Agent」，采用方案 A；若希望第一轮就产出疑问库，采用方案 B（即第一轮实际执行 5 个角色，但对外称「四名主 Agent + 一名收尾 Agent」）。

---

## 五、如何执行第一轮

1. **后端与状态**：确保后端已启动（`cd backend && npm run dev`），`data/shared_state/state.db` 存在（首次会自动创建）。
2. **触发**：管理员登录前端 → 进入「工作流」/「迭代」页 → 点击 **「立即执行一轮」**；或调用接口 `POST /api/workflow/trigger`，body 可为 `{}`。
3. **查看结果**：  
   - 轮次列表：`GET /api/workflow/rounds`  
   - 某轮详情：`GET /api/workflow/rounds/:roundId`  
   - 产出文件：`outputs/第1轮迭代/` 下查看 主题.md、路线图.md、行业最佳实践.json、用户体验测试报告.md、AB测试建议.md、技术实施方案.md、部署包/ 等。
4. **排查**：若某角色失败，查看轮次详情中的错误信息或后端日志；产出路径见 `backend/src/workflow_engine/output_collector.ts` 与 `role_dispatcher.ts`。

## 六、小结

| Agent | 角色名 | 第一轮核心任务 | 产出 |
|-------|--------|----------------|------|
| 1 | 规划师 | 定主题与路线图（首轮无上游） | 主题.md、路线图.md |
| 2 | 行业专家 | 据主题与路线图出最佳实践 JSON | 行业最佳实践.json |
| 3 | 资深用户 | 据行业方案出体验报告与 A/B 建议 | 用户体验测试报告.md、AB测试建议.md |
| 4 | 程序工程师 | 据上游出技术方案与部署包 | 技术实施方案.md、部署包/ |

四名 Agent 按 1→2→3→4 顺序执行，第一轮无需上一轮产出；从第二轮起规划师将读取上一轮主题与路线图做延续。小白角色可按上文「第一轮收尾」两种方案选用。
