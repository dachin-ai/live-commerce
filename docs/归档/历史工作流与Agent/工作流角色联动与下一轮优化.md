# 工作流角色联动与下一轮优化

## 一、角色联动（上游产出作为下游输入）

各角色按固定顺序执行，**下游角色会读取上游角色在本轮已写出的文件**，作为自己产出的依据，实现「规划师 → 行业专家 → 资深用户 → 程序工程师 → 小白」的联动。

| 角色 | 读取的上游产出（当轮） | 产出 |
|------|------------------------|------|
| 规划师 | **上一轮**的 主题.md、路线图.md（若有） | 主题.md、路线图.md |
| 行业专家 | 规划师的 主题.md、路线图.md | 行业最佳实践.json |
| 资深用户 | 行业专家的 行业最佳实践.json | 用户体验测试报告.md、AB测试建议.md |
| 程序工程师 | 行业最佳实践.json、用户体验测试报告.md、AB测试建议.md | 技术实施方案.md、部署包/README.md |
| 小白 | 当轮全部产出（主题、路线图、行业方案、报告、A/B 建议、技术方案） | 疑问库.md |

实现方式：

- **output_collector** 提供：
  - `readRoundFileContent(roundLabel, relativePath)`：读取某轮某文件内容，供角色内使用。
  - `getPreviousRoundLabel(roundLabel)`：由当前轮标签得到上一轮标签（如 第2轮迭代 → 第1轮迭代）。
- 各角色执行器在执行时调用上述接口，读取当轮或上一轮的文件，将摘要或引用写入自己的产出中。

## 二、下一轮优化（如何开始下一轮）

### 1. 谁在「下一轮」里读上一轮？

- **规划师**在**第 N 轮**（N≥2）执行时，会通过 `getPreviousRoundLabel(context.roundLabel)` 得到「第 N-1 轮迭代」，并读取：
  - `主题.md`
  - `路线图.md`
  将上一轮主题/路线图摘要写入本轮「季度优化主题」中，实现**跨轮延续与下一轮优化**。

### 2. 如何触发「下一轮」？

- **立即执行一轮（推荐）**  
  管理员在「工作流 / 迭代」页点击 **「立即执行一轮」**。  
  后端会：
  1. 根据 `state.db` 中的 `last_round_id` 自动生成下一轮 ID 与标签（如 第1轮迭代 → 第2轮迭代）。
  2. 按序执行五角色；规划师在本轮会读取上一轮产出（若存在）。
  3. 将本轮产出写入 `outputs/第N轮迭代/`，并更新检查点与游标。

- **定时/计划触发（可选扩展）**  
  若后续接入定时任务（如 node-cron 或外部调度），可配置每 4–7 天调用一次 `POST /api/workflow/trigger`（不传 `roundId`/`roundLabel`），效果与点击「立即执行一轮」相同，实现周期性的下一轮优化。

### 3. 数据回流与规划师决策（可选扩展）

- 当前规划师仅基于**上一轮的主题与路线图**做延续。
- 若需「根据业务数据决定是否/如何下一轮」，可扩展：
  - 在 state 或业务库中写入每轮结束后的指标快照（如 GMV、任务完成数）。
  - 规划师执行时除读取上一轮产出外，再读取这些指标，在主题/路线图中体现「数据驱动」的下一轮优化方向。
- 迭代周期（4–7 天）可由 scheduler 的定时配置控制，无需改角色逻辑。

## 三、小结

- **联动**：各角色通过 `readRoundFileContent(roundLabel, path)` 读取当轮或上一轮产出，并将依据写进自己的产出，形成链式输入。
- **下一轮**：点击「立即执行一轮」即会创建并执行下一轮；规划师在第二轮及之后会自动读取上一轮主题与路线图，实现下一轮优化；后续可加定时触发与数据回流，进一步自动化。
