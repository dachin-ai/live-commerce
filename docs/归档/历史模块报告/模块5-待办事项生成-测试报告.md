## 6. 功能与布局总结

- **功能**：按当前所选店铺与运营数据（含历史、阶段、节日、趋势、异常）生成待办；LLM 已配置时优先生成 AI 智能建议（场景化表述），否则或超时后为规则引擎兜底；条数不限制；支持紧急/普通筛选与完成操作。
- **布局**：主区「待处理任务」卡片（标题、共 N 任务 / N 紧急、智能生成按钮、全部/紧急/普通 Tab、任务列表）；列表项含图标、标题、店铺标签、描述、✓ 完成。
- **亮点**：LLM+规则双路径、来源可追溯（metadata）、快速调试开关（SKIP_TODO_LLM）、生成中提示「约30秒」。

## 7. 实现说明

- **后端**：`backend/src/routes/ai-refactored.ts` — `POST /generate-tasks`（挂载于 `/api/ai`）；多维度上下文收集 → 若已配置 LLM 且未设 SKIP_TODO_LLM则调用 `generateIntelligentTodosWithLLM`，否则或失败后走规则兜底；规则含异常、阶段、趋势、节日、动态阈值、冷启动短视频等；去重后批量写入 `tasks` 表。
- **前端**：`TaskList.tsx`（待办列表、智能生成按钮、筛选）、`AIFeatures.tsx`（入口）；请求超时 60s，Toast 展示「含 X 条 AI 智能建议」当 metadata.llmIntelligentCount > 0。
- **脚本**：`backend/scripts/test-generate-tasks-api.ts` — 登录 → 取店铺 → POST /api/ai/generate-tasks → 校验 message/tasks/metadata 及任务项结构。
