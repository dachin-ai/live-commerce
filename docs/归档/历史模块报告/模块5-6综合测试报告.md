### 数据分析逻辑（对标用户提供的参考）

#### 1. 分析哪些数据 ✅
- **店铺信息**: 名称、地区、类目、价格、受众、定位
- **直播指标**: GMV、订单数、观看数、互动数、时长、场次
- **时间维度**: 日期、季节、节日、旺季
- **历史数据**: 最近30场直播的场均数据、趋势

#### 2. 如何分析 ✅
- **指标计算**: 转化率、每小时GMV、互动率、客单价
- **阈值比较**: 与行业平均（3.5%）、合理范围对比
- **店铺分层**: 根据GMV分为小型、中型、大型、超大型
- **趋势判断**: 对比前后半段数据，判断上升、下降、稳定

#### 3. 如何生成待办 ✅
- **基于阈值**: 当指标低于/高于阈值时自动触发
- **优先级判定**: 根据偏离程度设定urgent/normal
- **量化描述**: 包含当前值、目标值、预期效果
- **具体措施**: 至少3条可执行的操作步骤

#### 4. 生成待办的质量要求 ✅
- ✅ 标题 < 20字
- ✅ 描述 < 100字（关键信息）
- ✅ 禁用模糊词（"优化""提升""完善"）
- ✅ 使用具体动词（"每天设置""每周发布"）
- ✅ 量化预期效果（百分比、金额、数量）
- ✅ 基于真实数据，有兜底机制

### 任务生成规则
| 任务类型 | 触发条件 | 优先级 | 量化标准 |

| 转化率优化 | 转化率 < 3% 且 观看数 > 100 | 转化率 < 1.5% → urgent | 目标3.5%，预期增加订单数 |
| 直播时长优化 | 时长 < 20小时 或 GMV/小时 < 5000 | 时长 < 15小时 → urgent | 目标+10小时，预期GMV增长 |
| 互动率优化 | 互动率 < 10% 且 观看数 > 50 | 互动率 < 5% → urgent | 目标15%，预期增加互动次数 |
| 客单价优化 | 客单价 < 200 且 订单数 > 10 | normal | 目标¥250，预期GMV增长 |
| 观看人数增长 | 观看数 < 500 且 时长 > 10小时 | 观看数 < 200 → urgent | 目标800人/周 |

#### 2. 关键指标 ✅
```
- GMV: 总交易额
- Order Count: 订单数
- View Count: 观看次数
- Conversion Rate: 转化率 = 订单数 / 观看数 × 100
- Duration: 直播时长
- Session Count: 直播场次
- Interaction Count: 互动数
- GMV per Hour: 每小时GMV = GMV / 时长
- Interaction Rate: 互动率 = 互动数 / 观看数 × 100
- Average Order Value: 客单价 = GMV / 订单数
```

#### 3. 多维度分析框架 ✅
- **店铺维度**: 地区、类目、价格、受众、定位
- **用户维度**: 角色（admin/operator/anchor）、语言

- **历史数据维度**: 场均GMV、平均转化率、店铺分层、趋势

#### 4. 待办生成方式 ✅
