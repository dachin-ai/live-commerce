### 数据分析逻辑

#### 1. 数据来源
- **stats表**: 店铺统计数据（totalGMV, totalDuration, totalViewers, totalOrders, totalInteractions）
- **兜底机制**: 如果店铺无数据，使用最近10条记录的平均值

#### 2. 关键指标计算
```typescript
- 转化率 (Conversion Rate) = orders / viewers × 100
- 每小时GMV (GMV per Hour) = GMV / duration
- 互动率 (Interaction Rate) = interactions / viewers × 100
- 客单价 (Average Order Value) = GMV / orders
```

#### 3. 任务生成规则

| 任务类型 | 触发条件 | 优先级判定 | 量化标准 |
|---------|---------|------------|----------|
| **转化率优化** | 转化率 < 3% 且 观看数 > 100 | 转化率 < 1.5% → urgent<br>否则 → normal | 目标转化率：3.5%<br>预期增加订单数<br>预期提升GMV |
| **直播时长优化** | 时长 < 20小时 或 每小时GMV < 5000 | 时长 < 15小时 → urgent<br>否则 → normal | 目标时长：当前+10小时（最高30小时/周）<br>预期GMV增长（80%效率） |
| **互动率优化** | 互动率 < 10% 且 观看数 > 50 | 互动率 < 5% → urgent<br>否则 → normal | 目标互动率：15%<br>预期增加互动次数 |
| **客单价优化** | 客单价 < 200 且 订单数 > 10 | normal | 目标客单价：¥250<br>预期GMV增长 |
| **观看人数增长** | 观看数 < 500 且 时长 > 10小时 | 观看数 < 200 → urgent<br>否则 → normal | 目标观看数：800人/周<br>预期增加人数 |

#### 4. 兜底任务
- **无数据时**: "上传基准数据"（urgent）、"完善店铺信息"（normal）
- **数据良好时**: "数据分析与优化"（normal）

## 📝 数据分析逻辑总结（根据用户提供的参考）

### 1. 分析哪些数据 ✅
- **店铺信息**: 名称、地区、类目、价格、受众、定位
- **用户信息**: 角色、语言
- **直播指标**: GMV、订单数、观看数、互动数、时长、场次
- **时间维度**: 日期、季节、节日、旺季
- **历史数据**: 最近30场直播的场均数据、趋势判断

### 2. 如何分析 ✅
- **指标计算**: 转化率、每小时GMV、互动率、客单价
- **阈值比较**: 与行业平均（3.5%）、合理范围（15%-30小时）对比
- **店铺分层**: 根据GMV分为小型、中型、大型、超大型
- **趋势判断**: 对比前后半段数据，判断上升、下降、稳定

### 3. 如何生成待办 ✅
- **基于阈值**: 当指标低于/高于阈值时自动触发
- **优先级判定**: 根据偏离程度设定urgent/high/medium/low
- **量化描述**: 包含当前值、目标值、预期效果
- **具体措施**: 至少3条可执行的操作步骤

### 4. 生成待办的质量要求 ✅
- ✅ **标题简短**: < 20字，包含具体数值
- ✅ **描述具体**: < 100字，避免模糊词
- ✅ **量化指标**: 明确的数值目标（百分比、金额、数量）
- ✅ **预期效果**: 说明完成后的量化提升
- ✅ **具体措施**: 可直接执行的操作（含频率、时间段、具体方式）
- ✅ **数据支撑**: 基于真实数据，而非凭空生成
