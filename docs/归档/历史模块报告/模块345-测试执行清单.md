# 模块 3、4、5 测试执行清单

按《各模块步骤总结》执行，逐项勾选。

---

## 前置条件

- [ ] 后端已启动：`cd backend && npm run dev`，http://localhost:3000/health 返回 ok
- [ ] 前端已启动：`cd frontend && npm run dev`，http://localhost:5173 可访问
- [ ] 已登录（如 admin@example.com / 123456）
- [ ] 已选择至少一个店铺（无店铺时先创建）

---

## 模块 3：数据筛选（Dashboard 看板）

| 步骤 | 操作 | 预期 | 结果 |
|------|------|------|------|
| 1 | 在「直播数据统计」卡片右上角点击 **「筛选数据」** | 筛选面板展开，出现「数据项目」「时间周期」 | ☐ |
| 2 | 在「数据项目」中取消勾选 **「直播时长」**（即取消高亮/选中） | 统计卡片中「总直播时长」卡片隐藏 | ☐ |
| 3 | 在「时间周期」中点击 **「本周」** | 选中「本周」，标签与数据周期一致 | ☐ |
| 4 | 点击 **「收起筛选」** | 筛选面板收起 | ☐ |

**通过标准**：筛选功能正常，数据实时更新 → 模块 3 通过

---

## 模块 4：布局设置

| 步骤 | 操作 | 预期 | 结果 |
|------|------|------|------|
| 1 | 点击页面 **右下角蓝色齿轮图标** | 布局设置面板打开 | ☐ |
| 2 | 在「功能区显示/隐藏」中关闭 **「店铺列表」**（点击使其变为灰色/隐藏） | 店铺列表区域从页面隐藏 | ☐ |
| 3 | 在「功能区大小」中把 **「数据统计」列数** 调整为 **2** | 数据统计区域变为 2 列宽度 | ☐ |
| 4 | 关闭 **「图标显示」**（显示图标 → 关闭） | 统计/趋势等图标隐藏 | ☐ |
| 5 | 按 **F5 刷新页面** | 上述设置保持（店铺列表仍隐藏、列数仍为 2、图标仍隐藏） | ☐ |
| 6 | 再次打开布局设置，点击 **「重置为默认」** | 恢复初始状态（店铺列表、图标等恢复显示） | ☐ |

**通过标准**：布局设置即时生效、持久化正常 → 模块 4 通过

---

## 模块 5：智能任务生成

| 步骤 | 操作 | 预期 | 结果 |
|------|------|------|------|
| 1 | 在右侧「待处理任务」卡片中点击 **「智能生成刷新」** | 发起生成请求，无报错；若有店铺未选会提示「请先选择店铺」 | ☐ |
| 2 | 查看生成的任务标题 | 含具体数值与目标（如「提升转化率至 3.5%」等） | ☐ |
| 3 | 查看任务描述 | 包含当前值/目标值、至少 3 条具体措施 | ☐ |
| 4 | 查看任务列表排序与标识 | 紧急任务排序在前、有红色紧急标识 | ☐ |

**通过标准**：任务生成质量符合要求 → 模块 5 通过

---

## 备注

- **模块 3**：筛选里的选项是「直播时长」，对应统计卡片标题「总直播时长」；取消选中「直播时长」后，总直播时长卡片应消失。
- **模块 4**：布局设置通过 localStorage 持久化，键名为 `layoutPreferences`。
- **模块 5**：智能生成依赖后端 `POST /api/ai/generate-tasks`，需先选择店铺；主播角色不显示任务管理区域。
