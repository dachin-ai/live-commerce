# 用户逻辑重构说明

## 📋 已完成的工作

### 1. ✅ 创建分析层页面（AnalysisPage）

**位置：** `frontend/src/pages/AnalysisPage.tsx`

**功能特性：**
- ✅ 数据筛选功能：支持选择数据项目（GMV、时长、观看人数、互动数、订单数、转化率）
- ✅ 时间周期筛选：今天、本周、本月、本季度、本年、自定义
- ✅ 诊断建议：基于实际数据生成量化、有逻辑的诊断建议
  - 转化率偏低提示（低于2%时）
  - 直播时长不足提示（低于20小时时）
  - 互动率偏低提示（低于10%时）
- ✅ 店铺对比功能（管理/运营角色可见）
- ✅ 响应式布局，支持不同屏幕尺寸

**访问路径：** `/analysis`

### 2. ✅ 改进待办事项生成逻辑

**位置：** `backend/src/routes/ai.ts` - `/generate-tasks` 接口

**改进内容：**
- ✅ **量化指标**：所有任务都包含具体的数据指标
  - 当前值 vs 目标值
  - 预计增长量（订单数、GMV、互动数等）
  - 具体百分比和金额
- ✅ **有逻辑的分析**：基于实际店铺数据生成任务
  - 转化率优化（低于3%时）
  - 直播时长优化（低于20小时或GMV/小时偏低时）
  - 互动率优化（低于10%时）
  - 客单价优化（低于200元时）
  - 观看人数增长（低于500人时）
- ✅ **避免空泛表达**：每个任务都包含：
  - 当前具体数值
  - 目标数值
  - 预计效果（量化）
  - 具体措施（3条以上）

**示例任务：**
```
标题：提升转化率至 3.5%
描述：当前转化率 2.54%，低于行业平均 3.5%。通过优化直播话术、增加互动环节、优化商品展示，预计可增加 12 单订单，提升 GMV ¥3,600。具体措施：1) 在直播前30秒强调核心优惠；2) 每10分钟设置一次互动抽奖；3) 优化商品展示顺序，将高转化商品前置。
```
