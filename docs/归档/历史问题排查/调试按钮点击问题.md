# 调试按钮点击问题

## 🔍 问题排查步骤

### 1. 检查浏览器控制台

**打开浏览器开发者工具：**
- 按 `F12` 或 `Ctrl+Shift+I`
- 切换到 "Console"（控制台）标签

**查看是否有错误信息：**
- 如果看到红色错误，说明有问题
- 如果看到 `点击按钮: XXX`，说明按钮点击事件已触发

### 2. 检查登录状态

**检查是否已登录：**
1. 打开浏览器控制台
2. 输入：`localStorage.getItem('token')`
3. 如果返回 `null`，说明未登录，需要先登录

**登录方法：**
- 访问 http://localhost:5173/login
- 使用测试账号：
  - 管理员：admin@example.com / 123456
  - 普通用户：operator@example.com / 123456

### 3. 检查后端服务

**检查后端是否在线：**
- 访问 http://localhost:3000/health
- 应该返回：`{"status":"ok","timestamp":"..."}`

**如果后端离线：**
- 运行 `单独启动后端.bat`
- 或运行 `快速启动.bat`

### 4. 检查网络请求

**在浏览器开发者工具中：**
1. 切换到 "Network"（网络）标签
2. 点击一个按钮
3. 查看是否有网络请求发出
4. 查看请求的状态码：
   - `200` = 成功
   - `401` = 未授权（需要登录）
   - `500` = 服务器错误
   - `404` = 接口不存在

### 5. 常见问题

#### 问题1：按钮点击后没有任何反应

**可能原因：**
- JavaScript 错误阻止了事件处理
- 按钮被禁用（`disabled` 属性）

**解决方法：**
1. 查看控制台是否有错误
2. 检查按钮是否显示为禁用状态（灰色）

#### 问题2：按钮点击后显示错误提示

**查看错误信息：**
- 如果显示"未授权"或"401"，需要登录
- 如果显示"网络连接失败"，检查后端服务
- 如果显示其他错误，查看控制台详细信息

#### 问题3：按钮点击后自动跳转到登录页

**原因：**
- Token 已过期或无效
- 需要重新登录

**解决方法：**
1. 重新登录
2. 检查 token 是否有效

## 📋 测试步骤

1. **打开浏览器开发者工具**（F12）
2. **切换到 Console 标签**
3. **点击任意按钮**
4. **查看控制台输出：**
   - 应该看到：`点击按钮: XXX`
   - 如果有错误，会显示详细错误信息
5. **切换到 Network 标签**
6. **再次点击按钮**
7. **查看网络请求：**
   - 请求是否发出
   - 状态码是什么
   - 响应内容是什么

## 💡 提示

- **如果按钮完全没有反应：** 检查控制台是否有 JavaScript 错误
- **如果显示"未授权"：** 需要先登录
- **如果显示"网络错误"：** 检查后端服务是否运行
- **如果按钮显示为禁用：** 可能是其他操作正在进行中
