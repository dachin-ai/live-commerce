# 问题修复说明

## 🔍 发现的问题

### 问题：`serialize binary: invalid int 32: 4294967295` 错误

**原因分析：**
1. **useMemo 误用**：在 Dashboard.tsx 中，格式化函数（formatCurrency, formatDuration等）被错误地用 `useMemo` 包装返回函数，这会导致：
   - 每次渲染时创建新的函数引用
   - `filteredStats` 的依赖项包含这些函数，导致 useMemo 失效
   - 可能触发无限重新渲染循环

2. **依赖项问题**：`filteredStats` 的 useMemo 依赖项包含了格式化函数，但这些函数每次都是新的引用，导致缓存失效。

## ✅ 修复方案

### 修复1: 格式化函数移到 useMemo 内部

**修改前：**
```typescript
const formatCurrency = useMemo(() => (value: number) => {
  return `¥${value.toLocaleString('zh-CN')}`
}, [])

const filteredStats = useMemo(() => {
  // 使用 formatCurrency
}, [stats, selectedDataItems, formatCurrency, ...])
```

**修改后：**
```typescript
const filteredStats = useMemo(() => {
  // 在 useMemo 内部定义格式化函数
  const formatCurrency = (value: number) => `¥${value.toLocaleString('zh-CN')}`
  const formatDuration = (hours: number) => { ... }
  // ...
  
  // 使用这些函数
}, [stats, selectedDataItems]) // 只依赖实际数据
```

**优势：**
- 格式化函数不会成为依赖项
- 避免不必要的重新计算
- 减少序列化负担

### 修复2: 优化 statsCols 计算

保持 useMemo 的依赖项最小化，只包含实际需要的值。

## 📋 其他优化

1. **图标显示控制**：在需要显示图标的地方都添加了 `preferences.showIcons` 检查
2. **布局计算**：使用 useMemo 缓存布局计算，避免每次渲染重新计算
3. **错误边界**：已添加 ErrorBoundary 组件捕获渲染错误
