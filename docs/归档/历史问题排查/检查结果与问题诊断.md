## 问题诊断

### 根本原因
**后端服务未运行或数据库连接失败**，导致所有 API 返回 500。

### 可能的具体原因
1. **后端服务未启动**：需要运行「单独启动后端.bat」或「启动项目.bat」
2. **数据库损坏或锁定**：SQLite 数据库文件可能被锁定或损坏
3. **数据库表结构不匹配**：如果数据库是旧版本，表结构可能与代码不匹配
4. **数据库中没有分类数据**：如果数据库为空或只有旧数据，新分类不会自动插入

## 解决方案

### 步骤 1：检查后端服务状态
```bash
# 检查端口 3000 是否被占用
netstat -ano | findstr ":3000"
```

### 步骤 2：重启后端服务
1. 如果后端未运行：运行 `单独启动后端.bat`
2. 如果后端已运行但报错：停止后端，然后重新启动

### 步骤 3：重置数据库（推荐）
1. 运行根目录 **「重置数据库.bat」**
2. 确认删除数据库
3. 重启后端服务
4. 后端启动时会自动创建新数据库并插入 18 个一级类目

1. 刷新前端页面（Ctrl+Shift+R）
