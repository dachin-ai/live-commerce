## 二、前端功能清单与验证项

### 1. 「导入数据」按钮（顶部）

- **行为**：点击后打开 `DataImportModal`，当前选中的店铺作为导入目标。  
- **校验**：仅当店铺平台为 **抖音 / TikTok** 时可导入；否则 Toast「该功能仅支持TikTok/抖音平台店铺」。  
- **流程**：选择 Excel → 点击「开始导入」→ 请求 `POST /api/data-import/tiktok` → 成功显示条数、统计摘要，约 3 秒后关闭弹窗。  
- **已修复**：`result` 可能为 undefined 时访问 `result.message` 导致报错，已改为 `result?.message` 及结果区可选链。

**验证建议**：选中抖音/抖音店铺，点击「导入数据」，选择 TikTok 格式 Excel，确认导入成功且无报错。

### 2. 商店列表 - 「上传」按钮（每行绿色上传图标）

- **原状**：仅有图标，无点击逻辑。  
- **现实现**：点击某店铺的「上传」→ 将该店铺设为当前选中店铺并打开「导入数据」弹窗（与顶部「导入数据」同一弹窗，只是预选了该店铺）。  
- **说明**：与「上传基准数据」待办一致，都是通过同一导入流程上传 GMV、直播时长、观看人数等基准数据。

**验证建议**：在商店列表某一行点击上传图标，应弹出导入数据弹窗且当前店铺为该行店铺。

### 3. 待处理任务 - 「上传基准数据」

- **含义**：提示用户为该店铺上传基准数据（GMV、直播时长、观看人数等）。  
- **操作**：用户通过顶部「导入数据」或商店列表「上传」完成上传后，可在该任务上点击「标记完成」。

### 4. 待处理任务 - 「标记完成」

- **行为**：点击后请求 `PUT /api/tasks/:id`，`body: { status: 'completed' }`，成功后列表刷新，该任务从待处理中移除。  
- **后端**：已验证更新成功并返回更新后任务。

**验证建议**：在待处理任务中任选一条，点击「标记完成」，该任务应消失或移出待处理列表。

### 5. 「智能生成刷新」

- **行为**：请求 `POST /api/ai/generate-tasks`，根据当前选中店铺生成/刷新任务列表，成功后刷新任务列表并 Toast「成功生成新任务！」。  
- **后端**：`/api/ai/generate-tasks` 会插入新任务到 DB。

**验证建议**：选择有数据的店铺，点击「智能生成刷新」，确认任务列表更新且无报错。

### 6. 直播数据统计与筛选

- **接口**：`GET /api/stats/live?timeRange=week|month|...&storeId=xxx`，后端已支持并按周期返回数据。  
- **前端**：筛选数据、时间周期、数据项目勾选等（属模块 3 数据筛选，不在此报告展开）。

### 7. 商店列表 - 下载 / 删除

- **下载**：当前为占位，无后端下载接口。  
- **删除**：调用 `DELETE /api/stores/:id`，已存在；需确认权限与二次确认弹窗。
