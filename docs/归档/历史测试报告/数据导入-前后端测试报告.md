# 数据导入（TikTok Excel）前后端测试报告

## 对标成品方案

| 对标对象 | 对标点 |
|----------|--------|
| **抖音电商罗盘 / TikTok Creator Center** | 直播数据导出为 Excel、再导入到看板；支持中英文表头、日期/观看/GMV/时长等指标 |
| **淘宝/京东商家后台** | 数据导入支持 Excel 上传、上传后看板数据更新、错误提示友好（不崩溃、有明确文案） |
| **行业共性** | 仅 TikTok/抖音店铺显示导入入口、选文件后开始导入、失败时 Toast 提示、成功后可刷新看板 |

---

## 后端确认

| 项 | 状态 |
|----|------|
| **接口** | `POST /api/data-import/tiktok`（multipart：file + storeId） |
| **stats.date** | INSERT 时必填：取导入数据最早日期或当天 `YYYY-MM-DD`，满足 NOT NULL |
| **错误处理** | catch 中使用 `error?.message`，避免 `undefined.message` 报错 |
| **返回** | 成功：`{ success, message, stats, importRecord }`；失败：`{ error, message }` |

## 对标结论

- **入口与权限**：仅 TikTok/抖音店铺显示「导入数据」，与抖音罗盘/TikTok Creator Center 仅直播数据可导入一致。
- **弹窗内容**：当前店铺、选择 Excel、格式说明（.xlsx/.xls、字段说明）、开始导入/取消，符合商家后台数据导入习惯。
- **校验**：未选文件时「开始导入」禁用，避免无效请求；前端另有「请选择Excel文件」「请先选择店铺」「该功能仅支持TikTok/抖音平台店铺」等 Toast。
- **后端**：stats.date 与 error?.message 已修复，上传流程可正常落库，错误时返回友好文案不崩溃。
