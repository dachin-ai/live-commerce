# 创建店铺弹窗 MCP 验证报告

通过 MCP 浏览器（cursor-browser-extension）对「创建店铺」弹窗的两项核心功能进行测试，**两项均通过，无需调试**。

## 测试环境

- 前端：http://localhost:5173/
- 后端：已启动（限流 500 次/15 分钟）
- 用户：Admin User（管理员）
- 弹窗入口：店铺管理页头部 **「创建店铺」按钮**、或商店列表「创建商店」、或无店铺时「请先创建店铺（点击创建）」/「创建新店铺」

## 1. 国家 → 货币联动

| 步骤 | 操作 | 预期 | 结果 |
|------|------|------|------|
| 1 | 打开创建店铺弹窗 | 国家默认「中国」，推荐货币为人民币 (¥ CNY) | ✅ |
| 2 | 点击国家选择器，选择「越南」 | 推荐货币、货币显示、价格区间单位更新为越南盾 (₫ VND) | ✅ |

**结论**：前端静态映射 `COUNTRY_CURRENCY` 与 `CreateStoreModal` 内 `useEffect` 联动正常，切换国家后推荐货币、货币与价格区间单位即时更新。

## 2. 产品分类（一级→二级→三级→勾选→提交）

| 步骤 | 操作 | 预期 | 结果 |
|------|------|------|------|
| 1 | 打开创建店铺弹窗 | 一级分类下拉有选项（非「加载失败」） | ✅ 显示：美妆护肤、服装配饰、食品饮料（重置 DB 后为抖店 11 个一级） |
| 2 | 选择一级「美妆护肤」 | 二级分类区域出现可多选项 | ✅ 显示：面部护理、彩妆 |
| 3 | 勾选二级「面部护理」 | 三级分类区域出现可多选项 | ✅ 显示：洁面、精华、面霜等 |
| 4 | 勾选三级「洁面」 | 已选分类以标签形式展示 | ✅ 标签「洁面」出现，可取消 |
| 5 | 提交店铺（带店铺名称） | 请求体含 `categoryIds`，店铺创建成功 | ✅ 分类与店铺一并保存 |

**结论**：产品分类前端已实现：一级下拉→二级多选→三级多选→勾选后纳入 `selectedCategories`，提交时 `categoryIds: selectedCategories` 随店铺保存；分类接口与联动正常。

## 网络与控制台

- 控制台：无 429、无接口报错；仅有 React Router 未来版本提示（可忽略）。
- 请求：`/api/categories`、`/api/regions/countries`、`/api/regions/currency` 等均正常返回。

## 维护建议

- 若后续出现「一级分类加载失败」或 429，请确认后端已重启且限流配置为 `rateLimitMiddleware(500, 15*60*1000)`。
- 若需更完整分类数据，可运行根目录「重置数据库.bat」后刷新页面。
