## 📐 数量限制

| 类型 | 条数 | 说明 |
|------|------|------|
| **LLM 主待办** | 6～10 条 | 智能生成的主待办（非节日、非自然因素）条数限制；提示词中要求 Coze 输出 6～10 条，后端合并去重后主待办最多保留 10 条。 |
| **节日 / 自然因素** | 不设上限 | 标题含节日、倒计时、大促、自然因素等内容的待办**单独计算、可超出主待办数量**，全部保留不截断。 |

合并与去重顺序：同节日合并 → 同标题/同主题去重 → 节日与自然因素全部保留 + 主待办按优先级与相关性保留 6～10 条 → 写入任务表。

## 📋 功能详细清单

### 1. LLM 智能待办 ✅

**功能**：调用 Coze bot API，基于店铺完整数据生成个性化待办

**输入数据**（符合 Coze 最佳实践）：
- ✅ 店铺基本信息（名称、平台、国家、类目）
- ✅ 核心销售指标（订单、观看、GMV、转化率、直播时长、阶段）
- ✅ 货币单位（泰铢、人民币等 7 种）
- ✅ 历史对比（含变化百分比）
- ✅ ⭐ 按日明细数据（最近 15 天，影响质量 80%）
- ✅ 业务上下文（阶段与重点、趋势、节日、气温）
- ✅ 已有待办（避免重复）
- ✅ 用户补充说明

**输出**：
- 6～10 条高质量待办（条数限制见下方「数量限制」）
- 引用具体数字（如"12月12日 GMV 最高"）
- 可执行动作
- 结合业务上下文

**来源标记**：`source: 'llm_intelligent'`

**响应时间**：10-30 秒

**文档**：[LLM 待办生成 - 给 Coze 的输入参数](./LLM待办生成-给Coze的输入参数.md)

---

### 2. 月同比待办 ✅（本次新增）

**功能**：按自然日对比上月同期数据，识别数据异常并生成待办

**对比规则**：
- **按自然日区间对比**（不按周）
- 如 2月1-10日 vs 1月1-10日（同样的月初10天）
- 自动处理跨年（1月 → 上年12月）
- 自动处理月份天数差异

**分析维度**（6个）：

| 维度 | 阈值 | 优先级 |
|-----|------|--------|
| **GMV** | 下降 > 10% | urgent |
| | 增长 > 15% | normal |
| **转化率** | 下降 > 0.5 个百分点 | urgent |
| | 提升 > 1.0 个百分点 | normal |
| **订单数** | 下降 > 15% | urgent |
| **观看数** | 下降 > 20% | urgent |
| | 增长 > 30% | normal |
| **时均GMV** | 下降 > 15% | urgent |
| **直播时长** | 减少 > 20% | normal |

**触发条件**：
- 店铺有至少 2 个月的运营数据
- 数据变化超过阈值

**来源标记**：`source: 'mom_comparison'`

**文档**：[月同比待办生成规则说明](./月同比待办生成规则说明.md)

---

### 3. 年同比待办 ✅

**功能**：对比去年同期数据，识别长期趋势

**对比规则**：
- 如 2025-12-01 ~ 2025-12-30 vs 2024-12-01 ~ 2024-12-30

**分析维度**（3个）：
- GMV 同比（阈值：下降 > 15%，增长 > 20%）
- 订单数同比（阈值：下降 > 10%）
- 观看数同比（阈值：下降 > 20%）

**触发条件**：
- 店铺有去年同期数据

**来源标记**：`source: 'yoy_comparison'`

---

### 4. 阶段待办 ✅

**功能**：基于店铺成长阶段生成通用待办

**阶段分类**：
- **冷启动期**：GMV < 1万 或 场次 < 10
  - 待办：完成首月10场直播、获取首批评价等
- **成长期**：GMV 1万-10万
  - 待办：提升转化率、优化客单价等
- **成熟期**：GMV > 10万
  - 待办：品牌建设、私域运营、供应链优化等

**触发条件**：
- LLM 失败或未配置时兜底
- 或作为补充待办

**来源标记**：`source: 'stage'`

## 🔄 待办生成流程

```
用户点击「智能生成」
    ↓
获取店铺数据（最近 30 天汇总 + 最近 15 天明细）
    ↓
┌─────────────────────────────────────┐
│  尝试 LLM 智能生成（Coze bot）       │
│  - 构建完整入参（符合 Coze 最佳实践）│
│  - 调用 Coze API (超时 40s)         │
│  - 解析 JSON 返回                   │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  追加月同比/年同比待办               │
│  - 获取上月同期数据                 │
│  - 获取去年同期数据                 │
│  - 计算变化百分比                   │
│  - 根据阈值生成待办（0-6条）         │
└─────────────────────────────────────┘
    ↓
┌─────────────────────────────────────┐
│  兜底：阶段待办                     │
│  - 若 LLM 返回 0 条                 │
│  - 生成基于阶段的通用待办            │
└─────────────────────────────────────┘
    ↓
去重 + 排序 + 节日合并
    ↓
返回前端展示（通常 5-10 条）
```
