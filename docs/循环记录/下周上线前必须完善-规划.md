# 下周系统上线前必须完善 — 规划

若**下周系统上线**，以下为按优先级整理的「必须/建议马上完善」项，便于排期与验收。依据：《系统问题-规划清单》《预留接口与上线说明》《角色与权限矩阵》《各模块步骤总结》、第 3 轮评估与分工。

## 一、上线前必须完成（P0）

### 1. 角色与权限验收（对应模块 8～10）

当前侧栏与路由已按 `admin` 显隐「用户管理」「工作流」，后端关键接口已加 `requireAdmin`。**上线前必须**：

| 项 | 动作 | 验收标准 |
|----|------|----------|
| 运营账号 | 用**运营**账号登录，走一遍看板、分析、执行工具 | 侧栏**无**「用户管理」「工作流」；直接访问 `/admin`、`/workflow` 被**重定向或 403** |
| 管理账号 | 用**管理员**账号登录，走一遍同上 + 用户管理 + 工作流 | 侧栏**有**「用户管理」「工作流」；可正常进入 `/admin`、`/workflow` 并操作 |
| 接口权限 | 用运营账号调用「仅管理员」接口（如 POST 话术 LLM 配置、POST 版本日志、用户管理 CRUD） | 返回 **403**，且前端有明确提示（不白屏、不暴露堆栈） |

**说明**：角色与权限矩阵已落页（`docs/角色与权限矩阵.md`），后端显隐与鉴权已存在；本周重点是**用真实账号走查 + 补漏**。

---

### 2. 认证与找回密码跳转（预留接口说明 1）

| 位置 | 预留内容 | 上线前动作 |
|------|----------|------------|
| `ForgotPassword.tsx` | 成功后 `navigate` | 若尚未实现：成功后**自动跳转登录页**（可带「3 秒后跳转」提示） |
| `ResetPassword.tsx` | 同上 | 同上 |

避免用户找回/重置密码后停留在成功页不知下一步。

---

### 3. 预留占位入口不踩空（预留接口说明 3～5）

| 位置 | 当前状态 | 上线前建议 |
|------|----------|------------|
| 侧栏「消息」 | 禁用占位 | 保持**禁用**或改为「即将上线」文案，**不要**可点击进空白页 |
| AdminPanel：设置/权限 Tab | 灰色占位 | 保持占位或隐藏 Tab，**不要**点进去报错或白屏 |
| 执行工具「直播中实时助手」 | 不可用占位 | 保持**不可用**或「即将上线」，**不要**可点击进未实现功能 |

原则：所有对用户可见的入口，要么可用，要么明确不可用/即将上线，不出现「点了没反应」或「白屏/报错」。

---

### 4. 关键错误与超时（用户体验）

第 3 轮已补：创建店铺失败 Toast、话术/报告 60s 超时、流式话术 90s + AbortController、智能待办 60s 超时。**上线前建议**：

- 再走一遍：**创建店铺失败**、**智能生成待办超时**、**话术生成超时**，确认均有**明确 Toast**，且不暴露后端堆栈或 500 原文。
- 重要操作（如删除店铺、删除用户）：已补**二次确认**（含「删除后无法恢复」）及**成功/失败 Toast**；失败时展示后端返回错误文案。

## 三、可上线后迭代（P2）

以下可不阻塞下周上线，上线后再做即可：

- **类型-前后端**：Task/Store/LiveStats 共享契约（当前前后端各自定义可用）
- **单文件过大**：ai-refactored.ts 继续拆分为 ai/tasks、ai/report 等（功能不受影响）
- **后端-限流/缓存**：多实例/Redis（单机上线可缓做）
- **后端-错误格式**：统一 `{ ok, data?, error?, code? }`（当前接口可用）
- **前端-超时**：按 path 细分配置（当前 60s/90s 已覆盖主流程）
- **模块 11～13**：导航与路由保护、错误处理、数值范围保护（可在上线后按需加）
- **消息、设置/权限、直播中助手**：占位保留，功能上线后再接
