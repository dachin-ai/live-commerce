# 产品分类更新说明

## 问题：前端没有显示新的 18 个一级类目

**原因**：数据库种子数据只在**数据库为空**时才会插入。如果你的数据库中已有旧的分类数据（如之前的 11 个一级类目），新增的 7 个一级类目（宠物生活、母婴、医药保健、虚拟商品、本地生活、农资绿植、玩乐潮玩）不会自动插入。

## 解决方案

### 方案 1：重置数据库（推荐，适用于开发/测试环境）

1. 运行根目录 **「重置数据库.bat」**
2. 确认删除数据库
3. 重启后端服务（运行「单独启动后端.bat」）
4. 后端启动时会自动插入新的 18 个一级类目及对应的二、三级
5. 刷新前端页面，创建店铺弹窗中即可看到新的分类

**注意**：重置数据库会删除所有现有数据（店铺、任务、用户等），仅适用于开发/测试环境。

### 方案 2：手动 SQL 更新（适用于生产环境，需保留现有数据）

如果不想重置数据库，可以手动执行 SQL 插入新的分类。SQL 脚本见 `backend/src/db.ts` 中的 `level1Categories`、`level2Categories`、`level3Categories` 数组。

**步骤**：
1. 使用 SQLite 工具（如 DB Browser for SQLite）打开 `backend/data.db`
2. 执行以下 SQL（示例，完整 SQL 见 `db.ts`）：

```sql
-- 插入新的一级类目（仅当不存在时）
INSERT OR IGNORE INTO categories (id, name, nameTh, level, parentId, sortOrder) VALUES
('cat-1-12', '宠物生活', 'สัตว์เลี้ยง', 1, NULL, 12),
('cat-1-13', '母婴', 'แม่และเด็ก', 1, NULL, 13),
('cat-1-14', '医药保健', 'ยาและสุขภาพ', 1, NULL, 14),
('cat-1-15', '虚拟商品', 'สินค้าเสมือน', 1, NULL, 15),
('cat-1-16', '本地生活', 'ชีวิตประจำวัน', 1, NULL, 16),
('cat-1-17', '农资绿植', 'เกษตรและไม้ประดับ', 1, NULL, 17),
('cat-1-18', '玩乐潮玩', 'ของเล่นและคอลเลกชัน', 1, NULL, 18);

-- 然后插入对应的二级、三级（见 db.ts 完整列表）
```

3. 重启后端服务
4. 刷新前端页面

### 方案 3：检查前端缓存

前端代码已配置 `staleTime: 0` 和 `refetchOnMount: true`，理论上不会缓存。如果仍看不到新分类：

1. **硬刷新浏览器**：Ctrl+Shift+R（Windows）或 Cmd+Shift+R（Mac）
2. **清除浏览器缓存**：开发者工具 → Application → Clear storage
3. **检查网络请求**：开发者工具 → Network，查看 `/api/categories?level=1` 返回的数据是否包含新的 18 个一级类目

## 验证新分类是否生效

1. 打开创建店铺弹窗
2. 查看「一级分类」下拉框
3. 应该看到 **18 个一级类目**：
   - 服饰鞋包、美妆个护、食品健康、亲子生活、家居家电、3C数码、运动户外、汽车、鲜花园艺、珠宝文玩、文化娱乐
