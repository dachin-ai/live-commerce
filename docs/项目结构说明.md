# 项目结构说明

## 根目录

```
lvbcsym/
├── frontend/           # React 前端应用
├── backend/            # Express 后端 API
├── docs/               # 项目文档（说明、测试、工作流等）
├── data/               # 共享数据目录（如 shared_state）
├── outputs/            # 工作流输出（迭代产物、部署包等）
├── index.html          # 统一测试入口（本地打开）
├── test-terminal.html  # API 测试终端（本地打开）
├── README.md           # 项目说明
├── 下一任智能体说明.md  # 交接说明
├── package.json        # 根 package（workspace 或脚本）
└── *.bat               # Windows 启动/修复/检查脚本
```

## 前端 (frontend/)

- **技术栈**：React 18、TypeScript、Vite、Tailwind CSS、Recharts、React Query
- **入口**：`index.html` → `src/main.tsx` → `App.tsx`

```
frontend/src/
├── components/     # 通用与业务组件
│   ├── AIFeatures.tsx
│   ├── Chart.tsx
│   ├── CountrySelector.tsx
│   ├── CreateStoreModal.tsx
│   ├── ErrorBoundary.tsx
│   ├── LayoutSettings.tsx
│   ├── LazyLoad.tsx
│   ├── PerformanceOptimizedList.tsx
│   ├── ProtectedRoute.tsx
│   ├── Sidebar.tsx
│   ├── StatCard.tsx
│   ├── StoreList.tsx
│   ├── StoreSelector.tsx
│   ├── TaskList.tsx
│   └── VersionLogs.tsx
├── contexts/      # React Context（布局、店铺、Toast）
├── hooks/         # 自定义 Hooks（防抖、交叉观察、布局偏好）
├── pages/         # 页面级组件（路由对应）
├── services/      # API 封装（与后端路由一一对应）
├── App.tsx
├── main.tsx
└── index.css
```

**前端分层**（按业务逻辑）：
- **定义层**：看板、分析、工具等入口与布局
- **分析层**：数据分析、图表、筛选
- **执行工具层**：任务、店铺、素材、工作流等操作

## 后端 (backend/)

- **技术栈**：Node.js、Express、TypeScript、SQLite3、JWT
- **入口**：`src/index.ts`，挂载各 API 路由与中间件

```
backend/src/
├── routes/           # API 路由（按资源划分）
│   ├── auth.ts       # 注册、登录、找回/重置密码、个人资料、改邮箱/改密码
│   ├── stats.ts      # 实时统计
│   ├── tasks.ts      # 任务 CRUD
│   ├── stores.ts     # 店铺 CRUD
│   ├── ai-refactored.ts # AI 入口：挂载 /script（话术）、/generate-tasks（智能任务）；话术逻辑在 ai/script.ts
│   ├── ai/
│   │   └── script.ts    # 话术 LLM 配置、话术生成（同步/流式）
│   ├── materials.ts  # 素材 CRUD、上传
│   ├── users.ts      # 用户管理（管理员）
│   ├── categories.ts # 分类树
│   ├── regions.ts    # 国家/区域/货币
│   ├── preferences.ts# 用户偏好
│   ├── version-logs.ts # 版本日志
│   └── workflow.ts   # 工作流触发、轮次、输出
├── middleware/       # 认证、缓存、限流
├── workflow_engine/ # 工作流引擎（角色、调度、上下文、输出）
├── types/            # 共享类型
├── utils/            # 工具（如性能）
├── db.ts             # 数据库初始化与访问
└── index.ts          # 应用入口
```

**API 前缀**：除健康检查 `/health` 与静态 `/uploads` 外，接口均为 `/api/<资源>`，例如 `/api/auth`、`/api/stores`、`/api/tasks`。

## 文档 (docs/)

- **README.md**：文档索引与分类
- **项目结构说明.md**：本文件
- 其余为功能说明、模块调试、测试报告、工作流、问题排查等，详见 `docs/README.md`。

## 脚本与入口

| 文件 | 用途 |
|------|------|
| 启动项目.bat | 检查环境、安装依赖、启动前后端 |
| 快速启动.bat | 快速启动服务 |
| 单独启动后端.bat | 仅启动后端 |
| 打开测试入口.bat | 打开根目录 index.html（统一测试入口） |
| 打开测试终端.bat | 打开 test-terminal.html（API 测试） |
| 修复npm配置并安装依赖.bat | 修复依赖问题 |
| 修复后端启动.bat | 诊断后端启动问题 |
| 检查状态.bat | 检查项目与依赖状态 |
| 重置数据库.bat | 重置 SQLite 数据库 |

前端开发服务器默认：http://localhost:5173  
后端 API 默认：http://localhost:3000
