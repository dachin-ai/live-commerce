# 项目结构说明

## 根目录

```
lvbcsym/
├── frontend/           # React 前端应用
├── backend/            # Express 后端 API
├── docs/               # 项目文档（说明、测试、工作流等）
├── data/               # 共享数据目录（如 shared_state）
├── outputs/            # 工作流输出（迭代产物、部署包等）
├── pgsql/              # PostgreSQL 18 二进制（.gitignore 已排除）
├── pgdata/             # PostgreSQL 数据目录（.gitignore 已排除）
├── skills/             # AI Agent Skills（洁癖等）
├── index.html          # 统一测试入口（本地打开）
├── test-terminal.html  # API 测试终端（本地打开）
├── README.md           # 项目说明
├── package.json        # 根 package（workspace 或脚本）
└── *.bat               # Windows 启动/修复/检查脚本
```

## 前端 (frontend/)

- **技术栈**：React 18、TypeScript、Vite、Tailwind CSS、Recharts、React Query
- **入口**：`index.html` → `src/main.tsx` → `App.tsx`

frontend/src/

│   ├── ErrorBoundary.tsx

**前端分层**（按业务逻辑）：
- **定义层**：看板、分析、工具等入口与布局
- **分析层**：数据分析、图表、筛选
- **执行工具层**：任务、店铺、素材、工作流等操作

## 后端 (backend/)

- **技术栈**：Node.js、Express、TypeScript、PostgreSQL 18（pg 驱动）、JWT
- **入口**：`src/index.ts`，挂载各 API 路由与中间件

```
backend/src/
├── routes/              # API 路由（按资源划分）
│   ├── auth.ts          # 注册、登录、找回/重置密码、个人资料
│   ├── stats.ts         # 实时统计
│   ├── tasks.ts         # 任务 CRUD
│   ├── stores.ts        # 店铺 CRUD
│   ├── ai-refactored.ts # AI 入口：挂载 /script、/generate-tasks
│   ├── ai/
│   │   ├── script.ts    # 话术 LLM 配置、话术生成
│   │   └── systemAgent.ts
│   ├── materials.ts     # 素材 CRUD、上传
│   ├── users.ts         # 用户管理（管理员）
│   ├── categories.ts    # 分类树
│   ├── regions.ts       # 国家/区域/货币
│   ├── config.ts        # 系统配置
│   ├── translate.ts     # 翻译服务
│   ├── feedback.ts      # 用户反馈
│   ├── messages.ts      # 站内信
│   ├── videos.ts        # 视频管理
│   ├── dataImport.ts    # 数据导入
│   ├── tt-import.ts     # TikTok 数据导入
│   ├── tt-analytics.ts  # TikTok 数据分析
│   ├── tt-bi.ts         # TikTok BI 报表
│   ├── preferences.ts   # 用户偏好
│   ├── version-logs.ts  # 版本日志
│   └── workflow.ts      # 工作流触发、轮次、输出
├── middleware/          # 认证(auth)、缓存(cache)、限流(rateLimit)、错误处理
├── services/            # 业务逻辑层（scriptLLM、TtImport 等）
├── repositories/        # 数据访问层（TtImportRepository 等）
├── database/            # 数据库连接与配置（PostgreSQL）
├── rules/               # 规则引擎（scriptGeneration、scriptResearch 等）
├── config/              # 应用配置
├── constants/           # 常量定义
├── workflow_engine/     # 工作流引擎（角色、调度、上下文、输出）
├── types/               # 共享类型
├── utils/               # 工具函数
├── db.ts                # 数据库初始化与种子数据（PostgreSQL）
├── db-migrate.ts        # 数据库迁移脚本
└── index.ts             # 应用入口
```

**API 前缀**：除健康检查 `/health` 与静态 `/uploads` 外，接口均为 `/api/<资源>`，例如 `/api/auth`、`/api/stores`、`/api/tasks`。

## 脚本与入口

| 文件 | 用途 |
|------|------|
| 快速启动.bat | 快速启动前后端服务（推荐日常使用） |
| start.bat | 启动脚本（英文入口） |
| 启动.ps1 / start.ps1 | PowerShell 菜单式启动（功能完整） |
| 修复npm配置并安装依赖.bat | 修复依赖问题 |
| 打开测试入口.bat | 打开根目录 index.html（统一测试入口） |
| 打开测试终端.bat | 打开 test-terminal.html（API 测试） |
| 检查状态.bat | 检查项目与依赖状态 |
| 重置数据库.bat | 重置数据库（迁移 PG 后需用 psql 命令，见《启动与数据库操作指南》） |

前端开发服务器默认：http://localhost:5173  
后端 API 默认：http://localhost:3000
