# 部署更新 - 复制用命令

> **注意**：`/var/www/lvbcsym` 是**服务器上的路径**，只能在**火山云 ECS 终端（浏览器里）**执行。  
> 不要在本机 Windows 的 PowerShell 里执行下面「二、服务器」里的命令，否则会报错「Cannot find path」。

## 一、本机（Windows，推送最新代码）

**上传前**：先保存所有改过的文件（Ctrl+S），再在项目根目录 `d:\Work space\lvbcsym` 打开 PowerShell，执行：

```powershell
cd "D:\Work space\lvbcsym"
git add .
git status
git commit -m "更新到最新版本"
git push origin main
```

- 用 `git status` 确认有「待提交」或「未跟踪」文件，再执行 commit 和 push，避免漏推。
- 务必看到 `git push origin main` 成功（如 `main -> main` 或 `Everything up-to-date`），服务器 pull 才能拿到最新代码。
- 若报错 `Recv failure: Connection was reset`，为网络问题，可稍后重试 `git push origin main`。

---

## 二、服务器 ECS 终端（拉代码并更新网站）

**必须在浏览器里操作**：打开火山云控制台 → 实例 → 远程连接 → **ECS 终端** → 立即登录。  
在**浏览器里出现的黑色终端**中，整段复制下面命令粘贴执行（不要在本机 PowerShell 里执行）：

```bash
cd /var/www/lvbcsym
git pull origin main
git log -1 --oneline
npm run install:all
npm run build
pm2 restart lvbcsym-api
```

执行完成后，在浏览器打开你的域名，按 **Ctrl+Shift+R** 或 **Ctrl+F5** 强刷即可看到最新版本。可用 `git log -1 --oneline` 核对是否与本机最新提交一致。

---

## 本机已实现、部署到网页后没有（排查清单）

若功能在本机正常，部署到网页后不生效（例如虚拟管理员仍 403、权限配置不生效等），按下面逐项核对：

1. **本机代码已推送到远程**  
   在本机执行 `git push origin main` 且**无报错**（看到 `main -> main` 或 `Everything up-to-date`）。若 push 失败，服务器 pull 拿不到最新代码。

2. **服务器拉的是最新提交**  
   在服务器执行完 `git pull origin main` 后，执行 `git log -1 --oneline`，与本机同目录下 `git log -1 --oneline` 对比，**提交号一致**才说明代码一致。

3. **服务器做了完整构建与重启**  
   必须依次执行：`npm run install:all` → `npm run build` → `pm2 restart lvbcsym-api`。只 pull 不 build、或只 build 不 restart，网页仍是旧版本。

4. **浏览器强刷**  
   打开域名后按 **Ctrl+Shift+R** 或 **Ctrl+F5** 强刷，避免使用缓存的旧 JS。

5. **权限配置在服务器数据库**  
   「可使用 LLM 的用户」「能够使用的功能」等存在**服务器上的数据库**里。若服务器是全新环境或重装过，需要在**网页上**用管理员账号登录 → 权限配置 → 重新勾选用户与功能并**保存配置**，本机配置不会自动同步到服务器。

6. **前端静态路径与 Nginx root 是否一致**（版本号正确但界面未更新时重点排查）  
   - `npm run build` 产出在 `frontend/dist/`，Nginx 的 `root` 必须指向该目录或包含 `index.html` 的实际目录。  
   - 在服务器执行：`ls -la /var/www/lvbcsym/frontend/` 和 `ls -la /var/www/lvbcsym/frontend/dist/`，确认 `index.html` 所在位置。  
   - 若 Nginx 配置为 `root /var/www/lvbcsym/frontend;`，但 `index.html` 在 `frontend/dist/` 下，则需改为 `root /var/www/lvbcsym/frontend/dist;`，或增加拷贝步骤：
     ```bash
     rm -rf /var/www/lvbcsym/frontend-public && mkdir -p /var/www/lvbcsym/frontend-public && cp -r /var/www/lvbcsym/frontend/dist/* /var/www/lvbcsym/frontend-public/
     ```
     并把 Nginx `root` 指向 `frontend-public`。  
   - 修改 Nginx 后执行：`sudo nginx -t && sudo systemctl reload nginx`。

6.4. **视频分析报「Vision 模型无法访问本地地址」**  
   视频分析需把视频 URL 发给 Coze/Vision 模型，模型必须能公网访问该 URL。若未配置公网地址，会使用 localhost，导致失败。  
   在服务器 `backend/.env` 中增加（按实际域名修改）：
   ```
   API_BASE_URL=https://atlvbcsy.com
   ```
   或 `PUBLIC_URL=https://atlvbcsy.com`。保存后执行 `pm2 restart lvbcsym-api`。

6.5. **视频上传 ERR_CONNECTION_RESET（Nginx 大文件与超时）**  
   视频上传失败、报 `net::ERR_CONNECTION_RESET` 时，需在 Nginx 中放宽上传大小与超时。  
   在服务器执行 `grep -rn "proxy_pass\|location /api" /etc/nginx/` 找到 API 代理所在文件与行号，然后编辑该文件：

   - **在 `location /api/ {` 的下一行**（即 `{` 后面第一行）插入下面 4 行：
     ```nginx
     client_max_body_size 500M;
     proxy_connect_timeout 300;
     proxy_send_timeout 300;
     proxy_read_timeout 300;
     ```
   - 若该文件没有 `location /api/`，则在 `server {` 块内、`location /` 之前新增：
     ```nginx
     location /api/ {
         client_max_body_size 500M;
         proxy_connect_timeout 300;
         proxy_send_timeout 300;
         proxy_read_timeout 300;
         proxy_pass http://127.0.0.1:3000;
         proxy_http_version 1.1;
         proxy_set_header Host $host;
         proxy_set_header X-Real-IP $remote_addr;
         proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
         proxy_set_header X-Forwarded-Proto $scheme;
     }
     ```
   - 修改后执行：`sudo nginx -t && sudo nginx -s reload`。

7. **浏览器强缓存**  
   按 **Ctrl+Shift+R** 或 **Ctrl+F5** 无效时，可尝试：打开开发者工具(F12) → Network 标签 → 勾选 "Disable cache" → 刷新；或使用无痕/隐私模式访问。

## 若项目不在 /var/www/lvbcsym

先确认实际路径，例如：

```bash
ls /var/www/
```

或

```bash
find /var -name "package.json" -path "*lvbcsym*" 2>/dev/null | head -5
```

把上面第二段里的 `cd /var/www/lvbcsym` 换成你的实际路径即可。
