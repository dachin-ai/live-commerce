## 三、用户反馈的「未被标记」场景分析

### 3.1 话术考核

| 项目 | 内容 |
|------|------|
| 场景 | 考核主播话术水平、评估话术效果 |
| 当前识别 | 含「话术」会打为 `script`，但「考核」易被忽略 |
| 建议 | 在 `script` 关键词中增加：**考核、评估、打分** |
| 推荐工具 | speech（话术生成）、screen-recording（视频分析） |

### 3.2 图片分析：主图

| 项目 | 内容 |
|------|------|
| 场景 | 商品主图分析、主图优化、主图点击率 |
| 当前识别 | **无** 图片/主图相关关键词 |
| 建议 | 新增 aiFeature `image_analysis`，关键词：**主图、图片分析、主图优化、商品图** |
| 推荐工具 | 待有对应执行工具后映射；或暂时归入 `product_recommend` |

### 3.3 直播场景打分

| 项目 | 内容 |
|------|------|
| 场景 | 直播场景、直播间布置、场景布置评估 |
| 当前识别 | 「直播」可能被 both 岗位匹配，但无对应 aiFeature |
| 建议 | 新增 aiFeature `scene_scoring` 或归入 `screen-recording`，关键词：**直播场景、场景打分、场景布置、直播间布置** |
| 推荐工具 | screen-recording（视频分析：直播录屏&短视频） |

### 3.4 产品竞争力分析

| 项目 | 内容 |
|------|------|
| 场景 | 竞品分析、产品竞争力、竞争格局 |
| 当前识别 | 无「竞争力、竞品」关键词；`product_recommend` 未覆盖 |
| 建议 | 在 `product_recommend` 中增加：**竞争力、竞品、竞争分析**；或新增 `product_competitiveness` 映射到 market-analysis |
| 推荐工具 | market-analysis, recommendations |

## 四、建议扩展方案

### 4.1 方案 A：扩展现有关键词（最小改动）

在 `autoTagTaskRoleAndTool` 中扩充正则：

- **script**：增加 `考核|评估|打分`
- **product_recommend**：增加 `竞争力|竞品|竞争分析`
- **content** 或 **stats**：增加 `主图|图片分析`（临时，待有专门工具再拆分）

### 4.2 方案 B：新增 aiFeature 类型（推荐）

| 新增 aiFeature | 关键词 | 显示名 | 映射工具 |
|----------------|--------|--------|----------|
| `script_assessment` | 话术考核、话术评估、话术打分 | 话术考核 | speech, screen-recording |
| `image_analysis` | 主图、图片分析、主图优化、商品图 | 图片分析 | 待接入 |
| `scene_scoring` | 直播场景、场景打分、场景布置 | 直播场景打分 | screen-recording |
| `product_competitiveness` | 竞争力、竞品、竞争分析 | 产品竞争力分析 | market-analysis, recommendations |

### 4.3 需修改的文件

1. **后端** `backend/src/routes/ai-refactored.ts`：`autoTagTaskRoleAndTool` 增加新关键词或新分支
2. **前端** `frontend/src/components/TaskList.tsx`：
   - `AI_FEATURE_TO_TOOLS` 增加新 aiFeature 映射
   - aiFeature 标签渲染分支增加新类型
3. **多语言** `frontend/src/locales/*.json`：增加 `tasks.aiFeatureXxx` 文案
