# Coze 待办生成 - 测试消息

在 Coze 里用下面两段内容测试 Bot，确认其**只回复 JSON、不输出分析/mermaid**。

---

## 方式一：一条完整消息（推荐）

在 Coze 的「预览/对话」里，**把下面整段复制成一条用户消息**发送：

```
你是直播电商待办助手。只输出 JSON，不要任何解释或 markdown。

【要求】
1. 每条任务必须针对**不同维度**，例如：时段/加播、互动/福袋、选品/上新、引流/短视频、节日备货、时长、转化、客单价、观看人数等。标题和描述不可重复或雷同。
2. 标题≤30字，场景化具体（如「今晚7点加播2小时」「每10分钟福袋拉互动」「周末上新3款高毛利品」），禁止泛泛的「提升X至Y%」「增加时长至X小时」类模板句。
3. 描述≤120字，含 1～2 个具体动作与可量化预期。
4. 输出格式（仅此 JSON）：{"tasks":[{"title":"","description":"","priority":"urgent或normal"}]}，条数按需，无上限。

【用户请求】

【店铺】greenpet（抖音）| 区域：CN | 类目：宠物用品、宠物食品

【本周数据】GMV 12500 | 时长 5.8h | 观看 3200 | 订单 72 | 转化率 2.2% | 时均GMV 2155

【历史对比】平均GMV 11800 | 平均转化 2.5%

【时间与节日】2月 | 即将节日：春节15天后、元宵节17天后

【阶段与重点】冷启动期：拉新、互动 | 趋势：declining（近3期GMV连续下降）

请根据以上**不同维度**生成若干条差异化待办 JSON，每条对应不同优化方向，条数按需。
```

**预期回复**：Bot 应**直接输出**一段 JSON，且**仅此 JSON**，例如：

```json
{"tasks":[{"title":"今晚7点加播2小时冲销量","description":"在晚间黄金时段加播，重点推高毛利品，预计可提升当日GMV约20%。","priority":"urgent"},{"title":"每10分钟福袋拉互动","description":"整场每10分钟抽1次福袋，引导点赞评论，提升互动率至25%以上。","priority":"normal"}]}
```

若回复里出现「### 泰国电商」「mermaid」、分析报告等，说明 Bot 未按指令执行，需在 Coze 人设/系统提示里强调「仅输出上述 JSON」。

---

## 方式二：拆成系统提示 + 用户消息（与后端一致）

若 Coze 支持分别填「系统提示」和「用户消息」，可这样填：

**系统提示（人设）：**

```
你是直播电商待办助手。只输出 JSON，不要任何解释或 markdown。

**用户消息：**

```

---

## 说明

- 后端调用 Coze 时，会把「系统提示」和「用户请求」拼成一条 `message` 发送，格式即：`系统提示 + "\n\n【用户请求】\n" + 用户消息`。
- 用方式一在 Coze 里测，等于模拟了后端发过去的那一条完整消息；若这里能稳定只回 JSON，接口 `stream_run` 一般也会只回 JSON。
