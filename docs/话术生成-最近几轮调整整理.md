# 话术生成 / Coze Agent 最近几轮调整整理

本文档整理近期围绕「Cursor 接入 Agent API」与「Coze SSE 流式输出」所做的多轮调整，便于回溯与排查。

---

## 一、后端：请求体与 SSE 解析

### 1. 请求体按 URL/环境变量分支（coze.site 用旧版，避免空响应）

- **当前逻辑**：`buildCozeStreamRunBody(message, options, requestUrl)` 当 `requestUrl` 含 `coze.site` 或 `COZE_LEGACY_BODY=1` 时使用**旧版** body（`content.query.prompt` + `type: 'query'` + `project_id`）；否则使用 Agent 格式（`content` + `session_id`）。发布站点 `*.coze.site` 返回空 body 时需用旧版。
- **文件**：`backend/src/services/scriptLLM.ts`。

### 2. Answer 解析对齐 Coze SSE 说明

- **变更**：`type === 'answer'` 时，**优先**从 `data.content.answer` 取正文（与 Coze 官方 SSE 说明一致），兼容顶层 `data.answer`。
- **说明**：Coze 流式下每个 answer 事件可能只带 1～2 个字符，由后端按顺序拼接后转发给前端。
- **文件**：`scriptLLM.ts` → `extractCozeContent()`。

### 3. 不把 tool_request 当正文

- **变更**：`isCozeToolCallEvent()` 增加对 `tool_request` 的判断；与 `tool_response` 一样，不产出正文。
- **文件**：`scriptLLM.ts` → `isCozeToolCallEvent()`。

### 4. tool_response 从 result 解析话术

- **变更**：按 Coze 规范，`content.tool_response.result` 为 **JSON 字符串**。`extractScriptFromToolResponse()` 改为：取 `content.tool_response` → 取 `result` 字符串 → `JSON.parse(result)` → 从解析结果中取 `script` / `content` / `data.content` 等作为话术；若解析失败则 fallback 为原 result 字符串。
- **输出**：提取到话术文本后，在流式循环中按块（如 80 字符）yield，实现模拟流式。
- **文件**：`scriptLLM.ts` → `extractScriptFromToolResponse()` 与 `streamCozeAgent` 循环。

---

## 二、前端：占位与模拟流式

### 1. 取消「连接中」环节

- **变更**：删除 `streamPhase` 状态（原 `idle` → `connecting` → `streaming`）。流式时不再单独显示「连接中…」。
- **当前行为**：流式且无内容时只显示「正在生成话术，请稍候…」（`tools.streamPlaceholder`）；一旦有内容即显示流式正文 + 闪烁光标。
- **文件**：`frontend/src/components/AIFeatures.tsx`。

### 2. 占位与无障碍

- **保留**：`<pre>` 的 `min-h-[8rem]`、`role="status"`、`aria-live="polite"`；有内容时文末闪烁光标。
- **多语言**：`tools.streamPlaceholder` 在 zh-CN / en-US / th-TH 中已配置（`streamConnecting` 已不再使用，可保留键名以备后用）。

### 3. 模拟流式（onDone 时补打字机效果）

- **逻辑**：`onDone(script)` 时，若已流式接收长度 &lt; `script.content` 的 20%，则用定时器将 `script.content` 分段追加到展示区，再写入最终结果并清空流式状态；用 `simulatingStreamRef` 避免 `finally` 中过早清除 loading。
- **文件**：`AIFeatures.tsx`（streamedLengthRef、simulatingStreamRef、onDone 内分支）。
