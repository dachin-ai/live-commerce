# 待办生成与执行工具 - LLM 接入与配置说明

本文说明**待办生成**和**执行工具**中哪些功能需要接入 LLM、当前是否已支持配置、以及如何完成配置。

---

## 一、需要接入 LLM 的功能与当前状态

| 功能 | 是否使用 LLM | 配置是否可完成 | 说明 |
|------|--------------|----------------|------|
| **话术生成**（创建脚本 / 话术生成） | ✅ 是 | ✅ **已支持** | 唯一已接入 LLM 的功能；管理员在「管理员」-「话术 LLM 配置」中填写 API 地址与密钥即可完成配置，详见 [话术生成 LLM 配置说明](./话术生成LLM配置说明.md)。 |
| **待办生成**（智能生成任务） | ✅ **LLM 为主、规则为辅** | ✅ **复用话术 LLM 配置** | 已配置话术 LLM 且有当前数据时：**优先调用一次**多维度智能待办（店铺、时间、历史、阶段、节日、趋势、异常摘要），生成 3～6 条待办（来源 `llm_intelligent`），规则仅做必要补充（节日 1 条、冷启动短视频 1 条）；若 LLM 未配置或调用失败，则走规则引擎全量兜底。无需单独配置，与话术生成共用「话术 LLM 配置」。 |
| **生成报告** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/report` 为占位，后端未实现；若未来用 LLM 实现，可复用话术 LLM 或单独配置。 |
| **市场分析** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/analyze-market` 为占位。 |
| **商品推荐** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/recommendations` 为占位。 |
| **店铺对比** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/compare-stores` 为占位。 |
| **数据统计** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/stats` 为占位。 |
| **市场调研** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/market-research` 为占位。 |
| **店铺时效对比** | ❌ 后端未实现 | ➖ | 前端调用 `/ai/compare-efficiency` 为占位。 |

结论：**当前需要且能够完成 LLM 配置的为「话术生成」与「待办生成」**，二者复用同一套「话术 LLM 配置」与「LLM 调用方式」（Coze Agent / OpenAI）；待办未配置或 LLM 失败时走规则兜底。其余执行工具见上表（多数后端未实现）。

---

## 二、如何完成话术生成 LLM 配置（唯一需配置项）

1. 使用**管理员账号**登录。
2. 进入 **管理员** 页面（侧栏「管理员功能」- 或直接访问 `/admin`）。
3. 在 **「话术 LLM 配置」** 卡片中：
   - **API 地址**：默认已填 Coze 智能体地址，可改为豆包、OpenAI 等；
   - **API 密钥**：必填（Coze 用 API Token，豆包/OpenAI 用控制台 API Key）；
   - **模型 / 接入点 ID**（可选）：Coze 不需要；豆包填 `ep-` 接入点 ID；OpenAI 可留空；
   - **可使用话术生成的用户**：勾选允许使用话术生成的用户（不勾选则按「全选」逻辑，见后台说明）。
4. 点击 **「保存配置（选定用户可用）」**。
5. 配置立即生效，无需重启后端；选定用户可在「执行工具」-「创建脚本」或「话术生成」中使用流式话术生成。

若希望通过环境变量配置（部署人员），见 [话术生成 LLM 配置说明 - 方式二](./话术生成LLM配置说明.md#方式二环境变量配置部署人员)。

## 三、后端与前端对应关系（便于排查）

- **话术 LLM 配置**
  - 后端：`GET /api/ai/script/config`（查询是否已配置、管理员可见 allowedUserIds）、`POST /api/ai/script/config`（仅管理员可保存）。
  - 前端：`AdminPanel` 中「话术 LLM 配置」表单；`AIFeatures` 中话术生成前会调 `getScriptLLMConfig()`，未配置或无权限时展示提示。
- **待办生成**
  - 后端：`POST /api/ai/generate-tasks`（有数据且已配置 LLM 时优先调用 LLM；未配置或 LLM 返回空则规则兜底）。LLM 来源包括 `llm_intelligent`、`llm_anomaly`。
  - 前端：Dashboard / 店铺管理「智能生成」调用 `generateTasks({ storeId })`。返回 `metadata` 中含 `llmStatusMessage`、`statsDateRangeUsed` 等。
  - **LLM 调用方式**：管理员在「LLM 调用方式」页可为待办/话术选择 **Coze Agent** 或 **OpenAI 兼容接口**，与话术 LLM 配置共用；详见 [LLM 交互流程说明](./LLM交互流程说明.md)。
- **其他执行工具**
  - 报告 / 市场分析 / 商品推荐等：前端调用 `/api/ai/report`、`/api/ai/analyze-market` 等，后端当前未在 `ai-refactored.ts` 中实现对应路由，前端用 `.catch()` 做占位展示。

## 四、若未来为待办或其他工具接入 LLM

- **方案 A**：复用现有「话术 LLM 配置」：同一套 API 地址/密钥用于话术 + 待办（或报告等），无需新增配置入口，仅在后端对应接口中调用 `getScriptLLMConfigSync()` 或现有流式/非流式 LLM 封装即可。
- **方案 B**：为待办或某类执行工具单独增加「待办 LLM 配置」或「通用 AI 配置」：在 `system_config` 中增加新 key，管理员页增加对应表单，适合需要不同模型或不同 API 的场景。

当前建议：**只需完成「话术 LLM 配置」即可使用话术生成**；待办生成与其余执行工具按上表状态使用即可，无需额外 LLM 配置。
