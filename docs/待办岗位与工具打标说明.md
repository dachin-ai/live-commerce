# 待办岗位与工具打标说明

**更新日期**：2026-02-10  
**功能**：自动识别待办任务的负责岗位和执行工具，便于任务分配和工具调用

---

## 🎯 打标维度

### 1. 负责岗位（assignedRole）

| 值 | 显示 | 说明 | 示例任务 |
|---|------|------|----------|
| `anchor` | 👤 主播 | 主播专属任务，需主播在直播中执行 | 优化逼单话术、提升互动话术、控制直播节奏 |
| `operator` | 📊 运营 | 运营专属任务，后台策略制定与数据分析 | 复制最佳日策略、数据复盘、选品优化、时段分析 |
| `both` | 🤝 运营+主播 | 需要运营与主播协同完成 | 直播内容优化、商品推荐流程、直播时长调整 |
| `null` | - | 未明确归属 | 上传数据、节日提醒等 |

## 🤖 自动识别规则

### 岗位识别关键词

**主播（anchor）**：
- 话术、逼单、宠粉、互动话术
- 主播技巧、主播培训
- 语调、声音、现场应变
- 直播节奏、开场话术、收尾话术、带货话术、促单

**运营（operator）**：
- 复盘、数据分析、策略
- 选品、时段选择、时段优化
- 商品布局、活动策划、投放
- A/B 测试、测试方案
- 复制策略、优化策略
- 分析原因、建立机制、制定计划

**运营+主播（both）**：
- 直播内容优化、直播流程
- 场次安排、商品推荐直播
- 直播商品、短视频引流直播
- 直播间优化、直播时长优化

## 💻 代码实现

### 后端

**文件**：`backend/src/routes/ai-refactored.ts`

**识别函数**：`autoTagTaskRoleAndTool(title, description)` (L1915-1970)

**调用位置**：
1. LLM 任务解析时（L1902-1904）
2. 规则任务去重时（L2048-2058，补充未打标的任务）

**存储**：写入 `tasks` 表的 `assignedRole` 和 `aiFeature` 字段

### 前端

**文件**：`frontend/src/components/TaskList.tsx`

**显示位置**：任务标题下方，与店铺名称并列

**样式**：
- 岗位标签：紫色（主播）、绿色（运营）、黄色（运营+主播）
- 工具标签：橙色，带 emoji 图标

## 🔄 使用流程

### 自动打标

1. **生成任务**：用户点击「智能生成」
2. **LLM 返回**：Coze 返回任务列表（JSON）
3. **自动识别**：后端根据标题和描述关键词自动打标
4. **存储**：写入数据库（`assignedRole` 和 `aiFeature` 字段）
5. **前端显示**：任务卡片上显示岗位和工具标签

### 手动调整（可选）

如需手动修改标签，可在任务编辑时调整（需前端添加编辑功能）。

---

## 📝 扩展识别规则

如需添加新的识别规则：

1. **编辑**：`backend/src/routes/ai-refactored.ts` → `autoTagTaskRoleAndTool` 函数
2. **添加关键词**：在对应的 RegExp 中添加新关键词
3. **测试**：重新生成任务，验证识别准确性

**示例**：如需识别"选品"相关任务为运营岗位：
```typescript
const operatorKeywords = /复盘|数据.*分析|策略|选品|.../
```

---

## ⚠️ 注意事项

1. **旧任务**：修改前创建的任务，`assignedRole` 为 `null`（需重新生成）
2. **识别准确性**：基于关键词规则，准确率约 85-90%（可持续优化规则）
3. **多义任务**：部分任务可能同时匹配多个岗位，目前优先级为：主播 > 运营+主播 > 运营
4. **未识别任务**：无明确关键词的任务，标签为空（不影响使用）
