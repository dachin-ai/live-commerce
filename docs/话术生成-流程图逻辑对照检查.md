# 话术生成 - 流程图逻辑对照检查

本文档对照「话术生成」流程图，逐项检查当前实现是否按该逻辑执行。

## 流程图要点摘要

- **起点**：话术模块
- **路径 1**：话术模块 → 用户在前端提供参数 → LLM 生成
  - **成功**：LLM 生成 → 原生内容 → 系统前端展示
  - **失败**：LLM 生成（调用失败）→ 兜底规则 → 生成内容处理 → 系统前端展示
- **路径 2**：话术模块 → 兜底规则 → 生成内容处理 → 系统前端展示
- **约束（黄色框）**：系统不需要做「自己构建提示词、维护提示词」

---

## 对照结果

| 检查项 | 流程图要求 | 当前实现 | 结论 |
|--------|------------|----------|------|
| 话术模块 + 用户参数 | 话术模块由用户在前端提供参数驱动 | 前端 `AIFeatures` 提交 productName、scriptType、storeId 等 → 后端 `parseScriptRequestBody` 解析，`runScriptResearch(userInput, storeContext)` 做市调 | ✅ 一致 |
| LLM 成功 → 原生内容 → 前端展示 | LLM 成功时输出「原生内容」并直接到前端展示 | 流式接口：`streamScriptFromLLM` 收集 `llmContent`，`resolveScriptContentWithFallback` 在内容非空且（使用 Coze 提示词或像话术）时保留原文；合规检查后送 `script`，前端展示 | ✅ 一致 |
| LLM 返回空 → 兜底 | LLM 未返回有效内容时走兜底 | `resolveScriptContentWithFallback` 在 `content` 为空时用 `synthesizeScript(research).content` 模板兜底；SSE 返回 `fallbackReason: 'llm_timeout_or_empty'`，前端 `onFallback` 提示「已为您切换为模板话术」 | ✅ 一致 |
| **LLM 调用失败 → 兜底** | 调用失败（超时、网络错误等）时应走兜底规则，最终仍展示内容 | **已修复**：`/stream` 中 `streamScriptFromLLM` 用 try-catch 包裹，异常时置空 `llmContent`，后续与「返回空」同一路径走模板兜底并正常返回 script，不再仅返回 error | ✅ 已按流程图实现 |
| 兜底规则 → 生成内容处理 | 兜底内容需经「生成内容处理」再展示 | 兜底内容来自 `synthesizeScript(research).content`（模板 + 合规 + 数据依据）；再经 `checkPlatformCompliance`、`injectVisualSuggestionsIntoContent`（仅兜底时）、翻译（若非中文）后写入 `script.content` | ✅ 一致 |
| 未配置 LLM 时 | 图中有「话术模块 → 兜底规则」路径，无 LLM 时也出模板 | **已实现**：未配置时不再返回 503，直接走市调 + 模板兜底，返回 `synthesizeScript(research).content`，并带 `fallbackReason: 'llm_not_configured'`，前端提示「未配置话术 LLM，已为您展示模板话术」 | ✅ 一致 |
| 提示词职责 | 系统不需要做：自己构建提示词、维护提示词 | **方案 1 已采用**：Coze 模式下仅发工具调用式消息（`buildScriptToolCallMessage`），不再维护长提示词；OpenAI 模式仍用本地 `buildCozeScriptPrompts` / `buildLLMSystemPrompt`。见 `docs/话术生成-Coze方案与当前实现对照.md` | ✅ Coze 侧已对齐 |

## 涉及代码位置

- **流式入口**：`backend/src/routes/ai/script.ts` — `POST /stream`
- **LLM 调用与失败兜底**：同上，`streamScriptFromLLM` 外 try-catch，异常时 `llmContent = ''`，再走 `resolveScriptContentWithFallback`
- **兜底内容来源**：`backend/src/rules/scriptSynthesis.ts` — `synthesizeScript(research)`
- **成功/兜底判定**：`backend/src/services/scriptOutputValidator.ts` — `resolveScriptContentWithFallback`
- **前端展示与 onFallback**：`frontend/src/services/ai.ts` — `generateScriptStream`；`frontend/src/components/AIFeatures.tsx` — `onFallback` 提示

---

## 总结

- **主流程**：话术模块 → 用户参数 → LLM 生成 → 成功则原生内容到前端、失败或为空则兜底规则 → 生成内容处理 → 前端展示，**已按流程图实现**。
- **本次修复**：LLM 调用抛错（如流式中断、超时抛错）时，由原先仅返回错误改为走模板兜底并正常返回话术内容，与流程图「调用失败 → 兜底规则 → 生成内容处理 → 系统前端展示」一致。
- **可选**：未配置 LLM 时改为仍返回模板话术；提示词职责以产品选型（Coze 方案 1 vs 方案 2）为准，见 `docs/话术生成-Coze方案与当前实现对照.md`。
