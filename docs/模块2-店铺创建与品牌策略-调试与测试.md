- **行业共性**：多步骤/分组表单、必填校验、类目联动、AI 辅助生成

## 一、调试清单（先做）

### 1. 环境与依赖
- 后端已启动：`cd backend && npm run dev`，http://localhost:3000/health 返回 ok
- 前端已启动：`cd frontend && npm run dev`，http://localhost:5173
- 已登录：使用 admin@example.com / 123456 登录并进入 Dashboard

### 2. 创建入口说明
- **无店铺时**：页面中央为「请选择店铺」，点击 **「创建新店铺」** 会打开创建店铺模态框（Dashboard 已监听 `openCreateStoreModal`）。
- **有店铺时**：左侧「商店列表」卡片上方有 **「创建商店」** 按钮，点击后同样打开同一模态框。

### 3. 相关代码与接口
- **前端**：`CreateStoreModal.tsx`（表单、分类联动、国家-货币联动、AI 生成品牌策略）、`StoreList.tsx`（「创建商店」按钮与模态框）、`Dashboard.tsx`（「创建新店铺」派发事件）
- **后端**：`/api/stores` POST 创建店铺、`/api/ai/brand-strategy` 品牌策略、`/api/categories` 分类树、`/api/regions/countries` 国家列表、`/api/regions/currency?region=国家名` 按国家返回货币
- **国家-货币**：仅国家下拉（无区域）；选择国家后，货币自动跟随该国（CNY/THB/VND 等），仅在请求完成后再更新表单，避免切换国家时沿用上一国货币

### 4. 若创建失败或 AI 无响应，请检查
- 控制台 Network：POST /api/stores、POST /api/ai/brand-strategy、GET /api/regions/currency、GET /api/categories 状态码与响应
- 后端终端：数据库、分类/地区初始化是否完成
- 店铺名称必填；分类/国家/价格等按表单提示填写

### 5. 货币与分类显示（严谨验证）
- **货币**：国家下方始终显示一行「推荐货币」（加载中 / 具体货币 / 默认人民币）；价格区间旁显示「(符号 代码)」如 (¥ CNY)；**选择泰国后必须显示泰铢 (฿ THB)**，选择越南显示越南盾 (₫ VND) 等，与所选国家一致。
- **产品分类**：参考抖店设置，一级为 服饰鞋包、美妆护肤、食品健康、亲子生活、家居家电、3C数码、运动户外、汽车、鲜花园艺、珠宝文玩、文化娱乐；选一级后出现对应二级（可多选），选二级后出现三级（可多选）。若分类不全，可运行 **重置数据库.bat** 后重新登录以加载抖店全量分类（会清空示例店铺/任务等）。

## 三、调试记录（国家-货币一致）

- **现象**：选择国家后，货币未同步，仍显示上一国家的货币。
- **原因**：切换国家时，`useCurrencyByRegion(country)` 会发起新请求，但 React Query 在请求完成前仍返回上一国家的缓存数据；原逻辑一有 `currencyInfo` 就写回表单，导致用旧数据覆盖。
- **修复**：在 `CreateStoreModal` 中从 `useCurrencyByRegion` 解构 `isFetching`，仅在 `!currencyOverride && !currencyFetching && currencyInfo` 时用 `currencyInfo` 更新表单，避免在「该国家的货币请求尚未完成」时沿用旧货币。
- **验证**：按步骤 5 → 5b → 5c 切换国家，确认推荐货币与所选国家一致。
