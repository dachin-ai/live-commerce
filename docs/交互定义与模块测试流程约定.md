# 交互定义与模块测试流程约定

本文档固化我们交互过程中达成一致的**术语定义**，以及**模块开发与测试的流程图逻辑**，供后续对话与 Agent 一致遵守，减少歧义。

---

## 一、交互过程中的定义约定

### 1. 时效

- **本项目中「时效」特指：每小时成交额**（GMV ÷ 直播时长），单位：店铺货币/时。
- **不表示**：数据更新时效（如 T+1）、人均观看时长等。
- **展示位置**：数据仪表盘中的统计卡片之一（与 GMV、直播时长等并列），可在「数据项目」中勾选显隐。

### 2. 数据周期与「总直播时长」等合计值

- **时间周期定义（符合用户习惯）**：
  - **本周**：自然周，周一 00:00 ～ 周日 23:59（周一为一周第一天）。
  - **本月**：自然月，本月 1 日 ～ 本月最后一天。
  - **本季度**：自然季度（Q1=1–3 月、Q2=4–6 月、Q3=7–9 月、Q4=10–12 月），本季度首日 ～ 末 day。
  - **本年**：**当前日历年**，当年 1 月 1 日 ～ 12 月 31 日（如 2026 年则仅 2026 年数据）。要看 **2025 年全年** 请用「选择年份」选 2025。
  - **选择月份**：可选 2020 年 1 月 ～ 当前月（含 2025 年各月）；**选择年份**：可选 2020 年起共 12 年（含 2025 年）。
- **仪表盘「总直播时长」「GMV」等**：均为**当前选择周期内的合计**（选「本周」= 该自然周内所有 stats 的合计，选「本月」= 该月内合计）。
- **同一周期内、不同日期多次导入**：每个自然日最多一条 stats（同店同日替换）；若一周内周一、周二各导入一次，则本周合计 = 周一那条 + 周二那条。
- **「当前周直播时长」与「总直播时长」**：选「本周」时一致，均为该自然周内 stats 的 totalDuration 之和。

### 3. 直播时长单位（Excel 导入）

| 列名/来源 | 单位 | 处理方式 |
|-----------|------|----------|
| **直播时长(秒)、时长(秒)**（TikTok 导出多为秒） | 秒 | 秒 → 分钟 → 小时，前端展示为小时 |
| 时长(小时)、直播时长(小时) | 小时 | 直接按小时参与合计 |
| 直播时长、时长(分钟) 等 | 分钟（数值>1440 时按秒转分钟） | 除以 60 得到小时 |

详见 `docs/数据统计口径说明.md`、`docs/TikTok数据导入使用指南.md`。

### 4. 货币与多国

- **店铺货币**：创建店铺时选的国家对应货币（如泰铢 ฿、人民币 ¥），为**数据原始单位**；导入结果、接口中的 GMV 均按店铺货币存储。
- **「显示为」**：仪表盘支持「店铺货币」与「人民币对多国」切换（人民币、美元、泰铢等），换算使用**前端静态汇率**；暂不接后端汇率接口（曾接入选型后已回退）。

### 5. 派生指标

- **人均观看时长**：总直播时长(分钟) / 总观看人数，单位分钟。
- **千次观看成交 (GPM)**：(GMV / 总观看人数) × 1000。
- 二者默认在仪表盘「数据项目」中勾选展示。

### 6. 模块测试流程（与流程图一致）

- **步骤 1**：按模块实现/确认**后端功能代码**（只做当前模块）。
- **步骤 2**：**调用 MCP 测试前端**，需**对标当前模块的市场成品方案**（见《各模块步骤总结》对标样本）。
- **步骤 3**：**判断是否存在问题**（失败项、不符合预期、或与市场方案不一致即视为有问题）。
- **是** → 回到**步骤 1**（修后端或与当前模块相关的前端代码，修完再从步骤 2 重测），循环直到无问题。
- **否** → **输出模块报告** → **人工终测**。

流程图已固化为 **Cursor 规则**（`.cursor/rules/module-test-workflow.mdc`）与 **Agent Skill**（`.cursor/skills/module-test-workflow/SKILL.md`）。做模块测试时 Agent 按此顺序执行，不跳过、不调换。

### 7. 启动与调试

- **后端**：`cd backend; npm run dev`，默认端口 3000；PowerShell 下用 `;` 连接命令，不用 `&&`。
- **前端**：`cd frontend; npm run dev`，默认 5173（被占用时 Vite 会改用 5174/5175）。
- **端口占用**：后端启动失败时先查占用（如 `netstat -ano | findstr ":3000"`），结束对应进程后再启动。
- **脚本编码**：.bat / 脚本内注释与输出用英文或 ASCII，避免中文导致乱码。

## 二、流程图逻辑（与图片一致）

```
┌─────────────────────────────────────┐
│ 按模块实现后端功能代码                 │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│ 调用 MCP 测试前端，需对标当前模块的     │
│ 市场成品方案                          │
└─────────────────┬───────────────────┘
                  ▼
┌─────────────────────────────────────┐
│ 判断是否存在问题                       │
└─────────┬───────────────────┬───────┘
          │ 是                 │ 否
          ▼                   ▼
    回到步骤 1          ┌──────────────┐
    （修后端/相关代码    │ 输出模块报告  │
     后重新 MCP 测）     └──────┬───────┘
                               ▼
                        ┌──────────────┐
                        │ 人工终测      │
                        └──────────────┘
```

- **约束**：未完成 MCP 测试并对标前，不输出「模块通过」报告；发现问题必须先修复再重测。
