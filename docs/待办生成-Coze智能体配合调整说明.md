# 待办生成 - Coze 智能体需配合的调整

**终端用户**：无 Coze 权限、也无需任何第三方。用户在本系统内选择店铺并点击「智能生成」后，由本系统**自动完成**（后端拉取该店铺数据、生成待办并写入），全程在系统内完成。

本文档说明在「智能生成」待办流程中，**Coze 控制台侧**需要配合的配置与注意事项，以便与后端优化后的流程一致、减少平台错用与解析失败。

---

## 1. 回复格式（必做）

- **只输出一个 JSON 对象**，不要在任何位置输出：
  - markdown 代码块（如 \`\`\`json ... \`\`\`）
  - mermaid 图或流程图
  - 分析段落、总结、建议说明等
- 后端会从整段回复中**自动提取** `{"tasks":[...]}`，但若 Bot 在 JSON 前输出大段其他内容，可能增加解析耗时或偶发提取失败；**直接以 `{"tasks":[` 开头回复**最稳妥。
- 在 Coze 该 Bot 的**人设/开场/回复逻辑**中，可再次强调：「本场景仅输出一个 JSON，格式为 {"tasks":[{"title":"","description":"","priority":"urgent或normal"}]}，不要任何前缀或后缀。」

---

## 2. 平台与数据来源（必做）

- **数据来源须与用户店铺平台一致**：后端已在请求中传入「店铺平台」（如 TikTok、抖音、快手）。Coze 生成的任务描述里若提到数据来源，必须写**该店铺的平台**，例如：
  - 店铺为 **TikTok** → 只写「TikTok 平台」数据，**不得**出现「抖音/快手」；
  - 店铺为 **抖音** → 写「抖音平台」；
  - 店铺为 **快手** → 写「快手平台」。
- 在 Coze 的**人设或提示**中可加一句：「根据用户消息中的【店铺】平台（如 TikTok/抖音/快手），任务描述中的数据来源必须与该平台一致，不得混用其他平台名称。」

---

## 3. 人设与知识库（建议）

- **人设**：不要写「抖音电商待办助手」等单一平台表述，建议改为「直播电商待办助手，支持多平台（TikTok、抖音、快手等）」，避免模型默认倾向抖音。
- **知识库**：若上传了「抖音大盘数据」「快手行业报告」等文档，建议：
  - 在文档或知识库说明中标注适用平台（如「仅用于抖音/快手店铺」），或
  - 为 TikTok 单独准备/标注知识，避免 TikTok 店铺请求时被误用为抖音数据。

---

## 4. 对话与记忆（建议）

- 待办生成由后端**单次请求**完成，不携带历史对话。若 Coze 开启了多轮记忆，建议在该 Bot 的配置中**关闭多轮记忆**或确保首轮回复不依赖历史，避免历史对话干扰「只输出 JSON」的行为。

---

## 5. 后端已做的优化（供参考）

以下为**当前**配合项；历史已移除项（如 Agent/store_id）见《LLM与智能生成-文档索引》第四节「废案/已停用」。

| 优化项 | 说明 |
|--------|------|
| ~~Agent 协议带 store_id~~（已移除） | 本系统已移除 `/api/agent`（2026-02）。现 Coze 直接传 prompt（content.query.prompt），见《Coze对接说明-待办与话术入参出参》。 |
| 传入店铺属性 | 目标人群、品牌定位、价格区间、品牌策略、店铺说明等已拼入 user 消息的【店铺属性】 |
| 传入已有待办标题 | 当前店铺下 pending 任务标题会传入【已有待办】，要求生成与已有不重复的任务 |
| 平台一致约束 | system 提示中明确「数据来源须与店铺平台一致」，并给出 TikTok/抖音/快手示例 |
| 仅输出 JSON | system 与 user 结尾均强调「仅输出一个 JSON」「不要 markdown/mermaid」 |
| 超时 | 待办生成使用 180s（3 分钟）流式超时（可配置），Coze 需在该时间内**在 answer/delta 或 tool_response 中**返回完整回复。与《Coze对接说明》表格一致，**推荐在 answer/delta 直接流式输出 JSON**，不依赖 tool。 |

## 6. 在 Coze 里自测

- 使用 `docs/Coze待办生成-测试消息.md` 中的**完整测试消息**（含【店铺】为 TikTok 或抖音的示例）在 Coze 对话中发送。
- 检查回复是否：  
  1）仅一个 JSON、无代码块无 mermaid；  
  2）描述中数据来源与【店铺】平台一致（TikTok 店铺无「抖音/快手」字样）。
- 若测试通过，实际「智能生成」时后端传入的 message 结构相同，仅店铺/数据变量不同，行为应一致。
