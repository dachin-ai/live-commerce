# 部署前全局检查清单

上线前按本清单逐项确认，避免生产环境踩坑。

---

## 一、安全与配置

| 检查项 | 说明 | 状态 |
|--------|------|------|
| JWT_SECRET | 生产环境**必须**在 `backend/.env` 中设置，且不能为默认值 `your-secret-key-change-in-production`；未设置时后端会拒绝启动 | ✅ 已加启动校验 |
| .env 不提交 | `backend/.env` 已在 `.gitignore`，仅放在服务器本地，不要提交到 Git | ✅ |
| 密钥类变量 | LLM API Key、BOT_API_KEY、AGENT_API_KEY 等仅通过 .env 或管理后台配置，不写进代码 | ✅ |

---

## 二、前端与 API

| 检查项 | 说明 | 状态 |
|--------|------|------|
| API 地址 | 前端使用相对路径 `baseURL: '/api'`，同域部署无需改；流式话术同样使用相对路径 | ✅ |
| 构建产物 | 使用 `npm run build` 生成 `frontend/dist`，无 localhost 硬编码 | ✅ |
| 超时 | 话术流式、长文本翻译等接口前端超时已设 90s/120s | ✅ |

---

## 三、后端与运行

| 检查项 | 说明 | 状态 |
|--------|------|------|
| .env 加载 | 从 `backend` 目录加载 `.env`（`path.join(__dirname, '..', '.env')`），PM2 从 backend 目录启动即可 | ✅ |
| 数据库 | SQLite 使用 `backend/data.db`，WAL 已开启；部署时保留或迁移该文件 | ✅ |
| uploads | 上传目录为 `backend/uploads/`，静态路由 `/uploads`；部署时需可写或自动创建 | ✅ |
| 限流 | 15 分钟内 500 次请求；多用户可考虑调大或按 IP 细分 | ✅ |

---

## 四、Nginx（同域部署）

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 反向代理 | `location /api/`、`location /uploads/` 代理到 `http://127.0.0.1:3000` | 见部署指南 |
| API 超时 | 话术流式、翻译等长耗时接口需 **≥120s**，建议 `proxy_read_timeout 120s; proxy_send_timeout 120s;` | ✅ 已写入部署指南 |
| 上传大小 | `client_max_body_size 10m;` 已满足素材上传 | ✅ |
| HTTPS | 生产建议 certbot 配置 HTTPS 并 HTTP→HTTPS 重定向 | 见部署指南 |

---

## 五、部署步骤速查

```bash
# 本地/CI
npm run install:all
npm run build
# 确认 frontend/dist/index.html、backend/dist/index.js 存在

# 服务器
# 1. 上传 frontend/dist、backend（含 dist、node_modules、.env、data.db、uploads）
# 2. backend/.env 中设置 JWT_SECRET=<强随机字符串>
# 3. cd backend && pm2 start dist/index.js --name lvbcsym-api && pm2 save
# 4. Nginx 配置见《部署到域名操作指南》第六章，注意 API 超时 120s
# 5. sudo nginx -t && sudo systemctl reload nginx
```

---

## 六、上线后快速验证

- 访问 `https://你的域名/` → 前端正常
- 访问 `https://你的域名/health` 或通过前端间接请求 → 后端健康检查（若 Nginx 未暴露 /health 可跳过）
- 登录 → 能进 Dashboard
- 选择店铺 → 统计/待办/话术入口可点
- 话术生成 / 翻译（若使用）→ 不超时、不 502

完成以上检查即可视为具备上线条件。详细步骤与故障排查见 **《部署到域名操作指南》**。
