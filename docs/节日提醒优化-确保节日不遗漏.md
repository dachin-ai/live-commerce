# 节日提醒优化 - 确保节日不遗漏

**日期**：2026-02-11  
**问题**：泰国店铺没有监测到接下来的情人节  
**原因**：当 LLM 成功生成任务时，规则里的"【节日提醒】"不会被添加

---

## 🐛 问题分析

### 原流程

```
用户点击「智能生成」
  ↓
后端调用 LLM（Coze）
  ↓
LLM 成功返回 8 条任务 → ✅ 直接使用，跳过规则补充
  ├─ 任务中可能包含情人节相关（如"情人节主题直播策划"）
  └─ 但没有明确的"【节日提醒】情人节倒计时X天"标签
  ↓
LLM 失败或返回 0 条 → ❌ 走规则兜底
  └─ 此时才会添加"【节日提醒】情人节倒计时3天 - 备货准备"
```

### 问题

- ✅ 系统能正确识别泰国情人节（2月14日，还有3天）
- ✅ LLM 也会生成情人节相关任务（如"情人节主题直播筹备"）
- ❌ 但**没有明确的节日倒计时提醒**（"【节日提醒】情人节倒计时3天"）

---

## ✅ 解决方案

### 修改逻辑

**无论 LLM 是否成功，都检查并补充节日提醒**：

```typescript
// LLM 成功后
if (intelligentTodos.length > 0) {
  // ... 保存 LLM 任务
  
  // 新增：检查 LLM 是否已生成节日相关任务
  const llmTitles = intelligentTodos.map(t => t.title)
  const hasEventInLLM = llmTitles.some(t => /节日|情人节|圣诞|宋干|水灯|倒计时|备货/.test(t))
  
  // 如果 LLM 没有生成节日任务，补充规则节日提醒
  if (!hasEventInLLM && upcomingEvents.length > 0) {
    console.log(`[节日补充] LLM 未生成节日相关任务，补充 ${upcomingEvents.length} 个节日提醒`)
    for (const event of upcomingEvents) {
      tasks.push({
        title: `【节日提醒】${event.name}倒计时${event.daysUntil}天 - 备货准备`,
        description: `${event.name} ${event.daysUntil}天后到来。建议：${event.recommendation}`,
        priority: event.daysUntil <= 7 ? 'urgent' : 'normal',
        aiFeature: 'event',
        source: 'event',
        assignedRole: 'operator',
      })
    }
  }
}
```

### 判断逻辑

- ✅ LLM 生成了"情人节主题直播筹备" → 已覆盖，不补充
- ✅ LLM 没有生成节日相关任务 → 补充"【节日提醒】情人节倒计时3天"
- ✅ LLM 失败/返回0 → 走原规则兜底（含节日提醒）

## 🎯 效果

### 修改前

| 情况 | 节日提醒 |
|------|----------|
| LLM 成功 | ❌ 可能遗漏（取决于 LLM 是否生成） |
| LLM 失败 | ✅ 有（规则兜底） |

### 修改后

| 情况 | 节日提醒 |
|------|----------|
| LLM 成功且生成节日任务 | ✅ 有（LLM 生成） |
| LLM 成功但未生成节日任务 | ✅ 有（规则补充） |
| LLM 失败 | ✅ 有（规则兜底） |

- LLM 的 2 条任务
