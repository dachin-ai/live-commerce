# 部署到具体域名操作指南

本文说明如何将「直播电商中台」前后端部署到一台服务器并绑定具体域名访问。

**当前配置域名：`atlvbcsy.com`**（火山引擎）

---

## 一、架构概览

- **前端**：Vite + React，构建后为静态文件（`frontend/dist`），通过 Nginx 或同一后端提供。
- **后端**：Node.js + Express，默认端口 3000，提供 `/api/*` 和 `/uploads`。
- **推荐**：同一域名，Nginx 反向代理：`/` → 前端静态，`/api`、`/uploads` → 后端。

---

## 二、部署前准备

### 2.1 服务器与域名

1. 有一台可公网访问的服务器（如 Linux），记下其**公网 IP**（例如 `1.2.3.4`）。
2. 在域名服务商处将 **A 记录** 指向该 IP，见下文「火山引擎域名解析」。

### 2.2 本地构建

在项目根目录执行：

```bash
# 安装依赖（若未安装）
npm run install:all

# 构建前端 + 后端
npm run build
```

产物位置：

- 前端：`frontend/dist/`
- 后端：`backend/dist/`，且需保留 `backend/node_modules`、`backend/.env`、`backend/data.db`（若用 SQLite）等运行所需文件。

---

## 三、服务器环境

### 3.1 安装 Node.js

建议 Node 18+（LTS）。例如 Ubuntu：

```bash
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs
```

### 3.2 安装 Nginx（推荐）

```bash
sudo apt-get update && sudo apt-get install -y nginx
```

### 3.3 安装 PM2（用于常驻运行后端）

```bash
sudo npm install -g pm2
```

---

## 四、上传与目录结构

将以下内容上传到服务器（可用 scp、rsync、Git 等）：

- `frontend/dist/` → 例如 `/var/www/lvbcsym/frontend/`
- `backend/` 整个目录（含 `dist/`、`node_modules/`、`.env`、`data.db`、`uploads/` 等）→ 例如 `/var/www/lvbcsym/backend/`

若在服务器上拉代码并构建：

```bash
# 在服务器上
cd /var/www/lvbcsym
git clone <你的仓库> .
npm run install:all
npm run build
```

---

## 4.5 火山引擎域名解析（atlvbcsy.com）

域名已在火山引擎，使用当前 DNS（`vip3.volcengine-dns.com` / `vip4.volcengine-dns.com`）即可。

1. 在域名详情页点击 **「域名解析」**（或从左侧进入 **云解析 DNS**）。
2. 为域名 **atlvbcsy.com** 添加解析记录：
   - **记录类型**：`A`
   - **主机记录**：`@`（表示根域名 `atlvbcsy.com`；若要用 `www.atlvbcsy.com` 再加一条主机记录 `www`）
   - **记录值**：你的**服务器公网 IP**（如 `1.2.3.4`）
   - **TTL**：600 或默认
3. 保存后等待几分钟生效，可用 `ping atlvbcsy.com` 检查是否解析到该 IP。

如需同时用 `www.atlvbcsy.com` 访问，再添加一条：主机记录 `www`，类型 `A`，记录值同一 IP。

## 五、后端配置与启动

### 5.1 环境变量

在 `backend/.env` 中配置生产环境（可参考 `backend/.env.example`）：

```env
PORT=3000
JWT_SECRET=你的强随机密钥-生产环境务必修改
# 若使用 Coze/豆包等，按需配置：
# SCRIPT_LLM_URL=
# SCRIPT_LLM_API_KEY=
# AGENT_API_KEY=
# BOT_API_KEY=
```

注意：**JWT_SECRET** 必须与开发环境不同且足够随机。

### 5.2 用 PM2 启动后端

```bash
cd /var/www/lvbcsym/backend
pm2 start dist/index.js --name lvbcsym-api
pm2 save
pm2 startup   # 按提示执行，实现开机自启
```

检查：

```bash
curl http://localhost:3000/health
# 应返回 {"status":"ok","timestamp":"..."}
```

## 六、Nginx 配置（atlvbcsy.com）

域名：**atlvbcsy.com**（含 www）；前端静态目录 `/var/www/lvbcsym/frontend`，后端本机 3000 端口。

新建站点配置（如 `/etc/nginx/sites-available/lvbcsym`），**可直接复制使用**：

```nginx
server {
    listen 80;
    server_name atlvbcsy.com www.atlvbcsy.com;

    # 前端静态
    root /var/www/lvbcsym/frontend;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端 API 与上传文件（超时 120s：话术流式/长文本翻译等接口耗时较长）
    location /api/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 60s;
        proxy_send_timeout 120s;
        proxy_read_timeout 120s;
    }
    location /uploads/ {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    client_max_body_size 10m;
}
```

启用并重载 Nginx：

```bash
sudo ln -s /etc/nginx/sites-available/lvbcsym /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

完成后可通过 **http://atlvbcsy.com** 或 **http://www.atlvbcsy.com** 访问（解析生效后）。API 走同域名 `/api`，无需改前端。

## 七、HTTPS（推荐）

使用 Let’s Encrypt 免费证书（需先完成 Nginx 配置且域名已解析到本机）：

```bash
sudo apt-get install -y certbot python3-certbot-nginx
sudo certbot --nginx -d atlvbcsy.com -d www.atlvbcsy.com
```

按提示选择是否重定向 HTTP→HTTPS。Certbot 会改写 Nginx 配置并自动续期。

---

## 八、前端与生产环境相关说明

- 前端请求使用 **相对路径** `baseURL: '/api'`，因此只要页面与 API 同域（通过 Nginx 代理），无需再配 API 地址。
- 素材/上传链接已使用当前页面 **同源**（`window.location.origin`），部署到域名后会自动指向当前域名，无需改配置。
- 若你使用 **子域名分离**（如 `api.atlvbcsy.com` 单独作为 API 域名），需要：
  - 在 Nginx 中为 API 子域名单独做 `proxy_pass` 到 3000；
  - 在前端构建时注入 API 根地址（例如通过 `VITE_API_BASE=https://api.atlvbcsy.com` + 前端读取该变量设置 `axios` 的 `baseURL`），并处理 CORS。

## 九、检查清单

| 项 | 说明 |
|----|------|
| 域名 A 记录 | 已指向服务器 IP |
| 防火墙 | 开放 80、443（若用 HTTPS） |
| 后端 | `backend/.env` 中 **JWT_SECRET 必须设置且不能为默认值**（否则进程不启动），PM2 运行正常 |
| Nginx | `server_name` 与域名一致，`/api`、`/uploads` 代理到 3000，**API 超时建议 ≥120s** |
| 前端 | 使用 `npm run build` 的 `frontend/dist` 部署，请求为相对路径 `/api`，无 localhost 硬编码 |
| .env | 仅存在于服务器本地，**不要提交到版本库**（已列入 .gitignore） |
| HTTPS | 生产建议开启并启用 HTTP→HTTPS 重定向 |

## 十、部署前最后检查（推荐执行）

在本地或 CI 执行一次，再上传/拉代码到服务器：

```bash
# 1. 安装依赖并构建
npm run install:all
npm run build

# 2. 确认产物存在
ls frontend/dist/index.html backend/dist/index.js

# 3. 生产环境会强制校验 JWT_SECRET，未配置则后端不启动
# 在服务器上确保 backend/.env 中已设置：JWT_SECRET=<强随机字符串>
```

服务器上首次或更新后：

- 后端：`cd backend && pm2 start dist/index.js --name lvbcsym-api && pm2 save`
- Nginx：`location /api/` 的 `proxy_read_timeout`、`proxy_send_timeout` 建议 **120s**（话术流式、长文本翻译会超 60s）。

## 十一、常见问题

- **访问域名一直 502**：检查后端是否在跑（`pm2 list`）、端口是否为 3000、Nginx `proxy_pass` 是否为 `http://127.0.0.1:3000`。
- **接口 404**：确认 Nginx 中 `location /api/` 存在且 `proxy_pass` 末尾带 `/` 或与后端路由一致。
- **登录后刷新又掉线**：检查 Cookie/Token 是否被正确携带；若用 HTTPS，确认 Nginx 已正确设置 `X-Forwarded-Proto`，后端若做 secure cookie 需信任代理。
- **话术生成/翻译到一半断开**：将 Nginx `location /api/` 的 `proxy_read_timeout`、`proxy_send_timeout` 调大至 120s 或以上。
- **后端启动报错「生产环境必须设置 JWT_SECRET」**：在 `backend/.env` 中设置 `JWT_SECRET` 为强随机字符串（不要使用默认值）。

按上述步骤即可将系统部署到具体域名并正常使用。
