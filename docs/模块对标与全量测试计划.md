## 原则

每个测试模块执行前，先明确**成熟样本（对标对象）**及**对标标准**，再按标准执行用例并记录结果。

## 模块 1：登录与认证

### 成熟样本
- **淘宝开放平台 / 抖音开放平台**：OAuth2.0、Token 刷新、多端登录
- **行业共性**：账号密码登录、用户注册、找回密码、登录后个人中心、Token 持久化、未登录重定向、登录态失效跳转登录

### 对标标准
| 维度 | 成熟表现 | 本系统验证点 |
|------|----------|--------------|
| 访问未授权页 | 重定向到登录页 | 访问 `/` 未登录 → 跳转 `/login` |
| 登录成功 | 跳转业务首页、写入 Token | 登录后到 `/`，localStorage 有 token |
| 登录失败 | 明确错误提示、不暴露敏感信息 | 错误提示为「邮箱或密码错误」等 |
| 用户注册 | 独立注册页、必填校验、成功后可登录 | 登录页「立即注册」→ `/register`，填写姓名/邮箱/密码/确认密码，注册成功跳转登录并提示 |
| 找回密码 | 输入邮箱获取重置链接、凭链接设置新密码 | 登录页「忘记密码？」→ 输入邮箱 → 开发环境可「前往重置密码」→ 设置新密码 → 用新密码登录 |
| 个人中心 | 查看/编辑资料、修改密码 | 侧栏「个人中心」→ `/profile`，可改姓名并保存、可修改密码（需当前密码） |
| 控制台 | 无未捕获异常 | 登录/注册/找回密码/个人中心流程无报错 |

## 模块 12：错误处理

### 成熟样本
- **成熟 Web 应用**：接口失败/网络错误/401 有明确提示或跳转，不白屏、不未捕获异常
- **行业共性**：用户可读提示、401 跳转登录、控制台可排查

### 对标标准
| 场景 | 成熟表现 | 本系统验证点 |
|------|----------|--------------|
| 后端不可用 | 友好提示（如网络/服务异常） | Toast 等提示、无整页崩溃 |
| 401 | 跳转登录 | 清除 token 后操作→跳转 `/login` |
| 控制台 | 有错误日志、无未处理 Promise | 可接受的服务/网络错误日志 |
