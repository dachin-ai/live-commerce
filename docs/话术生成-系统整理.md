# 话术生成 - 系统整理

本文档系统梳理与「话术生成」相关的代码、配置、API 与文档，便于排查与扩展。**遇到问题请先对照此处定位模块，再按文档修复。**

---

## 一、功能概览

- **能力**：根据产品名、价格、卖点、目标人群、话术类型、店铺等生成 TikTok 直播带货话术；支持 5 种话术类型、多国家/货币、Coze 仅输出纯中文 + 终端翻译。
- **入口**：前端「执行工具」→ 选择「话术生成」工具 → 填写表单 → 点击生成（流式）。
- **API**：`POST /api/ai/script/stream`（流式，主用）、`POST /api/ai/script`（同步，当前仅模板）、`GET/POST /api/ai/script/config`（配置）。

## 三、请求与入参（与 Coze 对齐）

- **流式**：`POST /api/ai/script/stream`，Body 为 JSON。
- **同步**：`POST /api/ai/script`，Body 同流式（当前实现仅用模板，不调 LLM）。

**Body 字段（与 Coze 话术工具入参对应）**：

| 前端/Body 字段 | 必填 | 说明 | Coze 对应 |
|----------------|------|------|-----------|
| productName | 建议 | 产品名称 | product_name |
| scriptType | 否，默认 full-sales | full_process / full-sales / interaction / scenario / promotion / closing | script_type |
| country | 否 | 国家名（如 泰国、菲律宾） | country |
| countryCode | 否 | 国家代码（如 TH、PH），优先级高于 country | - |
| price | 否 | 价格，可带货币符号 | price |
| features | 否 | 产品特点 | features |
| targetAudience | 否 | 目标人群 | target_audience |
| promotion_info / promotionInfo / promoCopy | 否 | 促销信息，三选一 | promotion_info |
| custom_requirements / customRequirements | 否 | 自定义要求 | custom_requirements |
| productSku | 否 | SKU，格式可参考 S码(尺寸,适用);M码(...) | sku_info |
| language | 否，默认 zh-CN | zh-CN / en-US / th-TH（终端翻译目标） | - |
| storeId | 否 | 店铺 ID，用于拉店铺上下文与 region | - |
| topic, duration, style | 否 | 主题、时长、风格，topic/style 可并入 custom_requirements | - |

**说明**：Coze 侧仅生成**纯中文**话术；若 `language !== 'zh-CN'`，后端在拿到中文后再用 `translateLongText` 翻译为目标语言。

## 四、流程简述（流式）

1. **权限**：若配置了「允许用户列表」，校验当前用户是否在列表中。
2. **解析**：`parseScriptRequestBody(req.body)` → `userInput`、`storeContext`、`promotionInfo`、`countryCode`。
3. **市调**：`runScriptResearch(userInput, storeContext)` → 人群解析、合规、品类实践、市调摘要。
4. **提示词**：若有 `promotionInfo` 则用 `buildCozeScriptPrompts(...)`（Coze 风格，仅中文输出）；否则 `buildLLMSystemPrompt` + `buildLLMUserMessage`。
5. **LLM**：`streamScriptFromLLM({ systemPrompt, userMessage, ... })` → 流式返回内容。
6. **兜底**：若 LLM 无内容则用模板；若**使用 Coze 提示词且返回非空**则直接展示（不做 isLikelyScriptContent 校验）；否则若 `!isLikelyScriptContent(content)` 用模板。
7. **合规**：`checkPlatformCompliance(content, ...)`，不通过则追加合规提示。
8. **可视化**：`injectVisualSuggestionsIntoContent`、`segmentForVisual`、`getVisualLegend`。
9. **翻译**：若 `userInput.language !== 'zh-CN'`，`translateLongText(content, userInput.language, 'zh-CN')`。
10. **响应**：SSE 推送 `data: { content?, done?, script?, error?, fallbackReason? }`。

## 五、配置与权限

- **配置来源**：环境变量（`SCRIPT_LLM_URL`、`SCRIPT_LLM_API_KEY`、`SCRIPT_LLM_MODEL` 等）或管理员在「管理员」页保存的库表配置（`system_config`）。
- **允许用户**：`system_config.script_llm_allowed_user_ids`；`null`/`*` 表示全体可用，否则为逗号分隔的 user ID 列表。
- **文档**：详见 [话术生成LLM配置说明](./话术生成LLM配置说明.md)、[话术生成-Coze入参对齐说明](./话术生成-Coze入参对齐说明.md)。

## 八、已做修复与注意点

1. **前端入参补全**：`GenerateScriptParams` 与 `generateScript`/`generateScriptStream` 的 body 已补全 `promotion_info`、`country`、`countryCode`、`custom_requirements`/`customRequirements`、`productSku`（同步接口也已带 productSku 等），与后端及 Coze 对齐。
2. **Coze 仅中文**：`buildCozeScriptPrompts` 中已明确「仅输出纯中文话术」；后端在 `language !== 'zh-CN'` 时对 Coze 返回的中文做翻译。
3. **无用占位符**：`scriptResearch.ts` 中已移除对已不存在的 `{{language}}` 的替换，避免误导。
4. **原生输出 + 截断**：话术部分不再对 Coze 输出做正文提取等额外处理，直接还原原生内容；仅当长度超过 10 万字符时截断并追加提示。
5. **同步接口**：`POST /api/ai/script` 当前**未调用 LLM**，仅使用 `synthesizeScript(research)` 模板话术；若需同步也走 LLM，需在路由中增加与流式类似的 LLM 调用与兜底逻辑。

## 九、常见问题定位

- **话术生成灰色/无权限**：检查 `script_llm_allowed_user_ids` 与当前用户 ID；未配置时全体可用。
- **一直模板话术**：检查是否配置 LLM、Coze 是否超时/空返回；看 `scriptOutputValidator` 是否判定跑题；可查 `coze-debug.log`、`coze-stream-debug.log`。
- **语言不对**：确认请求里 `language` 与前端界面语言一致；非中文时由后端翻译，若翻译失败会保留中文并设 `translationSkipped`。
- **入参不生效**：确认 body 字段名与上表一致（如 `promotion_info`、`custom_requirements`），后端优先 `promotion_info`、`custom_requirements`。
