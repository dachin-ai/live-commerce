# 角色与权限矩阵

角色已简化为**运营**与**管理**两类（主播已与运营合并）。本页写明两类角色各能访问的**菜单、页面、接口**，与《预留接口与上线说明》及当前前端显隐逻辑对应，便于模块 8～10 实现与验收。

---

## 一、角色定义

| 角色 | 说明 | 后端 role 值 | 前端判定 |
|------|------|--------------|----------|
| **运营** | 日常运营、看板/分析/执行工具，无用户管理与工作流 | `operator`、`user`（或后端统一为 operator） | `getCurrentUserRole() === 'operator'` 或非 admin |
| **管理** | 全部功能 + 用户管理 + 工作流 | `admin`、`manager` | `getCurrentUserRole() === 'admin' || getCurrentUserRole() === 'manager'`（侧栏「管理员」区、/admin、/workflow 均对两者开放） |

---

## 二、菜单与页面

| 菜单/入口 | 运营 | 管理 | 说明 |
|-----------|------|------|------|
| 店铺管理（Dashboard） | ✅ | ✅ | 看板、店铺切换、统计、待办、布局设置。**待办**：运营仅看本人待办；管理/经理查看某店铺时自动看到**该店铺下所有账号**生成的待办（历史汇总），列表展示创建人姓名便于区分。 |
| 数据分析 | ✅ | ✅ | 指标、趋势图、诊断建议；**店铺对比**仅运营/管理可见（当前实现为 isManager \|\| isOperator） |
| 执行工具 | ✅ | ✅ | 话术生成、报告、市场分析等；工具入口按 isAnchor/isOperator/isManager 显隐（主播已合并为运营，运营同管理可见范围） |
| 工作流 | ❌ | ✅ | 仅管理员可见侧栏「工作流」入口，路由 `/workflow` 仅 admin 可访问 |
| 用户管理 | ❌ | ✅ | 仅管理员可见侧栏「用户管理」与用户菜单内「用户管理」，路由 `/admin` 仅 admin 可访问 |
| 个人中心 | ✅ | ✅ | `/profile` |
| 消息 | 占位禁用 | 占位禁用 | 预留，当前不可用 |

## 三、接口权限（后端）

| 接口/能力 | 运营 | 管理 | 说明 |
|-----------|------|------|------|
| 登录、注册、找回/重置密码、/me、改资料/改密码 | ✅ | ✅ | 认证相关 |
| 店铺 CRUD、统计、任务、素材、数据导入、偏好、分类、区域 | ✅ | ✅ | 业务数据。**任务列表**：GET /api/tasks 对运营按 userId 过滤；对管理员/经理不过滤 userId，可查看任意店铺下全部用户的待办（历史记录）。 |
| 话术生成、待办智能生成、报告/分析/对比、品牌策略 | ✅ | ✅ | AI 能力（需登录） |
| 话术 LLM 配置 GET | ✅ | ✅ | 查看是否已配置 |
| 话术 LLM 配置 POST（保存） | ❌ | ✅ | `requireAdmin` |
| 话术规则配置 GET/PUT | ✅ 读 / ❌ 写 | ✅ | config script-rules 写需 `requireAdmin` |
| 版本日志 GET | ✅ | ✅ | 只读 |
| 版本日志 POST（创建） | ❌ | ✅ | `requireAdmin` |
| 用户管理 CRUD（/api/users） | ❌ | ✅ | `authenticate` + `requireAdmin`（admin 或 manager） |
| 工作流 trigger/rounds/outputs | ❌ | ✅ | `authenticate` + requireAdmin（admin 或 manager） |
| Agent 接口（/api/agent） | — | — | **已移除**；此前以 API Key 认证，现无该路由 |

## 四、与预留接口的对应关系

- **《预留接口与上线说明》** 中的 `isOperator`、`isManager`、`isAnchor`：  
  - 当前角色仅**运营**与**管理**，`isManager` 对应管理（admin/manager），`isOperator` 对应运营；`isAnchor` 已与运营合并，前端可按「运营」同一套显隐处理。
- **Dashboard**：运营与管理均可见待办、统计、布局；若后续要做「列/操作」差异化，用 `isOperator` / `isManager` 分支。
- **分析页**：店铺对比等已按 `isManager || isOperator` 显示，运营与管理均可见。
- **执行工具**：工具入口显隐已按角色；运营与管理当前可见范围一致，主播合并后无需单独 isAnchor 分支（除非再拆主播）。

---

## 五、验收要点（模块 8～10）

- 运营账号（user/operator）登录后：侧栏无「用户管理」「工作流」；访问 `/admin`、`/workflow` 被重定向到首页。
- 管理账号（admin 或 manager）登录后：侧栏有「用户管理」「工作流」；可正常访问 `/admin`、`/workflow`。
- 接口：非管理账号调用 POST 话术 LLM 配置、POST 版本日志、用户管理 CRUD、工作流 等应返回 403。
- 用户管理：支持四角色（普通用户、运营、经理、管理员）；注册时不允许自行选择 admin/manager，仅能由管理员在用户管理中创建或修改角色。
