# 功能场景与一键生成对应流程

用户只需在**对应功能场景下点击「生成」**，后台会自动运行该场景的完整 bot 逻辑，无需额外配置或步骤。**当前仅支持系统内入口**（`POST /api/ai/generate-tasks`、`POST /api/ai/script/stream` 等）。

---

## 1. 场景与入口一览

| 功能场景 | 用户操作（系统内） | 后端自动执行流程 |
|----------|--------------------|------------------|
| **智能待办** | 店铺管理 → 选择店铺 → 点击「智能生成」 | 拉取店铺与 30 天数据 → 阶段/趋势/节日/按日明细 → LLM（Coze Agent / OpenAI）或规则生成待办 → 去重与兜底 → 写库 |
| **话术生成** | 执行工具 → 话术生成（或创建脚本）→ 填写参数 → 点击「生成」 | 拼装 prompt → 按「LLM 调用方式」调用流式 LLM → 逐字返回话术 |
| **节日/大促查询** | 无单独系统内入口（可被待办/运营逻辑引用） | 按国家、年月查询即将节日（内部 `getUpcomingEvents`） |
| **异常分析待办** | 内置于「智能生成」流程 | 检测异常 → 若存在则调 LLM 生成 1～3 条异常相关待办，与主待办合并 |

---

## 2. 智能待办（一点即跑）

### 2.1 系统内

- **入口**：店铺管理页 → 选择店铺 → 待处理任务区域 → **「智能生成」** 按钮。
- **请求**：`POST /api/ai/generate-tasks`，body `{ storeId }`，需登录。
- **后端自动流程**：
  1. 校验店铺归属与任务归属（管理员为他人店铺生成时归属店铺主）。
  2. 清空该店铺下当前用户（或店铺主）的未完成待办。
  3. 调用 `generateSuggestedTodosForStore(storeId)`：
     - 拉取店铺信息、类目、已有待办标题、最近 30 天汇总与按日明细。
     - 若有「当天往前 30 天」或「系统最近 30 天」数据：调用 `generateIntelligentTodosWithLLM(...)`（按「LLM 调用方式」选 Coze/OpenAI）；若 LLM 返回空则用阶段规则兜底。
     - 若无数据：仅阶段规则任务。
     - 去重、日期判定（最近数据 >15 天则追加「请上传最近30天的运营数据」）、空结果兜底。
  4. 节日合并、同主题去重、主待办条数上限处理后**写入任务表**，返回本次生成条数及 LLM 状态。
- **用户**：仅需选店铺并点「智能生成」，无需配置 prompt 或接口。

---

## 3. 话术生成（一点即跑）

- **入口**：执行工具 → 话术生成（或创建脚本）→ 选择店铺、填写主题/时长等 → **「生成」**。
- **请求**：`POST /api/ai/script/stream`（流式），需登录且在使用 LLM 功能的允许名单内。
- **后端自动流程**：
  1. 读取「LLM 配置」与「LLM 调用方式」中**话术**所选方式（Coze Agent / OpenAI 兼容）。
  2. 拼装 system / user prompt，调用 `streamScriptFromLLM`，按所选方式发 Coze stream_run 或 OpenAI 流式请求。
  3. 逐块返回话术正文。
- **用户**：只需点「生成」，无需选择接口或写 prompt。

---

## 4. 异常分析（内置于智能待办）

- **入口**：无单独按钮；在「智能生成」流程中，若系统检测到该店铺存在数据异常，会**自动**调用 `analyzeAnomaliesWithLLM` 生成 1～3 条异常相关待办，与主待办合并。
- **后端**：复用「LLM 配置」与待办同一套「LLM 调用方式」（todo）；输出合并进本次智能待办结果。
- **用户**：无需任何操作，点「智能生成」即包含异常分析结果（若有）。

---

## 5. 统一说明

- **配置一次即可**：管理员在「LLM 配置」中填写 API 地址与密钥，并勾选「可使用 LLM 功能的用户」；用户在「LLM 调用方式」页按场景选择 Coze/OpenAI 等。此后在对应场景下**点击生成即可**，后台按上述流程自动运行。
- **智能待办**：系统内唯一入口为 `POST /api/ai/generate-tasks`，内部使用 `generateSuggestedTodosForStore`，按「LLM 调用方式」选 Coze Agent 或 OpenAI，失败则规则兜底并写库。
- **错误与无配置**：未配置 LLM 时，待办仍会生成（规则兜底）；话术会提示需配置。生成接口会返回 `metadata.llmStatus` / `llmStatusMessage`，便于前端提示「当前为规则待办」或「LLM 返回为空」等。
