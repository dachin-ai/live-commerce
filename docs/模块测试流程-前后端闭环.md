# 模块测试流程：前后端闭环

本文档对应流程图：**按模块实现后端 → MCP 测前端并对标市场方案 → 有问题则修后端再测 → 无问题则输出模块报告 → 人工终测**。  
做模块级测试/验收时，必须按此顺序执行。

---

## 步骤说明

### 1. 按模块实现后端功能代码

- **含义**：针对**当前模块**，完成或确认后端接口、数据、业务逻辑。
- **范围**：只做本模块，不扩大范围。
- **产出**：后端可运行、接口可调（如 `http://localhost:3000/health` 正常）。

### 2. 调用 MCP 测试前端，需对标当前模块的市场成品方案

- **含义**：用 **MCP 浏览器工具**（如 cursor-browser-extension）对前端做自动化测试。
- **对标**：测试用例需对齐《各模块步骤总结》里该模块的「对标样本」（如淘宝/抖音/飞书等成熟方案）。
- **执行**：按该模块的「测试步骤」逐项跑，记录通过/失败。
- **参考**：`docs/各模块步骤总结.md` 中各模块的「测试步骤」与「对标样本」。

### 3. 判断是否存在问题

- **存在问题**：任一步骤失败、结果与预期不符、或与市场方案不一致。
- **无问题**：该模块所有测试步骤通过且对标结论成立。

### 4. 存在问题 → 回到步骤 1（是）

- 修改与**当前模块**相关的后端或前端代码/配置。
- 修完后**重新从步骤 2** 用 MCP 测前端，直到「判断是否存在问题」为**否**。

### 5. 无问题 → 输出模块报告（否）

- 撰写**当前模块**的测试报告，包含：
  - 测试步骤与结果（通过/失败）
  - 对标结论（与市场方案的符合度）
  - 可选：MCP 操作记录、截图或引用
- 报告可写在 `docs/` 下该模块文档或《各模块步骤总结》对应小节。

### 6. 人工终测

- 报告输出后，由人工做最终确认，作为质量收口。

## 检查清单（每模块可复用）

每做一个模块，可按下面勾选，确保不跳过闭环步骤。

- [ ] **步骤 1**：当前模块后端已实现/确认，服务可访问
- [ ] **步骤 2**：已用 MCP 测前端，并按「对标样本」执行测试步骤
- [ ] **步骤 3**：已判断通过/失败，并记录
- [ ] 若失败：已修代码并重新执行步骤 2，直到通过
- [ ] 若通过：已输出该模块报告
- [ ] 流程进入人工终测
