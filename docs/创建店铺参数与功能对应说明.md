# 创建店铺参数与功能对应说明

本文档说明「创建店铺」时的各参数在系统各功能中的使用情况，便于维护与扩展。

## 一、创建店铺参数一览

| 参数 | 类型 | 说明 |
|------|------|------|
| name | string | 店铺名称（必填） |
| nameTh | string | 泰文名称（可选） |
| description | string | 店铺描述（可选） |
| platform | string | 平台：抖音/TikTok/淘宝/快手/微信视频号等 |
| userId | string | 归属用户（后端可自动） |
| region | string | 区域/国家，如 中国、泰国 |
| currency | string | 货币代码，如 CNY、THB |
| currencySymbol | string | 货币符号，如 ¥、฿ |
| minPrice | number | 价格区间下限（元） |
| maxPrice | number | 价格区间上限（元） |
| targetAudience | string | 目标人群描述 |
| brandPositioning | string | 品牌定位：无品牌/小型/中型/大型 |
| brandStrategy | string | 品牌影响力策略（可后续在待办中生成） |
| categoryIds | string[] | 店铺所属分类 ID 列表 |
| status | 'active' \| 'inactive' | 状态 |

**前端**：`CreateStoreModal` 使用 `CreateStoreData` 提交上述字段（含 `brandStrategy`）；创建成功后会自动创建两条待办（确认品牌定位、生成或确认品牌影响力策略）。

**后端**：`POST /stores` 接收并落库，并写入 `store_categories` 关联表。

## 二、各功能系统中的使用情况

### 2.1 智能任务生成（generate-tasks / ai-refactored）

- **使用字段**：name, platform, region, targetAudience, brandPositioning, brandStrategy, minPrice, maxPrice, currencySymbol，以及店铺分类（store_categories + categories 名称）。
- **用途**：按 storeId 查询店铺并组装 `storeInfo`，作为 Coze/LLM 生成待办任务时的上下文（平台、区域、人群、品牌、价格带、类目等）。

### 2.2 话术生成（ai/script）

- **使用字段**：name → storeName, platform, region, targetAudience，以及店铺分类（store_categories → categories 名称）、最近一条 stats（GMV/观看/订单/时长）。
- **用途**：`parseScriptRequestBody` 按 storeId 查店铺与类目，构建 `StoreContext`，供市调（scriptResearch）、合规（platformCompliance）、综合产出（scriptSynthesis）使用；用于话术类型、语种、平台禁词与品类话术规则。
- **已增强**：storeContext 已扩展 minPrice、maxPrice、brandPositioning、brandStrategy（若存在则写入 summaryForLLM/[店铺] 段），便于 LLM 生成更贴合价格带与品牌的话术。

### 2.3 数据导入（dataImport）

- **使用字段**：platform。
- **用途**：校验 `store.platform === 'TikTok' || '抖音'` 等，决定是否允许导入及审计记录。

### 2.4 统计与看板（stats + Dashboard）

- **使用字段**：storeId 用于按店铺拉取 stats；**前端**使用 `selectedStore.currency` 作为 `storeCurrencyCode`，用于金额展示与多币种换算（convertAmount、displaySymbol）。
- **说明**：stats 接口仅按 storeId 返回数值，不返回 currency；前端依赖已选店铺的 currency，满足当前看板展示需求。

### 2.5 前端展示与入口

- **Dashboard**：selectedStore.id / name / platform / currency；platform 控制「数据导入」等入口显隐；导出、导入弹窗传 store.id、store.name、store.platform。
- **StoreList**：展示 name、nameTh、region 等。
- **创建店铺待办**：待办描述中引用 formData 的 platform、region、minPrice、maxPrice、targetAudience、brandPositioning、categoryNames，用于引导运营完成「确认品牌定位」与「生成或确认品牌影响力策略」。

### 2.6 其他

- **分析页、素材、导出**：按 storeId 过滤数据；金额展示若需统一可继续使用 selectedStore.currency/currencySymbol。
- **nameTh**：目前仅在前端店铺列表等处展示，后端逻辑未使用。
- **description**：落库预留，当前未在业务逻辑中使用。

---

## 三、更新店铺（PUT /stores/:id）

当前 **仅支持** 更新：`name`、`description`、`platform`、`status`。

以下字段创建后**无法通过现有 API 修改**：region, currency, currencySymbol, minPrice, maxPrice, targetAudience, brandPositioning, brandStrategy, nameTh, categoryIds。若需支持「编辑店铺」时修改这些字段，需扩展 PUT 接口与前端编辑表单。

## 四、小结

| 参数 | 任务生成 | 话术生成 | 数据导入 | 统计/看板 | 前端展示/入口 |
|------|----------|----------|----------|-----------|----------------|
| name | ✓ | ✓ | - | - | ✓ |
| nameTh | - | - | - | - | 列表展示 |
| platform | ✓ | ✓ | ✓ 校验 | 入口显隐 | ✓ |
| region | ✓ | ✓ | - | - | 待办描述/可选展示 |
| currency / currencySymbol | - | 可选 | - | 前端金额 | 可选展示 |
| minPrice / maxPrice | ✓ | ✓（已接入 storeContext） | - | - | 待办描述 |
| targetAudience | ✓ | ✓ | - | - | 待办描述 |
| brandPositioning / brandStrategy | ✓ | ✓（已接入 storeContext） | - | - | 待办描述 |
| categoryIds / categories | ✓ | ✓ | - | - | 待办描述 |
| status | - | - | - | - | 列表/筛选 |

整体上，创建店铺参数已在**任务生成、话术生成、数据导入、看板与前端**中得到运用；话术侧已补充价格带与品牌相关字段，便于 LLM 生成更贴合店铺定位的话术。
