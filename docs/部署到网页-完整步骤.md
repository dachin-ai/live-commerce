# 部署到网页 - 完整步骤（留存）

本文是「直播电商中台」从本机代码发布到线上网页的**完整操作步骤**，按顺序执行即可。  
域名示例：**atlvbcsy.com**，服务器：火山云 ECS。

---

## 一、前提

- 本机项目目录：`D:\Work space\lvbcsym`（或你实际路径）
- 代码已推送到 Git 远程仓库（如 GitHub）
- 服务器上已有一份通过 `git clone` 拉取的项目，路径一般为 `/var/www/lvbcsym`
- 服务器已安装：Node.js、npm、PM2、Nginx，且 Nginx 已配置好站点

若**从未在服务器上部署过**，请先按 `docs/部署到域名操作指南.md` 完成「首次部署」（安装环境、git clone、PM2 启动后端、Nginx 配置），再回到本文做「日常更新」。

---

## 二、日常更新：把最新代码发布到网页

每次改完代码、要更新网站时，按下面两步做。

### 步骤 1：本机（Windows）— 推送代码

在 **PowerShell** 里打开项目目录并执行（整段复制）：

```powershell
cd "D:\Work space\lvbcsym"
git add .
git status
git commit -m "更新到最新版本"
git push
```

- 若提示 `nothing to commit, working tree clean`，说明当前没有未提交修改，可跳过本步或直接做步骤 2。
- 确保 `git push` 成功（无报错）。

---

### 步骤 2：服务器（ECS 终端）— 拉代码并构建

**不要在本机 PowerShell 里执行下面命令**，否则会报错（如 `Cannot find path`）。

1. 打开 **火山云控制台** → 进入你的实例 → **远程连接**。
2. 选择 **ECS 终端** → **立即登录**（在浏览器里会出现一个黑色终端）。
3. 在**浏览器里的终端**中，整段复制下面命令并粘贴执行：

```bash
cd /var/www/lvbcsym
git pull
npm run install:all
npm run build
pm2 restart lvbcsym-api
```

4. 等待命令执行完成（无报错即成功）。
5. 在浏览器中打开 **http://atlvbcsy.com**（或你的域名），按 **Ctrl+F5** 强刷，即可看到最新版本。

**若项目不在 `/var/www/lvbcsym`**：先在服务器上确认路径，例如执行 `ls /var/www/` 或 `find /var -name "package.json" -path "*lvbcsym*" 2>/dev/null`，把上面第一行 `cd` 换成实际路径即可。

## 三、常见问题

### 1. 构建报 43 个 TypeScript 错误

说明服务器上的代码还是旧版。请确认：

- 本机已执行过 **步骤 1** 且 `git push` 成功；
- 服务器上 **步骤 2** 里先执行了 `git pull`，再执行 `npm run build`。

若 `git pull` 提示 `Already up to date` 但构建仍报错，在服务器执行 `git log -1 --oneline` 和 `git remote -v`，确认远程地址和当前提交是否与本机一致。

### 2. 网页打开后 JS/CSS 被拦截（CORS / address space unknown）

在服务器上编辑 Nginx 站点配置（路径可能是 `/etc/nginx/conf.d/lvbcsym.conf` 或 `/etc/nginx/sites-available/lvbcsym`），在 `server { ... }` 里与 `root`、`index` 同级增加一行：

```nginx
add_header Cross-Origin-Resource-Policy "same-origin" always;
```

保存后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

然后浏览器强刷（Ctrl+F5）再试。更多说明见 `部署到域名操作指南.md` 的「常见问题」小节。

### 3. 本机执行了服务器命令导致报错

以 `/var/www/lvbcsym` 开头的命令是 **Linux 服务器路径**，只能在 **ECS 终端（浏览器里）** 执行。本机 PowerShell 只做 `cd "D:\Work space\lvbcsym"` 和 `git add/commit/push`。

## 四、流程小结

| 在哪里       | 做什么                         |
|--------------|--------------------------------|
| 本机 PowerShell | `cd` 项目目录 → `git add` → `git commit` → `git push` |
| 浏览器 ECS 终端 | `cd /var/www/lvbcsym` → `git pull` → `npm run install:all` → `npm run build` → `pm2 restart lvbcsym-api` |
| 浏览器       | 打开域名，Ctrl+F5 强刷看最新版 |
