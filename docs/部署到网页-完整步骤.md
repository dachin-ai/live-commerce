# 部署到网页 - 完整步骤（留存）

本文是「直播电商中台」从本机代码发布到线上网页的**完整操作步骤**，按顺序执行即可。  
域名示例：**atlvbcsy.com**，服务器：火山云 ECS。

---

## 一、前提

- 本机项目目录：`D:\Work space\lvbcsym`（或你实际路径）
- 代码已推送到 Git 远程仓库（如 GitHub）
- 服务器上已有一份通过 `git clone` 拉取的项目，路径一般为 `/var/www/lvbcsym`
- 服务器已安装：Node.js、npm、PM2、Nginx，且 Nginx 已配置好站点

若**从未在服务器上部署过**，请先按 `docs/部署到域名操作指南.md` 完成「首次部署」（安装环境、git clone、PM2 启动后端、Nginx 配置），再回到本文做「日常更新」。

---

## 二、上传方式（如何保证完整推送到远程）

**常见问题**：服务器上构建报错或看不到新功能，多半是**本机没有把修改完整推送到远程**，服务器 `git pull` 拿到的仍是旧代码。

**正确上传方式：**

1. **先保存所有文件**  
   在编辑器里保存你改过的每一个文件（Ctrl+S），避免只提交了部分改动。

2. **本机完整推送（顺序不要颠倒）**  
   在 **PowerShell** 里打开项目目录，**按顺序**执行：

   ```powershell
   cd "D:\Work space\lvbcsym"
   git add .
   git status
   ```

   - 看 `git status` 输出：若有 `Changes to be committed` 或 `Untracked files`，说明有内容会被提交。
   - 若显示 `nothing to commit, working tree clean`，说明当前工作区没有未提交修改；若你刚改过代码，请确认已保存且在该目录下（不是别的拷贝）。

   接着执行：

   ```powershell
   git commit -m "更新到最新版本"
   git push origin main
   ```

   - 必须看到推送成功（例如 `Writing objects: 100%`、`main -> main` 或 `Everything up-to-date`），才表示远程已是最新。
   - 若出现 `Recv failure: Connection was reset` 等网络错误，多试几次 `git push origin main`，或检查网络/代理后再推。
   - `git add` 时的 LF/CRLF 警告可忽略，不影响提交和推送。

3. **服务器在本机 push 成功后再拉取**  
   先在本机完成上面的 `git push origin main` 且无报错，再到服务器 ECS 终端执行 `git pull origin main`。否则服务器拉到的还是旧提交。

## 三、日常更新：把最新代码发布到网页

每次改完代码、要更新网站时，按下面两步做。

### 步骤 1：本机（Windows）— 推送代码

在 **PowerShell** 里打开项目目录并执行（整段复制）：

```powershell
cd "D:\Work space\lvbcsym"
git add .
git status
git commit -m "更新到最新版本"
git push origin main
```

- 若提示 `nothing to commit, working tree clean`，说明当前没有未提交修改，可跳过本步或直接做步骤 2。
- **务必确认 `git push origin main` 成功（无报错）**，这样远程才有最新代码，服务器 pull 才能拿到。
- 若 push 报错 `Recv failure: Connection was reset`，为网络问题，可稍后重试 `git push origin main`。

---

### 步骤 2：服务器（ECS 终端）— 拉代码并构建

**前提**：本机已执行步骤 1 且 `git push` 成功，这样服务器拉取的才是最新代码。

**不要在本机 PowerShell 里执行下面命令**，否则会报错（如 `Cannot find path`）。

1. 打开 **火山云控制台** → 进入你的实例 → **远程连接**。
2. 选择 **ECS 终端** → **立即登录**（在浏览器里会出现一个黑色终端）。
3. 在**浏览器里的终端**中，整段复制下面命令并粘贴执行：

```bash
cd /var/www/lvbcsym
git pull origin main
git log -1 --oneline
npm run install:all
npm run build
pm2 restart lvbcsym-api
```

4. 等待命令执行完成（无报错即成功）。可用 `git log -1 --oneline` 核对提交号是否与本机一致（本机可用 `git log -1 --oneline` 查看）。
5. 在浏览器中打开 **http://atlvbcsy.com**（或你的域名），按 **Ctrl+Shift+R** 或 **Ctrl+F5** 强刷，即可看到最新版本。

**若项目不在 `/var/www/lvbcsym`**：先在服务器上确认路径，例如执行 `ls /var/www/` 或 `find /var -name "package.json" -path "*lvbcsym*" 2>/dev/null`，把上面第一行 `cd` 换成实际路径即可。

## 四、常见问题

### 1. 构建报 43 个 TypeScript 错误（或服务器上看不到新功能）

通常是因为**没有完整推送**，服务器上的代码还是旧版。请确认：

- 本机改完代码后**已保存**，再执行 **步骤 1**（`git add .` → `git status` 确认有变更 → `git commit` → `git push`），且 **`git push` 成功**；
- 在**本机 push 成功之后**，再到服务器执行 **步骤 2**，且先执行 `git pull` 再执行 `npm run build`。

若 `git pull origin main` 提示 `Already up to date` 但构建仍报错或网页无更新，在服务器执行 `git log -1 --oneline`，与本机 `git log -1 --oneline` 对比；若提交号不一致，说明本机最新提交尚未推送到远程，请在本机再次执行 `git push origin main` 并确认成功后再到服务器 pull。

### 2. 网页打开后 JS/CSS 被拦截（CORS / address space unknown）

在服务器上编辑 Nginx 站点配置（路径可能是 `/etc/nginx/conf.d/lvbcsym.conf` 或 `/etc/nginx/sites-available/lvbcsym`），在 `server { ... }` 里与 `root`、`index` 同级增加一行：

```nginx
add_header Cross-Origin-Resource-Policy "same-origin" always;
```

保存后执行：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

然后浏览器强刷（Ctrl+F5）再试。更多说明见 `部署到域名操作指南.md` 的「常见问题」小节。

### 3. 本机 push 报错 Recv failure: Connection was reset

为连接 GitHub 时网络中断，可稍后重试 `git push origin main`，或检查本机网络/代理。推送成功后再到服务器执行 `git pull origin main`。

### 4. 本机执行了服务器命令导致报错

以 `/var/www/lvbcsym` 开头的命令是 **Linux 服务器路径**，只能在 **ECS 终端（浏览器里）** 执行。本机 PowerShell 只做 `cd "D:\Work space\lvbcsym"` 和 `git add` / `git commit` / `git push origin main`。

### 5. 本机功能正常，部署到网页后没有（如权限/虚拟管理员仍 403）

- **先确认代码已到服务器**：本机 `git push origin main` 成功 → 服务器执行 `git pull origin main`，再在**本机**和**服务器**分别执行 `git log -1 --oneline`，两处提交号一致。
- **再确认构建与进程**：服务器依次执行 `npm run install:all`、`npm run build`、`pm2 restart lvbcsym-api`，无报错。
- **浏览器强刷**：打开域名后按 **Ctrl+Shift+R** 或 **Ctrl+F5**，避免旧缓存。
- **权限在服务器数据库**：权限配置存在服务器数据库，不会从本机同步。若服务器是新环境或重装过，需在**网页上**用管理员登录 → 权限配置 → 重新勾选「可使用 LLM 的用户」和「能够使用的功能」并保存。

## 五、流程小结

| 在哪里       | 做什么                         |
|--------------|--------------------------------|
| 本机 PowerShell | `cd` 项目目录 → `git add` → `git status` → `git commit` → `git push origin main`（失败则重试） |
| 浏览器 ECS 终端 | `cd /var/www/lvbcsym` → `git pull origin main` → `npm run install:all` → `npm run build` → `pm2 restart lvbcsym-api` |
| 浏览器       | 打开域名，Ctrl+F5 强刷看最新版 |
