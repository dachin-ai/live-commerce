# 启动与数据库操作指南

## 🚀 启动项目（最简单）

### ⭐ 推荐方式：双击 `快速启动.bat`

1. 找到根目录的 `快速启动.bat`
2. **双击运行**
3. 会自动打开两个窗口：
   - 前端窗口（http://localhost:5173）
   - 后端窗口（http://localhost:3000）
4. 等待启动完成即可

**优点**：最简单，无需选择菜单，直接启动

### 其他启动方式

- **PowerShell菜单**：右键 `启动.ps1` → "使用PowerShell运行"（功能完整）
- **命令行**：`npm run dev`（最可靠）

## 💾 数据库操作（PostgreSQL 18）

> ⚠️ 自 2026-04-30 起，系统已从 SQLite 迁移至 **PostgreSQL 18**（本地压缩包部署）。

### 前置条件：启动 PostgreSQL

后端启动前需确保 PostgreSQL 服务已运行：

```powershell
# 启动 PostgreSQL
& "pgsql\bin\pg_ctl.exe" -D "pgdata" -l "pgdata\server.log" start

# 停止 PostgreSQL
& "pgsql\bin\pg_ctl.exe" -D "pgdata" stop

# 检查状态
& "pgsql\bin\pg_ctl.exe" -D "pgdata" status
```

### 连接信息

| 项目 | 值 |
|------|-----|
| 主机 | `localhost` |
| 端口 | `5432` |
| 数据库名 | `live_commerce` |
| 用户名 | `lvbcsym` |
| 密码 | `lvbcsym2026`（见 `backend/.env`） |

### 更新种子数据（不丢失业务数据）⭐ 推荐

1. 打开 PowerShell，进入项目目录
2. 执行：`cd backend; npm run db:update-seed`
3. 完成（保留所有业务数据）

### 重置数据库（清空所有数据）

```powershell
# 用 postgres 超级用户连接，清空并重建 schema
& "pgsql\bin\psql.exe" -U postgres -d live_commerce -c "DROP SCHEMA public CASCADE; CREATE SCHEMA public; GRANT ALL ON SCHEMA public TO lvbcsym;"
# 重启后端即可自动建表 + 插入种子数据
```

### 备份数据库

```powershell
# 导出完整备份（SQL 文件）
& "pgsql\bin\pg_dump.exe" -U lvbcsym -d live_commerce -f backup.sql

# 恢复备份
& "pgsql\bin\psql.exe" -U lvbcsym -d live_commerce -f backup.sql
```

### 直接查询数据库

```powershell
& "pgsql\bin\psql.exe" -U lvbcsym -d live_commerce
# 常用命令：\dt 列表、\d tablename 表结构、\q 退出
```

## 🔧 修改数据库种子数据

### 步骤（简化版）

1. **编辑数据**：修改 `backend/src/db.ts` 中的种子数据
2. **更新数据库**：执行 `cd backend && npm run db:update-seed`
3. **完成**：无需重启后端，数据已更新

### 注意事项

- **分类数据**：如果分类表已有数据，`Update Seed Data` 会跳过插入。如需更新，请先用 psql 重置数据库（见上方「重置数据库」命令）再重启后端。
- **其他种子数据**：用户、版本日志等会在表为空时自动插入。

```json
{

    "db:reset": "node -e \"...\""
  }
}
```
