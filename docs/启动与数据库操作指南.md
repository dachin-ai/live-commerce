# 启动与数据库操作指南

## 🚀 启动项目（最简单）

### ⭐ 推荐方式：双击 `快速启动.bat`

1. 找到根目录的 `快速启动.bat`
2. **双击运行**
3. 会自动打开两个窗口：
   - 前端窗口（http://localhost:5173）
   - 后端窗口（http://localhost:3000）
4. 等待启动完成即可

**优点**：最简单，无需选择菜单，直接启动

### 其他启动方式

- **PowerShell菜单**：右键 `启动.ps1` → "使用PowerShell运行"（功能完整）
- **命令行**：`npm run dev`（最可靠）

## 💾 数据库操作（已优化）

### 更新种子数据（不丢失业务数据）⭐ 推荐

**之前**：需要删除数据库 → 重启后端 → 重新创建（3-4步）

**现在**：
1. 打开 PowerShell，进入项目目录
2. 执行：`cd backend && npm run db:update-seed`
3. 完成（保留所有业务数据）

**适用场景**：
- 修改了 `backend/src/db.ts` 中的分类数据
- 需要更新种子数据但不想丢失现有业务数据

### 重置数据库（清空所有数据）

1. 删除 `backend\data.db` 文件
2. 重启后端服务（会自动创建新数据库）

**适用场景**：
- 需要完全清空数据重新开始
- 数据库结构发生重大变化

### 备份数据库

手动复制 `backend\data.db` 文件即可

### 重启后看不到之前的数据？

**数据保存在哪里**：所有店铺、统计、任务都保存在 **`backend\data.db`** 这一个文件里。重启前后端**不会**删除这个文件。

**请先确认**：
1. 启动后端时，控制台会打印一行 **「📂 数据库文件: …」**，请确认路径指向你项目下的 `backend\data.db`（例如 `d:\Work space\lvbcsym\backend\data.db`）。
2. 若路径不对（例如指向了别的目录），请用「快速启动.bat」或在本项目根目录执行 `npm run dev`，确保从**本项目**启动。
3. 若你曾手动删除、重命名或替换过 `backend\data.db`，历史数据会丢失；可从 `backend\data.db.backup` 恢复（如有备份）。

**按店铺区分**：  
- **直播数据统计**（GMV、时长、趋势图）来自「数据导入」时写入的 `stats` 表，**按店铺**存储。  
- 若你之前只为「测试店铺-抖音美妆」导入过数据，选「Greenpet」时统计会为 0，这是正常的。  
- 请在上方下拉框切换到**曾导入过数据的店铺**，或为当前店铺重新执行一次「导入数据」。

## 🔧 修改数据库种子数据

### 步骤（简化版）

1. **编辑数据**：修改 `backend/src/db.ts` 中的种子数据
2. **更新数据库**：执行 `cd backend && npm run db:update-seed`
3. **完成**：无需重启后端，数据已更新

### 注意事项

- **分类数据**：如果分类表已有数据，`Update Seed Data` 会跳过插入。如需更新，请先删除 `backend\data.db` 再重启后端。
- **其他种子数据**：用户、版本日志等会在表为空时自动插入。

## 📝 技术实现

### 数据库迁移系统

- 文件：`backend/src/db-migrate.ts`
- 功能：版本管理、种子数据更新、迁移记录

### NPM 脚本

在 `backend/package.json` 中：
```json
{
  "scripts": {
    "db:update-seed": "tsx src/db-migrate.ts",
    "db:reset": "node -e \"...\""
  }
}
```
